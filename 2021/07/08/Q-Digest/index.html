Q-Digest

Posted on 2021-07-08 Edited on 2021-07-09

Problem

Let d∈N+d \in \N^+d∈N+ be a positive integer and f=(0,…,0)∈Rdf = (0, \ldots, 0) \in \R^df=(0,…,0)∈Rd be an all-zero vector.
There is a sequence SSS of updates, each of which consists of a pair (x,w)∈[d]×R+(x, w) \in [d] \times \R^+(x,w)∈[d]×R+, increasing f[x]f[x]f[x] by www.

Model Description

  1. f←(0,…,0)∈Rdf \leftarrow (0, \ldots, 0) \in \R^df←(0,…,0)∈Rd;
  2. For (x,w)∈S(x, w) \in S(x,w)∈S do
  3. \qquad f[x]←f[x]+wf[x] \leftarrow f[x] + wf[x]←f[x]+w;

After witnessing the stream SSS, we call fff the frequency vector.
We are interested in answering the following query.

Rank Query
Input： x∈[d]x \in [d]x∈[d]
Returen: r[x]←∑i=1x−1f[i]r[x] \leftarrow \sum_{i = 1}^{x - 1} f[i]r[x]←∑i=1x−1​f[i].

We formally define the problem of approximate rank estimation.

Problem. Let ϵ\epsilonϵ be an error parameter.
Design an algorithm, such that
for a rank query with a given element x∈[d]x \in [d]x∈[d],
the summary will return an approximate rank r^[x]\hat r [x]r^[x], such that

r[x]−ϵ⋅W≤r^[x]≤r[x]+ϵ⋅W. r[x] − \epsilon \cdot W \le \hat r [x] \le r[x] + \epsilon \cdot W. r[x]−ϵ⋅W≤r^[x]≤r[x]+ϵ⋅W.

QQQ-Digest

The naive solution maintain the vector f∈Rdf \in \R^df∈Rd explicitly.
Assuming that each entry of fff can always be stored in one word, the space usage of the is O(d)O(d)O(d).

Theorem. There exists an algorithm, QQQ-digest, that has space usage of O(1ϵlog⁡d)O( \frac{1}{\epsilon} \log d)O(ϵ1​logd) words, insertion time O(log⁡d)O(\log d)O(logd) and query time of O(log⁡d)O(\log d)O(logd).

We first study a prototype which requires O(d)O(d)O(d) words.

Prototype

The algorithm maintains an array of size ddd to keep all values of fff.
Further, it construct a binary tree on the array, such that

  1. Each node vvv keeps the sum of frequencies of leaves in its subtree, denote as s[v]s[v]s[v].

By definition, if vvv is a leaf node, then s[v]s[v]s[v] equals the frequency of the node.
For each node vvv, denote L[v]L[v]L[v] and R[v]R[v]R[v] its left and right children respectively.

Query

The query algorithm is also very simple.
Given an input xxx, the algorithm walks down the path from the root to the leaf that contains xxx.
The rank of xxx equals the sum of of s[v]s[v]s[v] for the vvv such that

  1. it is a left child of some node on the path;
  2. it is not on the path.

QQQ-Digest

In this section, we show how to reduce the space usage to O(1ϵlog⁡d)O(\frac{1}{\epsilon} \log d)O(ϵ1​logd).

In the previous section, when the algorithm perform a rank query for x∈[d]x \in [d]x∈[d], it walks down a path (v1,…,vlog⁡d)(v_1, \ldots, v_{\log d} )(v1​,…,vlogd​), where v1v_1v1​ is the root of the tree, and vlog⁡dv_{\log d}vlogd​ is the leaf node that contains xxx.

There are log⁡d\log dlogd nodes in the path in total.
If each node introduces an error of O(ϵW/log⁡d)O(\epsilon W / \log d)O(ϵW/logd) to the estimation of r[x]r[x]r[x], then the final estimation has error bounded by O(ϵW)O(\epsilon W)O(ϵW).

This motivates to associate each node vvv in the tree with an error counter c[v]c[v]c[v], which is interpreted as the maximum error it can introduce to the rank estimation to a given element xxx.
It is initialized with 000 and is guaranteed to be bounded by some threshold CCC (except the leaf node).
The threshold CCC is updated along with the insertion of elements into the tree.
The algorithm maintains the invariant that

ϵ⋅W2log⁡d≤C≤ϵ⋅Wlog⁡d. \frac{\epsilon \cdot W}{2 \log d} \le C \le \frac{\epsilon \cdot W}{\log d}. 2logdϵ⋅W​≤C≤logdϵ⋅W​.

Under this invariant, the estimation of r[x]r[x]r[x] is always bounded by ϵ⋅W\epsilon \cdot Wϵ⋅W.

When an update (x,w)(x, w)(x,w) arrives, the algorithm walks through the path from the root to the leaf that contains xxx.
For each internal node vvv in the path, the amount of additional error it can make is C−c[v]C - c[v]C−c[v].

Intuitively, we insert xxx into this node.

Doing so increase the rank estimation error only for the element greater than xxx, but not for the elements less than or equal to xxx.

Maintenance

Finally, we discuss how to update the tree in a top-down approach.
For a node vvv, we try to increase c[v]c[v]c[v] to CCC, by borrowing weights from the its descendent.
There are two possible cases.
First, if s[v]≤Cs[v] \le Cs[v]≤C, then we can set c[v]=Cc[v] = Cc[v]=C and delete all descendants of vvv.

Space Usage

A node is called “saturated” if c[v]=Cc[v] = Cc[v]=C.
By the invariant of CCC, there are at most W/C=O((1/ϵ)log⁡d)W / C = O((1 / \epsilon ) \log d)W/C=O((1/ϵ)logd) “saturated” nodes.
By the algorithm, each node vvv in the tree with nonzero c[v]c[v]c[v], either “saturated”, or has a “saturated” parent.
However, each “saturated” can have at most two children.
Therefore, the total number of nodes with nonzero c[v]c[v]c[v] is bounded by O((1/ϵ)log⁡d)O((1 / \epsilon ) \log d)O((1/ϵ)logd).
