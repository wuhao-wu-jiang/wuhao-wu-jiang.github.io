K-Connected Spanning Subgraph

Posted on 2015-11-23 Edited on 2015-12-01

K-Connected Spanning Subgraph

Background

Consider that you are given a transportation network, which is a graph. Now you are asked to find out a sub-graph that the transportation capacity between any two sites in the sub-graph is above some threshold k.

This sub-graph could be viewed as kind of a backbone. Its importance is as follows: even if some lines in the transportation network break down, so long as the lines of this sub-graph operates, we can guarantee that the network is able to transport k capacity cargo between any two sites.

If we examine this problem deeply, we may ask the following questions:

  1. Does such a sub-graph exist?
  2. If such a sub-graph exist, how to find one with the minimum number of edges?

Solution

Existence could be answered right now. If the minimum cut of the graph is no less than k, then we could find such a sub-graph. A trivial case is the graph itself. But the second one is difficult. It is believed that this is a NP hard problem.

Approximation algorithms are proposed. One of them was given by Hiroshi Nagamochi and Toshihide ibaraki [A Linear-Time Algorithm for Finding a Sparse k-Connected Spanning Subgraph of a k-Connected Graph] in 1992.

We firstly formalize the notions. To simplify the problem, we focus on graphs that are undirected and consist of no multiple edges and self loops.

k-Connected Graph: A graph is k-Connected if the minimum cut of the graph is no less than k.

Nagamoci-Ibaraki Algorithm Given a graph $G=<V,E>$, if $F_i = <V, E_i>$ is the maximal spanning forest of $G = <V, E - E_1 - E_2 - .. - E_{i - 1}>$, the $G_i = <V, E_1 \cup E_2 \cup … \cup E_{i - 1}>$ is a k-Connected Sub-graph with $k|V| - k(k + 1)$ edges if $k \le \lambda(G)$, where $\lambda(G)$ is the minimum cut of G.

Here only a scratch of proof is given. Consider a particular cut C in G, whose value $\lambda(C)$ is no less than $\lambda(G)$. For any $G_i$ with $i <= \lambda(G)$, it must consist of at least one edge of C. Otherwise we could add an arbitrary edge in$C-E_1-E_2-…-E_{i - 1}$ to $F_i$ such that $F_i$ is still a forest, contradicting $F_i$ is maximal.

Finding a maximal forest in $G=<V,E>$ requires $O(|V| + |E|)$ time. So $O(k(|V| + |E|))$ is required for k-Connected Graph. Nagamochi and Ibaraki did it in $O(|V| + |E|)$ time. They scaned all nodes and edges once and classify edges into different $E_i$, here $i \in [1…|E|]$.

Pseudo Code:

  1. h(v) = 0, $\forall v \in V$

  2. $E_i = \emptyset, \forall i \in [1..|E|] $

  3. while $\exists u$ that has not been scanned

    • find u with maximum h(u)

    • marked u as scanned

    • for all $(u,v) \in E$ that has not been scanned

      • marked (u,v) as scanned.
      • if h(u) == h(v) then
        • ++h(u)
      • end if
      • ++h(v)
      • $E_{h(v)} = E_{h(v)} \cup {(u,v)}$

    • end for

      end while

    Analysis

To see that the algorithm indeed works, we have to check two things:

  1. $F_i=<V, E_i>$ is a forest, i.e., $F_i$ does not contain a cycle.
  2. $F_i=<V, E_i>$ is maximal.

$F_i$ doesn’t contain a cycle: suppose that when u is being scanned and (u,v) is added to $E_i$, a cycle is created. Then some edge incident to v belongs to $F_i$. Thus h(v) >= i before (u, v) is added to $E_i$. But (u,v) is added to $E_{h(v)}$ after $’++h(v)’$. So $(u,v) \notin E_i$. A contradiction.

$F_i$ is maximal. If $F_i$ is not maximal, there is some edge $(u,v) \in E_j$ for some $j > i$ such that $<V, E_i \cup {(u,v)}>$ is a forest. Denote the tree containing u in $F_i$ by $T_u$ and the one containing v in $F_i$ by $T_v$. Further, denote the order of nodes in $T_u$ and $T_v$ being scanned by $[x_1, x_2,…, u,…]$ $[y_1,y_2,…v,…]$ respectively. Without losing generality, suppose $x_1$ is scanned before $y_1$. When $x_1$ is scanned, we must have $h(x_1) == i - 1$ and $h(y_1) \le i - 1$. After $x_1$ is scanned, $h(x_1) == h(x_2) ==i$. For any other node $z \ne {x_1}$ such that $h(z) == i$, we guarantee that $h(z) == i -1$ before $x_1$ is scanned and $z \in [x_2, x_3,…]$, otherwise z is scanned before $x_1$. Now $h(x_2) =i > h(y_1)$, it’s scanned before $y_1$. Following this way, u is scanned before $y_1$. When u is scanned, $h(v) \le i - 1$. So (u, v) is add to some $E_j$ for $j \le i$. A contradiction.

The number of edges in $G_i$

Suppose the order of nodes being scanned in V is $v_1, v_2, ..$. The value of $h(v_i)$ could not exceed i, since there are at most one edge $(v_j, v_i)$ incident to $v_i$ for some j less than i. So $v_i$ does not appear in forest $F_k$ for $k > i$. Then $F_k$ contains at most |V| - 1 - (k - 1) = |V| - k edges. Thus the number of edges in $G_i$ is at most
$$
\Sigma_{i = 1}^{k} (|V| - i) = k|V| - k(k + 1) /2
$$

Connectivity of a Graph

Posted on 2015-10-16 Edited on 2018-07-02

Suppose the edge connectivity of a graph G is c. In the previous article we know that the number of cuts with weight up to kc is bounded by $O((2n)^{2k})$, where k is a real number and n is the number of vertices in G.

Let $f(x)$ to be the number of cut with value xc and define $F(k) = \Sigma_{x \le k} f(x)$. From the discussion above, $F(k) \le \lambda (2n)^{2k}$ for some parameter $\lambda$.

If we remove each edge with probability p, then the graph becomes disconnected with probability at most $P[Disconnected] = \Sigma_{x} f(x)p^{xc} $

To maximize $P[Disconnected]$, we take $F(k) = \lambda (2n)^{2k}$ for every k.

By the definition of $F(k)$, $f(k) = \lambda ((2n)^{2k} - (2n)^{2(k-1)})$.

Then, for $k \ge 2$,
$$
\begin{aligned}
f(k) p^{kc} & = \lambda ((2n)^{2k} - (2n)^{2(k-1)}) p^{kc} \
& = \lambda (\int_{k -1}^{k} {\partial (2n)^{2x} \over \partial x} dx) * p^{kc} \\
& \le \lambda \int_{k -1}^{k} {\partial(2n)^{2x} \over \partial x} p^{xc}dx
\end{aligned}
$$

Sum up this inequality over k,
$$
\begin{aligned}
P[Disconnected] & \le f(1)p^c + \lambda \int_1^\infty {\partial (2n)^{2x} \over \partial x} p^{xc} dx \
& = \lambda (2n)^2 p^c - \lambda {2\ln 2n \over 2 \ln 2n + c \ln p} (2n)^2 p^c
\end{aligned}
$$

If $p = (2n)^{-{a \over c} }$ for some parameter $a$, then $P[Disconnected] = O(n^2 p^c / (a - 2)) = O(n^{2 - a} / (a -2))$. When a = 3, the graph is disconnected with probability O(1/n). (p.s: for large c, $p \approx 1 - {a \ln 2n \over c} $)

Chernoff Bounds

Posted on 2015-10-10 Edited on 2021-01-21

Last Updated: Jan-21st-2021

Let $X_i \in { 0, 1 }: i \in [n]$ be a set of $n$ independent random variables with $\mathbb{E} [X_i] = \mu_i : i \in [n]$. Define
$$
\begin{aligned}
X &\doteq \sum_{i \in [n] } X_i, \
\mu &\doteq \mathbb{E} \left[ \sum_{i \in [n] } X_i \right] = \sum_{i \in [n] } \mu_i.
\end{aligned}
$$

Theorem 1 For any $0 < \epsilon < 1$,
$$
\Pr \left[ X \ge (1 + \epsilon) \cdot \mu \right] \le e ^ {- \epsilon^2 \mu / 3}.
$$

Proof:
Noting that $e^x$ is a non-decreasing function and by applying Markov’s Inequality, for any t > 0, we obtain
$$
\begin{aligned}
\Pr \left[ X \ge (1 + \epsilon) \cdot \mu \right]
& = \Pr \left[ e^{tX} \ge e^{ (1 + \epsilon) t \mu } \right] \
& \le \frac{\mathbb{E} \left[ e^{tX} \right] }{e^{(1 + \epsilon)t\mu} }.
\end{aligned}
$$

From independence of $X_i$, we get
$$
\begin{aligned}
\mathbb{E} \left[ e^{tX} \right]
&= \prod \mathbb{E} \left[ e^{t X_i} \right] \
& = \prod (e^t \mu_i + (1 - \mu_i)) \
& = \prod(1 + \mu_i (e^t -1)) \
&\le \prod e^{ \mu_i (e^t - 1)} \
&= e^{\mu (e^t -1)}.
\end{aligned}
$$

Hence:
$$
\Pr \left[ X \ge (1 + \epsilon) \cdot \mu \right] \le \frac{\mathbb{E} \left[ e^{tX} \right] }{ e^{ (1 + \epsilon)t\mu} } = \frac{e^{\mu (e^t -1)} }{e^{(1 + \epsilon)t\mu} }.
$$

By minimizing the inequality over t, we have
$$
\Pr[X \ge (1 + \epsilon)\mu] \le \underset{t > 0}{\min } \frac{e^{\mu (e^t -1)} }{e^{(1 + \epsilon)t\mu} }.
$$

It can be shown that when $t = \ln (1 + \epsilon )$, the bound is tightest:
$$
\Pr[X \ge (1 + \epsilon)\mu] \le e ^ {\mu ( \epsilon - ( 1 + \epsilon) \ln(1 + \epsilon))} = \left( \frac{e^\epsilon}{ (1 + \epsilon)^{1 + \epsilon} } \right)^\mu.
$$

By Taylor series expansion for $\ln(1 + \epsilon)$,
$$
\begin{aligned}
(1 + \epsilon)\ln(1 + \epsilon) &= \epsilon + \sum_{i=2}^{\infty} (-\epsilon)^i \left( \frac{1}{i - 1} - \frac{1}{i} \right) \
& \ge \epsilon + \epsilon^2
\frac{1}{2} - \epsilon^3 \frac{1}{6} \
& \ge \epsilon + \epsilon^2 \frac{1}{3}.
\end{aligned}
$$

Substitute it back into the inequality, we obtain the desired result:
$$
\Pr[X \ge (1 + \epsilon)\mu] \le e ^ {- \epsilon^2 \frac{\mu}{3} }.
$$

Theorem 2 For any $0 < \epsilon < 1$,
$$
\Pr[X \le (1 - \epsilon)\mu] \le e ^ {- \epsilon^2 \frac{\mu}{2} }.
$$
Proof:
As before,
$$
\begin{aligned}
\Pr \left[ X \le (1 - \epsilon) \mu \right]
& = \Pr \left[ e^{-tX} \ge e^{-(1 - \epsilon) t \mu} \right] \
& \le \frac{ \mathbb{E} \left[ e^{-tX} \right] }{e^{-(1 - \epsilon) t \mu} } \
& \le \frac{e^{ (1 - \epsilon) t \mu} }{e^{ \mu(e^t -1) } }.
\end{aligned}
$$

Let $t = \ln (1 + \epsilon )$,
$$
\Pr \left[ X \le (1 - \epsilon) \cdot \mu \right] \le
e ^ {\mu ( ( 1 - \epsilon) \ln(1 + \epsilon) - \epsilon ) }.
$$

Applying Taylor expansion, we get
$$
\begin{aligned}
(1 - \epsilon)\ln(1 + \epsilon) &= \epsilon + \sum_{i=2}^{\infty} (-\epsilon)^i \left(- \frac{1}{i - 1} + \frac{1}{i} \right) \
& \ge \epsilon - \epsilon^2 \frac{1}{2}.
\end{aligned}
$$

Finally,
$$
\Pr \left[ X \le (1 - \epsilon) \cdot \mu \right] \le e ^ {- \epsilon^2 \frac{\mu}{2} }.
$$

$\square$

Notes on Introduction to Complexity

Chapter Two
Context-Free Grammar

A context-free grammar is a four-tuple $\langle N, T, S, R\rangle$, where

  • N is a finite set of symbols called the variables(non-terminals),
  • T is a finite set of symbols, disjoint from N, called the terminals,
  • $S \in N$ is the start symbol and
  • R is a finite set of rules(productions), with each rule of the form $X \to \omega$, where $X \in N$ and $\omega \in (N \cup T)^+$.
Chomsky Normal Form

A context-free grammar $\langle N, T, S, R\rangle$ is in chomsky normal form if each rule in R is of one of the two forms

  • $X \to YZ$, where $X, Y, Z \in N$,
  • $X \to a$, where $X \in N, a \in T$.

In addition, we permit $S \to \epsilon$, where $S$ is the start variable.

Pushdown Automata

A pushdown automaton is a six-tuple $\langle Q, \Sigma, \Gamma, \delta, S, F \rangle$, where

  • Q is a finite set of states,
  • $\Sigma$ is a finite set of input symbols,
  • $\Gamma$ is a finite set of stack symbols,
  • $\delta : Q \times \Sigma_{\epsilon} \times \Gamma_{\epsilon} \to P(Q \times \Gamma)$ is the transition function, where $P(Q \times \Gamma)$ is the power set of $Q \times \Gamma$,
  • $S \in Q$ is the start state and
  • $F \subseteq Q$ is a finite set of accept states.
Equivalence between Context-Free Grammar and Pushdown Automata

  1. For every context-free grammar G, there is an automaton M, such that M recognizes the language generated by G.

  2. If a pushdown automaton recognizes some language, then it is context-free.

Proof: For the convenience of proof, we modify a pushdown automaton $M = \langle Q, \Sigma, \Gamma, \delta, S, F \rangle$ to give it three features:

  • M accepts by entering empty stack.
  • These is only one accept state $q_{accpet}$ in F.
  • Each transition either pushes a symbol onto the stack or pops one off the stack, but it does not do both at the same time. The way to construct a context-free grammar G from the push-down automaton M is given by follow (Introduction To The Theory Of Computation 2nd Edition by Michael Sipser, p120):

    (1) $\forall p \in Q$, we put the rule $N_{pp} \to \epsilon$ in G.

(2) $\forall p,q,r \in Q$, we add to G a new rule $N_{pq} \to N_{pr} N_{rq}$,
(3) $\forall p, q, r, s \in Q$, if $\exists a, b\in \Sigma_\epsilon$, $t \in \Gamma$, s.t. $(r,t) \in \delta(p,a,\epsilon)$ and $ (q,\epsilon) \in \delta(s,b,t)$, then we put the rule $N_{pq} \to a N_{rs}b$ in G.

It is easy to see that (1) and (2) may give rise to many meaningless grammar rules, i.e., grammar rules that will never be used to derive terminal strings. (3) seems to be the only exception at the first glance. However, it indeed does the same. Consider the following example,

Heading down this way, we can get the grammar for pushdown automaton M1. But following (3), we also get productions like:

$N_{dn} \to N_{em}$
$N_{ek} \to N_{fj}$

These rules are useless. They can not derive any string of terminals. To understand the real cause of the problem, take $N_{dn} \to N_{em}$ for example. When the push down automaton goes from state $q_d$ to state $q_e$, it pushes a $Y$ into the stack. And a $Y$ is poped during the transition from state $q_m$ to $q_n$. But the $Y$ poped is not the same $Y$ pushed during the transition from state $q_d$ to $q_e$. At state $q_n$, the stack does not recover to its initial state before the automaton entering state $q_d$. Some additional non-terminal symbols are still in the stack.

When entering state $q_j$, the stack may not be what it is before going to state $q_d$. But the rule $N_{dj} \to N_{ec}$ is valid. It can indeed derive terminal strings.

Non-context-free Languages

Pumping lemma for context-free language: for every context-free language $L$, there is a number p, such that for every $\omega \in L$, if the length of $\omega$ is greater than p, then $\omega$ can be devided into five pieces $\omega = axyzb$, satisfying the following three conditions:

  1. $ax^iyz^ib \in L$, for all $i \ge 0$,
  2. $|xyz| \le p$ and
  3. $|x| \ge 0$ or $|z| \ge 0$.

Chapter One
Finite State Automata(DFA)

A finite state automaton is a five-tuple $\langle Q, \Sigma , \delta, S, F \rangle $, where

  • $Q$ is a finite set called the states,
  • $\Sigma$ is a finite set called the alphabet,
  • $\delta : Q \times \Sigma \to Q$ is a transition function: ,
  • $S \in Q$ is the start state,
  • $F \subseteq Q$ is the set of accpet states.
Nondeterministic Finite State Automata(NFA)

A nondeterministic finite state automaton is a five-tuple $\langle Q, \Sigma , \delta, S, F \rangle $, where

  • $Q$ is a finite set called the states,
  • $\Sigma$ is a finite set called the alphabet,
  • $\delta : Q \times \Sigma_{\epsilon} \to P(Q)$ is a transition function, where $P(Q)$ is the power set of $Q$,
  • $S \in Q$ is the start state,
  • $F \subseteq Q$ is the set of accpet states.
Regular Language

A regular language is the language that can be recognized by some finite state automaton.

Regular Expression

The set of regular expressions of an alphabet $\Sigma$ is defined as follows:

  • ${a} \in RE$ for some $a \in \Sigma$,
  • ${\epsilon} \in RE$,
  • $R \in RE$ , if is
    1. $(R_1 \cup R_2)$, where $R_1, R_2 \in RE$,
    2. $(R_1 \circ R_2)$, where $R_1, R_2 \in RE$,
    3. $(R_1^*)$, where $R_1 \in RE$.
Theorem 1

Every nondeterministic finite automaton has an equivalent deterministic finite automaton.

Theorem 2

Every regular language has a regular expression that describes it.

Theorem 3

For every regular language $L$, there is a number p where, for every $\omega \in L$ of length at least p, $\omega$ can be devided into three pieces $\omega = xyz$, such that

  • $xy^iz \in L$, for every $i \ge 0$,
  • $|xy| \le p$, and
  • $|y| > 0$. ​ ​ ​ ​
