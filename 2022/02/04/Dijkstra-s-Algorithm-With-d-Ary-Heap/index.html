Dijkstra's Algorithm With d-Ary Heap

Posted on 2022-02-04

A \(d\)-ary min heap with \(n\) elements has the following time complexity:

  • Insertion. \(O\big((\log n) / (\log d) \big)\).
  • Extract-Min. \(O\big(d \cdot (\log n) / (\log d) \big)\).
  • Decrease-Key. \(O\big((\log n) / (\log d) \big)\).

Recall that for a single source shortest path query on a graph with \(n\) vertices and \(m\) edges, the Dijkstra's Algorithm performs \(n\) extract-min operations, and \(m\) decrease-key operation. Working with \(d\)-ary min heap, it has time complexity \[ O \left( n \cdot d \cdot \frac{\log n}{\log d} + m \cdot \frac{\log n}{\log d} \right) \]

Setting \(d = m / n\) minimizes this to \[ O \left( m \cdot \log_{m / n} n \right) = O \left( m \cdot \frac{\log n}{\log (m / n)} \right). \]

Consider the following cases:

  • \(m = O(n)\), then the time complexity is \(O(n \log n)\).

  • \(m = \Theta (n \log n)\), then the time complexity is \[ O \left( n \cdot (\log n) \cdot \frac{\log n}{\log \log n } \right). \]
    Example:

    1. when \(n = 2^{20} = 1\)M, \((\log n) / (\log \log n) = 20 / (\log 20) \approx 4.6\).
    2. when \(n = 2^{30} = 1\)G, \((\log n) / (\log \log n) = 30 / (\log 20) \approx 6.1\).

  • \(m = \Theta \big( n^{1 + \epsilon} \big)\) for some \(\epsilon > 0\), then the time complexity is \[ O \left( \frac{m}{\epsilon} \right). \]
