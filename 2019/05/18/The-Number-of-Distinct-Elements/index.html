The Number of Distinct Elements

Posted on 2019-05-18 Edited on 2022-02-14

Problem

In many applications, data comes as a stream S={x1,x2,...,xn}S = \{x_1, x_2, ..., x_n \}S={x1​,x2​,...,xn​}, where each xi,i∈[n]x_i, i \in [n]xi​,i∈[n] belongs to the domain U\mathcal{U}U, with size ∣U∣=N|\mathcal{U}| = N∣U∣=N.
With out lost of generality, suppose each element in U\mathcal{U}U appear at least once in SSS.

Problem. Estimate NNN, i.e., the number of distinct elements in SSS.

Naive solutions include

  1. Sorting. With quicksort, this can be preformed in O(nlog⁡n)O(n \log n)O(nlogn) expected time and O(n)O(n)O(n) space.

  2. Hashing. With hashing (e.g., Cuckoo hashing), this can be done in O(n)O(n)O(n) time and O(n)O(n)O(n) space with two level hashing.

In real world application, nnn could be extremely large.
Therefore, the demand for O(n)O(n)O(n) space is expansive or even impossible.

Question. Can we perform this with O~(1)\tilde O(1)O~(1) space?

If we allow some error, this is possible.

Flajolet and Martin Algorithm

Let H\mathcal{H}H be the collection of all functions from U\mathcal{U}U to [0,1][0, 1][0,1].
Suppose that we pick a function hhh uniformly at randomly from H\mathcal{H}H, then it maps elements in U\mathcal{U}U independently and uniformly at random to real numbers in [0,1][0, 1][0,1].

Question: can we find another family of U\mathcal{U}U so that the following analysis still hold?

The algorithm works as follows.

Flajolet and Martin Algorithm.

  1. Denote h(S)={h(x1),h(x2),...,h(xn)}h(S) = \{ h(x_1), h(x_2), ..., h(x_n)\}h(S)={h(x1​),h(x2​),...,h(xn​)};
