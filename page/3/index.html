van Emde Boas Tree

Posted on 2022-03-18 Edited on 2022-03-26

Problem

Let \(d \in \mathbb{N}^+\) such that \(\log \log d \in \mathbb{N}^+\). We want to design a data structure \(V\), that maintains \(n\) elements from the finite domain \(\mathbb{Z}_d \doteq \{0, \ldots, d - 1 \}\), with the following properties:

It supports insert, successor operations in \(O(\log \log d)\) time.

Base Case

If \(d = 2\), then the tree consists of a bit array \(V\) of size two. Insertion and deletion involves setting the corresponding bit to \(1\) and \(0\), respectively. Finding successor of an element involving scanning the array. All operations has time complexity \(O(1)\).

General Cases

Naive Array

We could maintain a bit array \(V\) of size \(d\). Insertion and deletion can still be performed in \(O(1)\) time. However, successor has time complexity \(O(d)\).

Naive Recursion

We can partition the bit array \(V\) into \(\sqrt{d}\) clusters, denoted \(V\).cluster\([0]\), \(\ldots\), \(V\).cluster\([\sqrt{d} - 1]\).
Each cluster consists of \(\sqrt{d}\) consecutive elements, and maintains a summary bit for deciding whether it is empty. The summary bits are grouped into an array \(V\).summary.

Now insertion time is still \(O(1)\). Deletion and successor can be performed in \(O(\sqrt{d})\) time.

van Emde Boas Tree

van Emde Boas Tree (vEB Tree) is an advanced data structure based on the divide-and-conquer strategy.

Definition

Denote the structure over domain \(d\) as \(\text{vEB}(d)\). It is defined recursively as follows:

Definition \(\text{vEB}(d)\):
1. If \(d = 2\), then \(\text{vEB}(d)\) is just a simple bit array of size \(2\).
2. Otherwise, \(\text{vEB}(d)\) has the following components:
* Two elements, max and min, each of size \(\log d\) bits.
* Children trees, \(c_0, \ldots, c_{\sqrt{d} - 1}\), each of type \(\text{vEB} \left(\sqrt{d}\right)\).
* Summary tree, \(s\), of type \(\text{vEB} \left(\sqrt{d}\right)\).

Hence, if \(d > 2\), then \(\text{vEB}(d)\) contains \(\big(1 + \sqrt d\big)\,\) \(\text{vEB} \left(\sqrt{d}\right)\) subtrees.

Let \(V\) be an instance of \(\text{vEB}(d)\). If an element \(x \in \mathbb{Z}_{d}\) is stored in \(V\), we write \(x \in V\). The components of \(V\) have the following functionalities. > * \(V\).max and \(V\).min maintain the maximum and minimum elements in the tree. These two elements will not be stored in the children trees or the summary tree.
> * For each \(i \in \mathbb{Z}_{\sqrt{d}}\), the child tree \(V.c_i\) stores elements \(x \in V\) for which 1) \(x \notin \{ V.\max, V.\min \}\); 2) \(\lfloor x / \sqrt{d} \rfloor = i\).
> * The summary tree \(V.s\) stores the \(i \in \mathbb{Z}_{\sqrt d}\) for which \(V.c_i\) is not empty.

Operations

We need the following subroutine.

Procedure exchange\((V, x)\):
If \(x < V.\min\) then
\(\qquad\) swap\((V.\min , x)\);
Elif \(V.\max < x\) then
\(\qquad\) swap\((V.\max , x)\);

Next we consider inserting an element \(x \in \mathbb{Z}_d\) into and \(\text{vEB}(d)\) structure \(V\).

If \(V\) is empty, then the algorithm sets \(V.\max\) and \(V.\min\) to \(x\).

If \(V\) consists of one element (in this case, \(V.\max = V.\min\)), then \(x\) is stored in either \(V.\max\) or \(V.\min\).

Otherwise, \(x\) is swapped with \(V.\max\), if \(x > V.\max\), or \(x\) is swapped with \(V.\min\), if \(x < V.\min\). Then the algorithm extracts the first and the last \((1 / 2) \cdot \log d\) bits, denoted as \(x_h\) and \(x_\ell\) respectively, from the binary representation of \(x\). The value of \(x_\ell\) will be inserted into the child tree \(V.c_{x_h}\). If \(V.c_{x_h}\) is empty, the algorithm needs to insert \(x_h\) into the summary tree \(V.s\), indicating that \(V.c_{x_h}\) is not empty after inserting \(x_\ell\) into it.

Procedure insert\((V, x)\):
If \(V\) is empty then
\(\qquad\) \(V.\min \leftarrow x\); \(\,V.\max \leftarrow x\);
If \(V\) contains only one element then
\(\qquad\) exchange\((V, x)\);
else
\(\qquad\) exchange\((V, x)\);
\(\qquad\) \(\big< x_h, x_\ell \big> \leftarrow \left( \lfloor x / \sqrt{d} \rfloor, x \mod{\sqrt{d}} \right)\)
\(\qquad\) If \(V.c_{x_h}\) is empty then
\(\qquad\) \(\qquad\) insert\((V.s, x_h)\);
\(\qquad\) insert\((V.c_{x_h}, x)\);

To find the successor of \(x\) from \(V\), the algorithm first checks whether \(x \ge V.\max\). If so, it returns null.

Otherwise, it inspects the child tree \(V.c_{x_h}\). If it is non-empty, and \(x\) is smaller than the largest element in \(V.c_{x_h}\), the algorithm returns the successor of \(x_\ell\) in \(V.c_{x_h}\).

If not, the algorithm try to find the smallest \(x_h' > x_h\) such that the child tree \(V.c_{x_h'}\) is non-empty subtree. If such \(x_h'\) exists, the algorithm returns the smallest element from \(V.c_{x_h'}\). Otherwise, \(V.\max\) is returned.

Procedure successor\((V, x)\):
If \(x \ge V.\max\) then
\(\qquad\) return null;
\(\big< x_h, x_\ell \big> \leftarrow \left( \lfloor x / \sqrt{d} \rfloor, x \mod{\sqrt{d}} \right)\);
If \(V.c_{x_h}\) is empty or \(V.c_{x_h}.\max \le x\) then
\(\qquad\) \(x_h' \leftarrow\) successor\((V.s, x)\);
\(\qquad\) If \(x_h' \neq\) null then
\(\qquad\) \(\qquad\) return \(V.c_{x_h'}.\min\);
\(\qquad\) Else
\(\qquad\) \(\qquad\) return \(V.\max\)
else
\(\qquad\) return successor\((V.c_{x_h}, x_\ell)\).

Time Analysis

Theorem. Inserting an element \(x\) into an \(\text{vEB}(d)\) instance \(V\) has time complexity \(O(\log \log d)\).

Proof. The claim holds trivially if \(V\) contains zero or one element. Otherwise, let \(T(d)\) be the time complexity for inserting \(x\) into \(V\). We show that \[ T(d) \le T(\sqrt{d}) + O(1). \] Solving the recursion gives \(T(d) \in O(\log \log d)\).

If \(V.c_h\) is non-empty, the only one recursion, i.e., insert\((V.c_{x_h}, x)\), is executed. Therefore, \(T(d) \le T(\sqrt{d}) + O(1)\) holds.

If \(V.c_h\) is empty, two recursions, i.e., insert\((V.s, x)\), insert\((V.c_{x_h}, x)\), are executed. However, in this case, the second recursion has time complexity \(O(1)\). Hence, \(T(d) \le T(\sqrt{d}) + O(1)\) still holds.

\(\square\)

Theorem. Finding the successor of an element \(x\) from an \(\text{vEB}(d)\) instance \(V\) has time complexity \(O(\log \log d)\).

Proof. The successor operation of \(V\) invokes at most one recursive call, therefore its running time \(T(d)\) satisfies \(T(d) \le T(\sqrt{d}) + O(1)\), which solves to \(T(d) \in O(\log \log d)\).

\(\square\)

Space Analysis

Theorem. \(\text{vEB}(d)\) can be implemented with \(O(d)\) space.

Proof. Denote the space usage of \(\text{vEB}(d)\) as \(S(d)\). Then we have \(S(2) = O(1)\). For \(d > 2\), a \(\text{vEB}(d)\) structure stores

  1. Elements \(\min\) and \(\max\), which takes \(\Theta(1)\) words.
  2. Pointers to \(1 + \sqrt{d}\) subtrees, which takes \(\Theta(\sqrt{d})\) words.
  3. \(1 + \sqrt{d}\) subtrees, each taking space \(S(\sqrt{d})\).

Therefore, the total space usage is given by \[ S(d) = (1 + \sqrt{d}) \cdot S(\sqrt{d}) + \Theta (\sqrt{d}). \]

To solve the recursion, assume that \(S(4) = 1\) (by proper scaling). Further, assuming that \(S(d) \le d - 2\). Then, we can verity that the recursion holds: \[ S(d) \le (1 + \sqrt{d}) \cdot (2 \sqrt{d} - 2) + \sqrt{d} = d - 2. \]

\(\square\)

Theorem. \(\text{vEB}(d)\) can be implemented with \(O(n \log \log d)\) space.

Proof. One improvement to the previous approach is to avoid maintaining all \(\sqrt{d}\) pointers to the children trees \(c_0, \ldots, c_{\sqrt{d}}\), and constructing all of them explicitly.

Instead, we construct a child tree only if it is non-empty, and keep a hash table in the \(\text{vEB}(d)\) structure, which stores the pointers to these non-empty children trees.

Inserting an element has time complexity \(O(\log \log d)\), therefore it can creates at most \(O(\log \log d)\) pointers, as well as subtrees. Since when a subtree is created, it takes space \(O(1)\), therefore the total space usage is \(O(n \log \log d)\).

This bound is tight. Suppose we want to insert an element \(x\) into a \(\text{vEB}(d)\) structure. If \(c_{x_h}\) is empty, then we invoke the recursion insert\((s, x_h)\) to insert \(x_h\) into the summary. However, in this case, we need to crete the subtree for \(c_{x_h}\) (which takes \(O(1)\) space), insert the pointer for \(c_{x_h}\) into the hash table (which also takes \(O(1)\) space). This can happen for each recursion, therefore the total space overhead is \(O(\log \log d)\) for a single insertion.

\(\square\)

Reference

[1] T. H. Cormen, Ed., Introduction to algorithms 3rd, 3rd ed. Cambridge, Mass: MIT Press, 2009.

Report Noisy Max

Posted on 2022-03-06

Let \(d \in \mathbb{N}^+\) be a positive integer, and \(u \in \mathbb{N}^d\) be a vector.

Definition. Two vectors \(u, v \subset \mathbb{N}^d\) are neighboring, denoted as \(u \sim v\), if \(\Vert u - v \Vert_\infty = 1\).

We are interested in the following research problem.

Problem. Design an \(\epsilon\)-differentially private algorithm to report \[ \arg \max_{i \in [d] } u_i. \]

Laplacian Mechanism

Since \(\Vert u - v \Vert_\infty = 1\), it is possible that \(\Vert u - v \Vert_1 = d\). A naive approach would be adding independent Laplacian noise \(X_i \sim \mathcal{Lap}( \epsilon / d)\) to each coordinate of \(u\), and then return \[ \mathcal{M}(u) \doteq \arg \max_{i \in [d] } \big( u_i + X_i \big). \]

Theorem (Privacy Guarantee). The algorithm is \(\epsilon\)-differentially private.

Proof.

This follows directly from Laplacian mechanism.

\(\square\)

Theorem (Utility Guarantee). With probability at least \(1 - \beta\), it holds that \[ u_{ \mathcal{M}(u) } \ge \left( \max_{i \in [d] } u_i \right) - \frac{d}{\epsilon} \ln \frac{d}{\beta}. \] Proof. For each \(i \in [d]\), \[ \Pr \left[ |X_i| \ge \frac{d}{\epsilon} \ln \frac{d}{\beta} \right] \le \frac{\beta}{d}. \] Via union bound, \[ \Pr \left[ \exists i \in [d] : |X_i| \ge \frac{d}{\epsilon} \ln \frac{d}{\beta} \right] \le \beta. \] \(\square\)

Report Noisy Max

It is possible to add Laplacian noise \(X_i \sim \mathcal{Lap}(\epsilon / 2)\) to each coordinate of \(u\), and then return \[ \mathcal{M}(u) \doteq \arg \max_{i \in [d] } \big( u_i + X_i \big). \]

Theorem (Privacy Guarantee). The algorithm is \(\epsilon\)-differentially private.

Proof. Fix a \(u \in \mathbb{N}^d\). We prove that for each \(i \in [d]\), and each \(v \sim u\), it holds that \[ e^{-\epsilon} \cdot \Pr[ \mathcal{M}(v) = i] \le \Pr[ \mathcal{M}(u) = i ] \le e^\epsilon \cdot \Pr[ \mathcal{M}(v) = i]. \] By symmetry, we prove the claim for just \(i = n\).

Let \(X_j \sim \mathcal{Lap}(\epsilon / 2), j \in [n]\), and \(Z_u = \max_{j \in [n - 1] } \big( u_j + X_j \big)\). The event \(\mathcal{M}(u) = n\) happens if and only if
\[ Z_u < u_n + X_n. \]

Denote \(\vec X_{n - 1} \doteq (X_1, \ldots, X_{n - 1})\). Fix a sequence \(\vec s = (s_1, \ldots, s_{n - 1} ) \in \mathbb{R}\). Conditioned on \(\vec X_{n - 1} = \vec s\), the probability density of \(\Pr\big[ \mathcal{M}(u) = n \big]\) is given by \[ \begin{aligned} p\big[ \mathcal{M}(u) = n \big] = p\big[ Z_u < u_n + X_n \big] = \int p(x) \cdot \mathbf{1}\big[ Z_u < u_n + x \big] \, dx, \end{aligned} \] where \(p(x)\) are probability densities of \(X_n\).

Similarly, let \(Y_j \sim \mathcal{Lap}(\epsilon / 2), j \in [n]\) and \(Z_v = \max_{j \in [n - 1] } \big( v_j + Y_j \big)\). The conditional probability density of \(\Pr\big[ \mathcal{M}(v) = n \big]\), when \(\vec Y_{n - 1} = \vec s\), is given by \[ \begin{aligned} p\big[ \mathcal{M}(v) = n \big] = p\big[ Z_v < v_n + Y_n \big] = \int p(y) \cdot \mathbf{1}\big[ Z_v < v_n + y \big] \, dy. \end{aligned} \]

Suppose that the following equations holds: \[ \begin{aligned} u_n + x - Z_u &= v_n + y - Z_v. \end{aligned} \]

Then \[ \mathbf{1}\big[ Z_u < u_n + x \big] = \mathbf{1}\big[ Z_v < v_n + y \big]. \]

But this implies that \[ \begin{aligned} |y - x| = \big| u_n - Z_u - \big( v_n - Z_v \big) \big| = \big| u_n - v_n + Z_v - Z_u \big| \le 2. \end{aligned} \]

The inequality follows as conditioned on \(\vec X_{n - 1} = \vec Y_{n - 1} = \vec s\), \[ \begin{aligned} | u_n - v_n | &\le \Vert u - v \Vert_\infty \le 1, \\ | Z_v - Z_u | &= \left| \max_{j \in [n - 1] } \big( v_j + s_j \big) - \max_{j \in [n - 1] } \big( u_j + s_j \big)\right|, \\ &\le \max_{j \in [n - 1] } \left| v_j - u_j \right| \le \Vert u - v \Vert_\infty \le 1. \end{aligned} \]

Therefore, \[ \begin{aligned} \frac{p\big[ \mathcal{M}(u) = n \big] }{p\big[ \mathcal{M}(v) = n \big] } = \frac{ \int p(x) \cdot \mathbf{1}\big[ Z_u < u_n + x \big] \, dx } { \int p(y) \cdot \mathbf{1}\big[ Z_v < v_n + y \big] \, dy } \le \exp \Big( 2 \cdot \frac{\epsilon}{2}\Big) \le \exp(\epsilon). \end{aligned} \]

Integrating over \(\vec X_{n - 1} = \vec s\) (\(\vec Y_{n - 1} = \vec s\)) for all possible \(\vec s\) finishes the proof.

\(\square\)

Theorem (Utility Guarantee). With probability at least \(1 - \beta\), it holds that \[ u_{ \mathcal{M}(u) } \ge \left( \max_{i \in [d] } u_i \right) - \frac{1}{\epsilon} \ln \frac{d}{\beta}. \] Proof. For each \(i \in [d]\), \[ \Pr \left[ |X_i| \ge \frac{1}{\epsilon} \ln \frac{d}{\beta} \right] \le \frac{\beta}{d}. \] Via union bound, \[ \Pr \left[ \exists i \in [d] : |X_i| \ge \frac{1}{\epsilon} \ln \frac{d}{\beta} \right] \le \beta. \] \(\square\)

Exponential Mechanism

The exponential mechanism is defined by its output distribution:
\[ \Pr[\mathcal{M}(u) = i] \propto \exp \left( \epsilon \cdot u_i / 2 \right), \quad \forall i \in [d]. \]

Normalizing these probabilities give \[ \Pr[\mathcal{M}(u) = i] = \frac{ \exp \left( \epsilon \cdot u_i / 2 \right) }{ \sum_{j \in [d] } \exp \left( \epsilon \cdot u_j / 2 \right) }, \quad \forall i \in [d]. \]

Theorem (Privacy Guarantee). The algorithm is \(\epsilon\)-differentially private.

Proof. Fix a \(u \in \mathbb{N}^d\). We prove that for each \(i \in [d]\), and each \(v \sim u\), it holds that \[ e^{-\epsilon} \cdot \Pr[ \mathcal{M}(v) = i] \le \Pr[ \mathcal{M}(u) = i ] \le e^\epsilon \cdot \Pr[ \mathcal{M}(v) = i]. \] By symmetry, we prove the claim for just \(i = n\).

\[ \begin{aligned} \frac{\Pr[ \mathcal{M}(u) = n] }{\Pr[ \mathcal{M}(v) = n] } &= \frac{ \exp \left( \epsilon \cdot u_n / 2 \right) }{ \exp \left( \epsilon \cdot v_n / 2 \right) } \cdot \frac{ \sum_{j \in [d] } \exp \left( \epsilon \cdot v_j / 2 \right) }{ \sum_{j \in [d] } \exp \left( \epsilon \cdot u_j / 2 \right) } \\ &\le \exp( \epsilon / 2 ) \cdot \exp( \epsilon / 2 ) \\ &= \exp( \epsilon ). \end{aligned} \]

The inequality follows since \[ \begin{aligned} u_i - v_i \le \Vert u - v \Vert_\infty \le 1, \implies \exp( \epsilon (u_i - v_i) / 2) \le \exp( \epsilon / 2), \, \forall i \in [d]. \end{aligned} \]

\(\square\)

Theorem (Utility Guarantee). With probability at least \(1 - \beta\), it holds that \[ u_{ \mathcal{M}(u) } \ge \left( \max_{i \in [d] } u_i \right) - \frac{2}{\epsilon} \cdot \ln \frac{|S|}{|S_{OPT}| \cdot \beta}. \] Proof. For each \(t \in \mathbb{R}\), the un-normalized probability mass assigned to the set \(S \doteq \{ i \in [d] : u_i \le t \}\) is at most \(|S| \cdot \exp \left( \epsilon \cdot t / 2\right)\).

On the other hand, the un-normalized probability mass assigned to the set \(S_{OPT} \subseteq [d]\) consisting of optimal solutions is \(|S_{OPT}| \cdot \exp \left( \epsilon \cdot \left( \max_{i \in [d] } u_i \right) / 2\right)\).

\[ \begin{aligned} \Pr \left[ u_{ \mathcal{M}(u) } \le t \right] &\le \frac{ |S| \cdot \exp \left( \epsilon \cdot t / 2\right) } { |S_{OPT}| \cdot \exp \left( \epsilon \cdot \left( \max_{i \in [d] } u_i \right) / 2 \right) } \\ &\le \frac{|S|}{|S_{OPT}|} \exp \left( \epsilon \left( t - \max_{i \in [d] } u_i \right) / 2 \right). \end{aligned} \]

To bound the last term by \(\beta\), we can set \[ t = \left( \max_{i \in [d] } u_i \right) - \frac{2}{\epsilon} \cdot \ln \frac{|S|}{|S_{OPT}| \cdot \beta}. \]

\(\square\)

Approximate Densest Subgraph

Posted on 2022-02-11 Edited on 2022-02-12

Let \(G = (V, E)\) be an undirected graph with \(|V| = n\) vertices and \(|E| = m\) edges.
For each \(S \subset V\), denote \(E(S)\) the set of edges in the subgraph induced by \(S\), i.e., \[ E(S) \doteq \{ (u, v) \in E : u, v \in S \}. \] The density of the subgraph induced by \(S\) is \[ d_G(S) \doteq \frac{|E(S)|}{|S|}. \]

Problem. Find \(S^* \subset V\), s.t. \(d_G(S^*)\) is maximized.

Greedy Algorithm

Definition. Given \(S \subset V\), for each \(u \in S\), define the set of edges incident to \(u\) in \(S\) as \[ E(u, S) \doteq \{ (u, v) \in E(S) \}. \]

Definition. A function \(w: E \rightarrow V\) is called an edge assignment, if it maps each edge \((u, v) \in E\) to one of its endpoints \(\{u, v \}\).

There is a linear time, 2-approximation greedy algorithm [1] which works as follows.

Greedy Algorithm. 1. \(S_1 \leftarrow V\);
2. for \(i \in [n]\) do
3. \(\qquad\) \(u_i \leftarrow \arg\min_{u \in S_i} |E(u, S_i)|\);
4. \(\qquad\) \(S_{i + 1} \leftarrow S_i \setminus \{ u_i \}\);
5. \(\qquad\) for \(e = (u_i, v) \in E(u_i, S_i)\) do
6. \(\qquad\) \(\qquad\) \(w(e) \leftarrow u_i\);
7. \(k \leftarrow \arg\max_{i \in n]} |E(u_i, S_i)|\);
8. return \(S_k\);

Approximation Guarantee

Definition. Given an edge assignment \(w\), for each \(u \in V\), denote the set of edges assigned to \(u\) as \[ A_w(u) \doteq \{ e = (u, v) \in E : w(e) = u \}. \]

The approximation guarantee relies on the following lemma.

Lemma. For edge assignment \(w\) and each \(S \subset V\), it holds that \[ \max_{u \in V} |A_w(u)| \ge d_G(S). \]

Proof. Observe that all edges in \(E(S)\) are assigned to vertices in \(S\). It follows that \[ \max_{u \in V} |A_w(u)| \ge \max_{u \in S} |A_w(u)| \ge \frac{|E(S)|}{|S|} = d_G(S). \] \(\square\)

Theorem. The solution \(S_k\) satisfies \[ d_G(S_k) \ge \frac{1}{2} \cdot d_G (S^*). \]

Proof.
Let \(w\) be the assignment constructed by the greedy algorithm, and denote \[ d_{\max} \doteq \max_{u \in V} |A_w(u)| = \max_{i \in [n]} |E(u_i, S_i)| = |E(u_k, S_k)|. \] Via the previous lemma, it holds that \[ d_G(S^*) \le d_{\max} = |E(u_k, S_k)|. \]

Further, as \(u_k\) is the vertices with minimum degree in the subgraph induced by \(S_k\), \(|E(u_k, S_k)|\) should be upper bounded by the average degree in \(S_k\): \[ d_G(S^*) \le |E(u_k, S_k)| \le 2 \cdot \frac{|E(S_k)|}{ |S_k| } = 2 \cdot d_G(S_k). \]

\(\square\)

Running Time

Theorem. The algorithm can be implemented in \(O(m + n)\) time.

Proof.

We can maintain the degrees of the vertices in \(S_i, i \in [n]\) dynamically.

  1. At the beginning of the algorithm, set \(d(u) = |E(u, V)| = |E(u, S_i)|\) for each \(u \in V\). We maintain the invariant that \(d(u) = |E(u, S_i)|\) at the beginning of each iteration \(i \in [n]\).

  2. At the \(i^{(th)}\) iteration, when \(u_i\) is removed from \(S_i\) to construct \(S_{i + 1}\), we decrease \(d(v)\) by \(1\), if \((u_i, v) \in E(S_i)\).

It is easy to see that the invariant is maintained, and the total update time is \(O(m)\).

Next, we design a data structure that supports the extraction of \(u_i\), the vertex with minimum degree in \(S_i\), in \(O(1)\) time. We keep an array with size \(n\).

  • Each vertex \(u\) is kept in the \(d(u)\)-th bucket of the array. When \(d(u)\) is updated, we update the bucket of \(u\) correspondingly. The total update cost is \(O(m)\).

  • At the beginning of the algorithm, we search bucket \(0, 1, \ldots\) , until we find the first non-empty bucket \(j \in [n]\). Then we extract a vertex \(u\) from the bucket \(j\), and updates the \(d(v)\) for which \(v\) is neighbor of \(u\) in the induced subgraph. The update decreases \(d(v)\) by \(1\). Therefore, the bucket \(\max \{ 0, j - 1 \}\) can become non-empty at the next iteration, from which we should try to find the next vertex with minimum degree. There are \(n\) iteration, and we can move backward at the array by at most \(n\) steps. It follows that the total searching cost in the array is \(O(n)\).

\(\square\)

Reference

[1] M. Charikar, “Greedy Approximation Algorithms for Finding Dense Components in a Graph,” 2000.
