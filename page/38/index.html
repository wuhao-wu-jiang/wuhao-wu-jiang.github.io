Given a set S⊂US \subset US⊂U of nnn points, a metric ddd defined on UUU, and a query point vvv, the nearest neighbor search (NNS) problem finds vvv’s nearest point in SSS:

u=arg⁡min⁡s∈Sd(v,s)u = \arg\min_{s \in S} d(v, s) u=args∈Smin​d(v,s)

Potential applications include web search and clustering.

If UUU is a Euclidean space Rd\mathcal{R}^dRd for some ddd, the hardness of NNS depends largely on the value of ddd. Suppose d=1d = 1d=1, then we can pre-process the points by building a balanced binary search tree on SSS. Finding the nearest point to v∉Sv \notin Sv∈/S reduces to searching for its predecessor and successor, which takes O(log⁡n)O(\log n)O(logn) time. For d=2d = 2d=2, we can construct a Voronoi diagram.

For higher dimensions ddd it is not easy to extend the index structures. A naïve search loops over all points and takes O(nd)O(nd)O(nd) time. It is natural to ask whether we can do better. If we are willing to sacrifice accuracy for efficiency, this is possible. Instead of finding the exact nearest neighbor, we relax our goal to finding an approximate nearest neighbor whose distance is within 1+ϵ1 + \epsilon1+ϵ of the minimum one.

One possible solution is Johnson–Lindenstrauss decomposition. We can generate a matrix A∈RO(log⁡nϵ2)×dA \in R^{ O( \frac{ \log n }{ \epsilon^2 } )\times d}A∈RO(ϵ2logn​)×d and multiply it with every element in SSS. The mapping u→Auu \rightarrow Auu→Au preserves pair-wise distance up to a relative error of ϵ\epsilonϵ. The pre-processing takes O(ndlog⁡nϵ2)O( \frac{n d \log n}{\epsilon^2} )O(ϵ2ndlogn​) time. To answer an query, we map vvv to AvAvAv, which takes O(dlog⁡nϵ2)O(\frac{ d\log n }{ \epsilon^2 } )O(ϵ2dlogn​) time. Then we perform a loop over all elements of {Au:u∈S}\{ Au : u \in S\}{Au:u∈S}, which takes O(nlog⁡nϵ2)O(\frac{ n \log n }{ \epsilon^2 } )O(ϵ2nlogn​) time. Therefore, the total time complexity is O((n+d)log⁡nϵ2)O(\frac{ (n + d) \log n }{ \epsilon^2 } )O(ϵ2(n+d)logn​).

Hashing

[1] solves the problem by hashing. We call the pair of points that are close to each other the similar pair and the pair that are far from each other the dissimilar pair. Intuitively, we want similar pairs to be hashed to the same bucket and that dissimilar pairs to be hashed to different buckets.

Instead of finding the (1+ϵ)(1 + \epsilon)(1+ϵ) approximate nearest neighbor directly, [1] further reduces it to the (c,r)(c, r)(c,r)-approximate neighbor search (ANNS(c,r)(c, r)(c,r)) problem, which is defined as follows:

Given a query point vvv, if ∃s∈S\exists s \in S∃s∈S, then return a point uuu, such that

d(v,u)≤c⋅rd(v, u) \le c \cdot r d(v,u)≤c⋅r

The reduction from 1+ϵ1 + \epsilon1+ϵ to ANNS(c,rc, rc,r) is not easy. Here we show one that returns the (1+ϵ)2(1 + \epsilon)^2(1+ϵ)2 approximate nearest neighbor. Let dmax⁡=max⁡s,s′d(s,s′)d_{\max} = \max_{s, s'} d(s, s')dmax​=maxs,s′​d(s,s′) and dmin⁡=min⁡s,s′d(s,s′)d_{\min} = \min_{s, s'} d(s, s')dmin​=mins,s′​d(s,s′). We build a set of structures that answers ANNS(1+ϵ1 + \epsilon1+ϵ, r) queries for the values of rrr:

dmin⁡, dmin⁡(1+ϵ), dmin(1+ϵ)2, ..., dmax⁡d_{\min}, \ d_{\min} (1 + \epsilon ), \ d_{min} (1 + \epsilon)^2,\ ...,\ d_{\max} dmin​, dmin​(1+ϵ), dmin​(1+ϵ)2, ..., dmax​

The Problem Reduction Chain

(r,cr,p1,p2)(r, cr, p_1, p_2)(r,cr,p1​,p2​)-Locality Sensitive Hashing Family

Now we show how to solve the ANNS(c,r)(c, r)(c,r) problem with hashing. Designing a perfecting hashing that solves the problem directly is not easy. It is desirable to have some hashing which successes with some probability and then boost its probability systematically. To achieve this, we introduce the idea of (r,cr,p1,p2)(r, cr, p_1, p_2)(r,cr,p1​,p2​)-locality sensitive hashing family.
