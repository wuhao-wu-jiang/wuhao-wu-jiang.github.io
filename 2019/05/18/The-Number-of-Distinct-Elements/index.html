The Number of Distinct Elements

Posted on 2019-05-18 Edited on 2021-03-29

In many network applications, data comes as a stream S={x1,x2,...,xn}S = \{x_1, x_2, ..., x_n \}S={x1​,x2​,...,xn​}, where each xix_ixi​ (1≤i≤n1 \le i \le n1≤i≤n) belongs to the domain DDD. Suppose each element in DDD appear at least once in SSS, we want to evaluate the size of DDD, i.e., the number of distinct elements in SSS (denoted as N=∣D∣N = |D|N=∣D∣).

Potential solutions include

  1. We can sort the elements in SSS and count the number of distinct elements. This requires O(nlog⁡n)O(n \log n)O(nlogn) time (with quicksort) and O(n)O(n)O(n) space.
  2. We can use hashing to count the distinct elements. This can be done in O(n)O(n)O(n) time and O(n)O(n)O(n) space with two level hashing.

In real world application, nnn could be extremely large. Therefore, the demand for O(n)O(n)O(n) space is expansive or even impossible. Can we do this with O(1)O(1)O(1) space? If we allow some error, this is possible.

Flajolet and Martin Algorithm

Suppose that there is an ideal hash family H\mathcal{H}H, such that if we pick a function hhh uniformly at randomly from H\mathcal{H}H, it maps elements in U\mathcal{U}U independently and uniformly at random to real numbers between [0,1][0, 1][0,1].

Question: to what extend is this possible?

  1. Denote h(S)={h(x1),h(x2),...,h(xn)}h(S) = \{ h(x_1), h(x_2), ..., h(x_n)\}h(S)={h(x1​),h(x2​),...,h(xn​)};
