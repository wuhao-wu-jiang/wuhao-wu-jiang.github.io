Consistent Hashing

Posted on 2018-08-27

Suppose that we would like to implement a distributed file system. There are mmm files and nnn servers to store the file. We want to spread the work load to the servers so that each one keeps approximate mn\frac{m}{n}nm​ files. One way to achieve this is by hashing. Let p≫mp \gg mp≫m be a prime number. Then

h(x)=ax+bmod  pmod  nh(x) = ax + b \mod p \mod n h(x)=ax+bmodpmodn

The problem is that what if nnn changes constantly?

Consider mapping the nnn servers uniformly and independently to real numbers in [0,1][0, 1][0,1]. Let’s divide [0,1][0, 1][0,1] into nnn equally sized intervals [0,1/n][0, 1/n][0,1/n], [1/n,2/n][1/n, 2/n][1/n,2/n], …, [(n−1)/n,1][(n - 1)/n, 1][(n−1)/n,1]. Now, for an interval that a particular server belongs, the probability that no server maps to the 2log⁡n2 \log n2logn intervals to the left of this interval is given by

(1−2lognn)n−1≤e−2lognn(n−1)=1n2e2lognn≤en2(1 - \frac{2 log n}{n})^{n - 1} \le e^{-\frac{2 log n}{n} (n - 1)} = \frac{1}{n^2} e^{\frac{2 log n}{n}} \le \frac{e}{n^2} (1−n2logn​)n−1≤e−n2logn​(n−1)=n21​en2logn​≤n2e​

By union bound, this happens to some server is bounded by en\frac{e}{n}ne​.

What is the probability that the interval between some pair of servers is O(1/n2)O(1 / n^2)O(1/n2). If we divide [0,1][0, 1][0,1] into Θ(n2)\Theta(n^2)Θ(n2) equally size intervals, by Birthday’s Paradox, there is high probability that some interval contains two servers.

Suppose that we throw mmm balls into nnn bins uniformly at random, where nnn is a fixed number. What is the probability that some pair of balls are placed in the same bin?
