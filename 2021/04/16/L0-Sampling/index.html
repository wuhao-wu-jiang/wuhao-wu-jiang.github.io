Problem

Let x∈Rnx \in \mathbb{R}^nx∈Rn be a vector initialized with all zeros. There is a stream SSS of updates, each of which is a pair (t,Δ)∈[n]×R(t, \Delta) \in [n] \times \mathbb{R}(t,Δ)∈[n]×R, causing xt←xt+Δx_t \leftarrow x_t + \Deltaxt​←xt​+Δ.

Define the support Supp(x)\mathbb{Supp}(x)Supp(x) to be the set of all non-zero indexes of xxx:

Supp(x)≐{i∈[n]:xi≠0}. \mathbb{Supp}(x) \doteq \{ i \in [n] : x_i \neq 0 \}. Supp(x)≐{i∈[n]:xi​=0}.

ℓ0\ell_0ℓ0​ sampling: output an index uniformly at random from the support Supp(x)\mathbb{Supp}(x)Supp(x).

We assume that at any time, ∀t∈[n],xt≤M\forall t \in [n], x_t \le M∀t∈[n],xt​≤M for some M∈RM \in \mathbb{R}M∈R.

Naive Solutions

We can uses an array to keep xxx explicitly. But this has size O(n⋅log⁡M)O( n \cdot \log M)O(n⋅logM) bits.

Goal: design an algorithm for ℓ0\ell_0ℓ0​ sampling with space less than O(n⋅log⁡M)O( n \cdot \log M)O(n⋅logM) bits.

If there are only insertion updates, i.e., ∀(t,Δ)∈S\forall (t, \Delta) \in S∀(t,Δ)∈S, it holds that Δ>0\Delta > 0Δ>0, then we can use a hash set to keep the non-zero elements and their frequencies, which has size O(∣Supp(x)∣⋅log⁡M)O( |\mathbb{Supp}(x) | \cdot \log M)O(∣Supp(x)∣⋅logM). If xxx is dense, i.e., ∣Supp(x)∣|\mathbb{Supp}(x)|∣Supp(x)∣ is large, this is still infeasible.

Sampling With Known Support Size

We use the technique developed in the Sampling Game section, to avoid using O(∣Supp(x)∣log⁡M)O( | \mathbb{Supp}(x) | \log M)O(∣Supp(x)∣logM) bits. We have seen that ppp controls the tradeoff between the size of BBB, and the successful probability of sampling an element (B≠∅B \neq \emptysetB=∅).

We begin with the assumption that the size of Supp(x)\mathbb{Supp}(x)Supp(x) is known.

Insertion Only

As before, we consider insertion only model first.

Claim. There is an algorithm such that

  1. with probability 1−δ1 - \delta1−δ, it returns an element from Supp(x)\mathbb{Supp}(x)Supp(x) uniformly at random;
  2. with probability δ\deltaδ, it returns nothing.

Further, the space usage is O(ln⁡1δ)O \left( \ln \frac{1}{\delta} \right)O(lnδ1​) with probability at least 1−δ1 - \delta1−δ.

For simplicity, we assume a perfect hash function hhh that maps elements in [n][n][n] to {0,1}\{0, 1\}{0,1} independently with probability ppp (to be determined). For each update (t,Δ)(t, \Delta)(t,Δ) in the stream, we process it only if h(t)=1h(t) = 1h(t)=1. As we are dealing with insertion only model, we can use a hashset to maintain the elements with hash value 1.

Handling Deletion

When deletions are possible, we can’t use a hashset to keep the sampled updates. E.g., suppose we have a sequence (1,6),(1,−6),(2,5),(3,7),(4,9)(1, 6), (1, -6), (2, 5), (3, 7), (4, 9)(1,6),(1,−6),(2,5),(3,7),(4,9), and that h(1)=h(2)=1,h(3)=h(4)=0h(1) = h(2) = 1, h(3) = h(4) = 0h(1)=h(2)=1,h(3)=h(4)=0. We need a hashset of size 2 to keep both the frequencies of element 1 and element 2. Even through the frequency of element 111 is 0 finally, we don’t this before we see the entire string.

In general, hhh maps npnpnp elements in [n][n][n] to 111. If we use a hashset, its size can grow to O(np)O(n p)O(np).

However, there is data structure that can handle deletions while using less than O(np)O(np)O(np) space.

Sparse Recovery. Given an integer ddd at the beginning of the stream, if Supp(x)≤d\mathbb{Supp}(x) \le dSupp(x)≤d when the stream stops, there is a data structure that returns

  1. indices in Supp(x)\mathbb{Supp}(x)Supp(x);
  2. and their values,

using O(d(log⁡n+log⁡M))⊂O(dlog⁡(nM))O(d (\log n + \log M) ) \subset O( d \log (nM) )O(d(logn+logM))⊂O(dlog(nM)) bits. It processes each update (t,Δ)(t, \Delta)(t,Δ) in the stream in O(d)O(d)O(d) time and recovers xxx in O(d2polylog(nM))O(d^2 polylog(nM))O(d2polylog(nM)) time. Its recovery can be wrong if Supp(x)>d\mathbb{Supp}(x) > dSupp(x)>d.

It suffices to use such a data structure for the sampled set BBB with d∈O(ln⁡1δ)d \in O( \ln \frac{1}{\delta} )d∈O(lnδ1​). We have seen that if we sampled each element in Supp(x)\mathbb{Supp}(x)Supp(x) with probability p=1−δ1/kp = 1 - \delta^{1 / k}p=1−δ1/k, then with probability at least 1−δ1 - \delta1−δ, we obtain a sampled set BBB with size at most ddd. Then we return one element from BBB uniformly at random.

Analyzing the probability of sampling an element is more subtle than the previous section. In previous section, each element is under-sampled with probability 1−(1−p)kk\frac{1 - \left(1 - p \right)^k }{ k }k1−(1−p)k​. Here, conditioned on ∣B∣≤d|B|\le d∣B∣≤d, each element is sampled equally likely. However, ∣B∣>d|B| > d∣B∣>d, the recovery of sampled elements fails, some element can be returned more frequently and some can be returned less frequently (the recovery behavior is un-defined). As the failure probability is bounded by δ\deltaδ, each element is returned with probability

[1−(1−p)kk±δ]⊆[1−(1+1k)⋅δ,1+(1−1k)⋅δ]. \left[ \frac{1 - \left(1 - p \right)^k }{ k } \pm \delta \right] \subseteq \left[ 1 - \left( 1 + \frac{ 1 }{ k } \right) \cdot \delta, 1 + \left( 1 - \frac{ 1 }{ k } \right) \cdot \delta \right]. [k1−(1−p)k​±δ]⊆[1−(1+k1​)⋅δ,1+(1−k1​)⋅δ].
