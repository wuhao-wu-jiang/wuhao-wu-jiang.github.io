Priority Sampling

Posted on 2020-01-12 Edited on 2021-02-24

Suppose there are a set of nnn items with nonnegative weights w1,w2,...,wnw_1, w_2, ..., w_nw1​,w2​,...,wn​. We want to maintain a subset of samples S⊂[n]S \subset [n]S⊂[n] in order to answer queries like given arbitrary I⊂[n]I \subset [n]I⊂[n] , what is ∑i∈Iwi\sum_{i \in I} w_i∑i∈I​wi​?

A trivial solution is to set S=[n]S = [n]S=[n] and we can answer the query exactly. This is an overkill if our goal is to only answer the query approximately.

Alternatively, we can sample each item i∈[n]i \in [n]i∈[n] independently with probability pip_ipi​ (to be determined), and maintain a random variable

Xi={wipii is chosen0 else \begin{aligned} X_i = \begin{cases} \frac{w_i}{p_i} & i \text{ is chosen} \\ 0 & \text{ else } \end{cases} \end{aligned} Xi​={pi​wi​​0​i is chosen else ​​

Note that

E[Xi]=wi\mathbb{E} \left[ X_i \right] = w_i E[Xi​]=wi​

And therefore,

E[∑i∈IXi]=∑i∈IE[Xi]=∑i∈Iwi\mathbb{E} \left[\sum_{i \in I} X_i \right] = \sum_{i \in I} \mathbb{E}[X_i] = \sum_{i \in I} w_i E[i∈I∑​Xi​]=i∈I∑​E[Xi​]=i∈I∑​wi​

Threshold Sampling

One possible implementation of this sampling scheme is called threshold sampling.

First, a threshold TTT is chosen. Then, for each item i∈[n]i \in [n]i∈[n], if wi≥Tw_i \ge Twi​≥T, then the algorithm record a value Xi=wiX_i = w_iXi​=wi​ deterministically. Otherwise, the algorithm record a value Xi=TX_i = TXi​=T with probability wiT\frac{w_i}{T}Twi​​. If XiX_iXi​ is not recorded, it has default value 000. In both cases, E[Xi]=wi\mathbb{E} [ X_i ] = w_iE[Xi​]=wi​.

Mathematically, the two cases can be combined into one. Let uiu_iui​ be a random number generated between (0,1)(0, 1)(0,1). Then

Xi={max⁡{wi,T}ui<wiT0 else \begin{aligned} X_i = \begin{cases} \max \{ w_i, T \} & u_i < \frac{w_i}{T} \\ 0 & \text{ else } \end{cases} \end{aligned} Xi​={max{wi​,T}0​ui​<Twi​​ else ​​

Observe that if wi≥Tw_i \ge Twi​≥T, then wi/T≥1w_i / T \ge 1wi​/T≥1 and Xi≡max⁡{wi,T}=wiX_i \equiv \max \{ w_i, T \} = w_iXi​≡max{wi​,T}=wi​. The expectation of XiX_iXi​ is given algebraically by

E[Xi]=max⁡{wi,T}⋅Pr⁡[ui≤wiT]=max⁡{wi,T}⋅min⁡{1,wi/T}=wi.\begin{aligned} \mathbb{E} [ X_i ] = \max \{ w_i, T \} \cdot \Pr \left[ u_i \le \frac{w_i}{T} \right] = \max \{ w_i, T \} \cdot \min \{ 1, w_i / T \} = w_i. \end{aligned} E[Xi​]=max{wi​,T}⋅Pr[ui​≤Twi​​]=max{wi​,T}⋅min{1,wi​/T}=wi​.​

It follows by linearity of expectation that

E[∑i∈IXi]=∑i∈Iwi. \mathbb{E} \left[ \sum_{i \in I} X_i \right] = \sum_{i \in I} w_i. E[i∈I∑​Xi​]=i∈I∑​wi​.

Space Usage

Denote SSS the set of items with Xi≠0X_i \neq 0Xi​=0 and let pip_ipi​ be the probability of iii being sampled. Then the expected size of SSS is given by

∑i=1npi=∑i=1nmin⁡{1,wiT}\sum_{i = 1}^n p_i = \sum_{i = 1}^n \min \left\{ 1 , \frac{w_i}{T} \right\} i=1∑n​pi​=i=1∑n​min{1,Twi​​}

The problem with threshold sampling is that it is hard to control the memory usage of the sample set SSS. If the nnn items are known offline, and we would like ∣S∣≈k|S| \approx k∣S∣≈k for some integer k≤nk \le nk≤n, then we can search for some TTT such that

∑i=1npi=k\sum_{i = 1}^n p_i = k i=1∑n​pi​=k

Then ∣S∣=k|S| = k∣S∣=k in expectation.

The question is, is it possible to force ∣S∣=k|S|= k∣S∣=k exactly? Further, what if the items come online (one by one)?

Priority Sampling

Priority sampling achieves this. First recall that in threshold sampling we decide whether to keep iii by checking whether a uniform sampled real number uiu_iui​ is less than wiT\frac{w_i}{T}Twi​​:

ui<wiTu_i < \frac{w_i}{T} ui​<Twi​​

This is equivalent to

T<wiuiT < \frac{w_i}{u_i} T<ui​wi​​

Define the priority of an item i∈[n]i \in [n]i∈[n] as qi=wiuiq_i = \frac{w_i}{u_i}qi​=ui​wi​​. Note that qiq_iqi​ is a random variable. In threshold sampling, item iii is kept with weight max⁡{wi,T}\max \{ w_i, T \}max{wi​,T} if

T<qiT < q_i T<qi​

Now, instead of using a fixed value of TTT, the Priority sampling sets

T=(k+1)-th largest {qi}T = (k + 1) \text{-th largest } \{ q_i \} T=(k+1)-th largest {qi​}

By the definition of TTT, only the kkk items with largest priorities qiq_iqi​’s are kept in the sample set SSS:

S={k items with largest qi ’s }S = \{ k \text{ items with largest } q_i \text{ 's } \} S={k items with largest qi​ ’s }

It can be seen that SSS and TTT can be easily maintained by a binary heap when the items come in stream fashion.
