Heavy Hitter

Posted on 2018-08-28 Edited on 2019-06-05

Given a sequence of numbers S={x1,x2,...,xn}S = \{ x_1, x_2, ..., x_n \}S={x1​,x2​,...,xn​} where each element comes from a finite domain UUU, the heavy hitter problem asks for the element that appears at least nk+1+1\frac{n}{k + 1} + 1k+1n​+1 times in the sequence. Note that there could be at most kkk such elements.

Deterministic

If false positive is allowed, we have a deterministic algorithm that has O(k)O(k)O(k) space overhead.

To see the correctness of the algorithm, we give each element in SSS a token. There are nnn tokens in total. If a element is not added to the list LLL, k+1k + 1k+1 token are throw away. If an elements appears more than nk+1\frac{n}{k + 1}k+1n​ times, it must appears in LLL when the algorithm terminates.

Another way to prove this is that, each time an element is thrown away, it is not in the counter and the sum of counter decremented is kkk. Let fxf_xfx​ be the frequency of an element xxx. Then the maximum number of times xxx is thrown away is bounded by kfx≤n−fxk f_x \le n - f_xkfx​≤n−fx​, which implies that fx≤nk+1f_x \le \frac{n}{k + 1}fx​≤k+1n​.

Randomized

An alternate approach is randomized algorithm. In high level, we would like to maintain a set of mmm bins and assign each element in UUU uniformly at random one of the bins. Then we put each number in SSS into the mmm bins according which bin it is assigned to. In expectation, each bin will receive nm\frac{n}{m}mn​ elements. To estimate the number of time an element xxx appears in SSS, we just return the number of elements of the bin xxx is assigned. Note that we might overestimate xxx's frequency, whose expectation is at most nm\frac{n}{m}mn​. We might repeat this several time to reduce variance.
