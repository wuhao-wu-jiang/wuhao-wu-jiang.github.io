Dijkstra's Algorithm With d-Ary Heap

Posted on 2022-02-04

A ddd-ary min heap with nnn elements has the following time complexity:

  • Insertion. O((log⁡n)/(log⁡d))O\big((\log n) / (\log d) \big)O((logn)/(logd)).
  • Extract-Min. O(d⋅(log⁡n)/(log⁡d))O\big(d \cdot (\log n) / (\log d) \big)O(d⋅(logn)/(logd)).
  • Decrease-Key. O((log⁡n)/(log⁡d))O\big((\log n) / (\log d) \big)O((logn)/(logd)).

Recall that for a single source shortest path query on a graph with nnn vertices and mmm edges, the Dijkstra’s Algorithm performs nnn extract-min operations, and mmm decrease-key operation.
Working with ddd-ary min heap, it has time complexity

O(n⋅d⋅log⁡nlog⁡d+m⋅log⁡nlog⁡d) O \left( n \cdot d \cdot \frac{\log n}{\log d} + m \cdot \frac{\log n}{\log d} \right) O(n⋅d⋅logdlogn​+m⋅logdlogn​)

O(m⋅log⁡m/nn)=O(m⋅log⁡nlog⁡(m/n)). O \left( m \cdot \log_{m / n} n \right) = O \left( m \cdot \frac{\log n}{\log (m / n)} \right). O(m⋅logm/n​n)=O(m⋅log(m/n)logn​).

Consider the following cases:

  • m=O(n)m = O(n)m=O(n), then the time complexity is O(nlog⁡n)O(n \log n)O(nlogn).

  • m=Θ(nlog⁡n)m = \Theta (n \log n)m=Θ(nlogn), then the time complexity is

    O(n⋅(log⁡n)⋅log⁡nlog⁡log⁡n). O \left( n \cdot (\log n) \cdot \frac{\log n}{\log \log n } \right). O(n⋅(logn)⋅loglognlogn​).

    2. when n=230=1n = 2^{30} = 1n=230=1G, (log⁡n)/(log⁡log⁡n)=30/(log⁡20)≈6.1(\log n) / (\log \log n) = 30 / (\log 20) \approx 6.1(logn)/(loglogn)=30/(log20)≈6.1.

    Then
