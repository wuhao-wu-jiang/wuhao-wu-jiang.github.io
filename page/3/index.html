Range Min Query

Posted on 2021-12-16 Edited on 2022-01-08

Problem

Definition (Range Min Query). Let $a_1, \ldots, a_n \in \mathbb{R}$.
Given $\ell, r \in [n]$, $\ell \le r$, return $\min \{ a_\ell, \ldots, a_r \}$.

Basic Solutions

A data structure for the Range Min Query (RMQ) has complexity $\left< p(n), q(n) \right>$ if it has pre-processing and query-answering time $p(n)$ and $q(n)$ on input instance of size $n$, respectively.

$\mathcal{A}_1$ Precompute-None

Scan $a_\ell, \ldots, a_r$ to find the min.

  • This is an $\left< 0, O(n) \right>$ solution.

$\mathcal{A}_2$ Complete Binary Tree

Construct a complete binary tree with $a_1, \ldots, a_n$ being the leaf nodes.
Each internal node of the tree is augmented with the minimum value of its subtree.

  • This is an $\left< O(n), O(\log n) \right>$ solution.

$\mathcal{A}_3$ Precompute-All

Create a $n \times n$ table $M$.
For each $\ell, r \in [n]$, $\ell \le r$, the $(\ell, r)$-th entry records $\min \{ a_\ell, \ldots, a_r \}$.

  • This is an $\left< O(n^2), O(1) \right>$ solution.

  • $p(n) = O(n^2)$.

    • Dynamic Programming.
    • $\forall \ell, r \in [n], \ell \le r$,

  • $q(n) = O(1)$.

$\mathcal{A}_4$ Sparse Table

Create a $n \times \lfloor \log n \rfloor$ table $M$.
For each $\ell \in [n], k \in \big[ \lfloor \log n \rfloor \big]$, $r \doteq \ell + \big( 2^k - 1 \big) \le n$, the $(\ell, k)$-th entry records $\min \{ a_\ell, \ldots, a_{\ell + 2^k - 1} \}$.

  • This is an $\left< O(n \log n), O(1) \right>$ solution.

  • $p(n) = O(n \log n)$.

    • Dynamic Programming.
    • For each $\ell \in [n], k \in \big[ \lfloor \log n \rfloor \big]$, $r \doteq \ell + \big( 2^k - 1 \big) \le n$, let

  • $q(n) = O(1)$.

Divide and Conquer

Preprocessing

  1. Pick $s \in \mathbb{N}_+$, and let $m = n / s$.

  2. Partition $a_1, \ldots, a_n$ into $m$ blocks $B_1, \ldots B_m$, such that $B_1$ contains the first $s$ elements, $B_2$ contains the next $s$ elements, and so on. In general,

  3. Denote $b_1, \ldots, b_m$ be the minimum element in $B_1, \ldots B_m$, respectively.

Query

  1. Given $\ell, r \in [n]$, $\ell \le r$, consider

    • $C_1$ be the collection of indexes of the blocks $B_i \subset \{ a_\ell, \ldots, a_r \}$:

      If $C_1 \neq \emptyset$, let $\ell’ = \min_{i \in C_1} i$ and $r’ = \max_{i \in C_1} i$.

    • $C_2$ be the collection of indexes of the blocks $B_i$ such that $B_i \nsubseteq \{ a_\ell, \ldots, a_r \}$ and $B_i \cap \{ a_\ell, \ldots, a_r \} \neq \emptyset$

      It must be that $|C_2| \le 2$.

  2. To return $\min \{ a_\ell, \ldots, a_r \}$, return the minimum of

    • $\min_{i \in C_1} b_i$, if $C_1 \neq \emptyset$.

    • $\min_{i \in C_2} \min \big( B_i \cap \{ a_\ell, \ldots, a_r \}\big)$.

      Note that

      • The first one is a range-min query on $b_1, \ldots, b_m$.
      • The second one consists of range-min queries on the blocks $B_i$ for $i \in C_2$

Approach One

  1. Suppose we invoke an algorithm $A_{up}$ to answer the range-min query on $b_1, \ldots, b_m$, with complexity $\left< p_1(m), q_1(m) \right>$.

  2. Suppose we invoke an algorithm $A_{down}$ to answer the range-min query range-min queries on the blocks $B_i$ for $i \in C_2$, with complexity $\left< p_2(s), q_2(s) \right>$.

  3. Since the set $C_2$ depends on the range min query, to answer arbitrary range min query, we need to run the pre-processing of $A_{down}$ on each block $B_i, i \in [m]$.

Instantiation

$\mathcal{A}_5$ Two-Level Precompute-None

Choose Precompute-None $\mathcal{A}_1$ for both $A_{up}$ and $A_{down}$.

  • Total preprocessing time: $O \big( n \big)$.
  • Total query time: $O \big( m + s \big) = O \big( \sqrt{n} \big)$, if $m = s = \sqrt{n}$.

Hence, it has complexity $\left< O \big( n \big), O \big( \sqrt{n} \big) \right>$.

$\mathcal{A}_6$ Two-Level Binary Tree

This is just a complete binary tree over $a_1, \ldots, a_n$. Therefore,

  • Total preprocessing time: $O \big( n \big)$.

  • Total query time: $O \big( \log n \big)$.

Hence, it has complexity $\left< O \big( n \big), O \big( \log n \big) \right>$.

Approach Two

Can we achieve $\left< O \big( n \big), O \big( 1 \big) \right>$ under approach one?
Recall that approach one has complexity

Suppose we want to achieve preprocessing time $O(n)$ and query time $O(1)$, then $p_2(s)$ and $q_2(s)$ needs to satisfy that $p_2(s) = s$ and $q_2(s) = 1$.
But this is exactly the property we want to achieve!

It seems to be impossible to achieve the desired complexity under the divide and conquer framework.

The key is to observe that the following are two different problems.

  1. Finding a RMQ algorithm with complexity $\left< O(n), O(1) \right>$.
  2. Finding an algorithm that preprocess the all block $B_i$ in $O(n)$ time, such that the range-min query on each $B_i$ can be answered in $O(1)$ time.

For the divide-and-conquer framework to work to achieve complexity $\left< O(n), O(1) \right>$, it suffices to find an algorithm for the second problem.

Theorem. There exists some $s \in \Theta(\log n)$, such that we can find an algorithm that preprocess the all block $B_i$ in $O(n)$ time, such that the range-min query on each $B_i$ can be answered in $O(1)$ time.

Denote such an algorithm as $\mathcal{A}^*$.
Since $s \in \Theta(\log n)$, choosing sparse table algorithm $\mathcal{A}_3$ to answer the range-min query on $b_1, \ldots, b_m$ incurs additional preprocessing time $O( n / s \cdot \log (n / s) ) = O(n)$.
Further, answering a query requires only $O(1)$ time.

It is left to show the construction of $\mathcal{A}^*$.

Block Type

W.L.O.G., consider the first block $B_1 = \{ a_{ 1 }, \ldots, a_{ s} \}$ be a block of $s$ elements.
We can construct a binary tree over the elements, such that it satisfies

  1. Min Heap Property: each node of the tree is the smallest one in the subtree rooted at this node.
  2. The in-order traversal of the tree gives the array $B_1$.

Such a binary tree is called a Cartesian Tree and the type of the block.

On the other hand, suppose we are given a Cartesian Tree tree constructed on $B_1$, and suppose that the value of each node of the tree is hidden.
Hence, we only know the topology or shape of the binary tree.
For all $1 \le \ell \le r \le s$, we can still return the index of the minimum element among $a_\ell, \ldots, a_s$, purely based on the topology of the binary tree.

In this sense, the answers to the RMQ on $B_1$ is entirely determined by its corresponding Cartesian Tree.

Enumeration

This motivates the following strategy.

  1. Enumerate all possible Cartesian Trees.

  2. For each Cartesian Tree, invoke the sparse table algorithm $\mathcal{A}_3$ algorithm on this tree, so that all possible range min query for this tree can be answer in $O(1)$ time.

  3. For each block $B_i, i \in [m]$, determine its Cartesian Trees.
    To answer a RMQ on $B_i$, access the pre-computed result on its corresponding Cartesian Trees.

Determining the Block Type

In this section, we show how to construct the corresponding Cartesian Tree given the block $B_1$, and to create a unique identifier $\in O(n)$ for the Cartesian Tree.

We illustrate the idea with an example.
In this example, $B_1 = (2, 4, 3, 1)$.
We will construct the Cartesian Tree, by including the elements from $B_1$ one by one.
Further, we maintain a stack and a stack operation sequence, both of which are initially empty.
The stack keeps elements from the right spine of the tree, and the stack operation sequence records the “in stack” and “out stack” operations as we include more elements from $B_1$ into the tree.

We first include the element $2$ into the tree.
Since the tree consists of only one node, it is the root of the tree.
It is also the only node in the right spine of the tree, so we put $2$ into the stack, and append $1$ to the stack operation sequence, to record the in-stack operation.

Next we include the element $4$ into the tree.
Since the tree satisfies min-heap property, $4$ must be the child of $2$.
Since the in-order traversal of the tree gives the array $2, 4$, the node $4$ must be the right child of $2$.
We put $4$ into the stack, and append $1$ to the stack operation sequence, to record the in-stack operation.

We want to include the element $3$ into the existing tree.
Since after inclusion, the in-order traversal of the tree gives the array $2, 4, 3$, the node $3$ must be in the right spine of the tree.
Indeed, this is a general principle for adding a new element into the tree.

Lemma (In the Right Spine). For each $j \in [s]$, let $T_j$ be the Cartesian tree constructed on $a_1, \ldots, a_j$.
Then $a_j$ be must in the right spine of $T_j$.

Proof.

  1. If $a_j$ has right-child in $T_j$, then it is not the last element visited by the in-order traversal of $T_j$.
  2. If $a_j$ has a parent node in $T_j$ and is the left-child of its parent, then it is not the last element visited by the in-order traversal of $T_j$.

$\square$

In this case, the right spine consists of $2$ and $4$.
We compare $3$ with $4$.
As it is smaller than $4$, we pop $4$ out of the stack.
We compare $3$ with $2$.
As it is larger than $2$, we put $3$ into the stack.
We include $3$ as the right-child of $2$, and set $4$ as the left child of $3$.
We see that after these operations, the stack still contains the nodes in the right spine of the tree.
Further, the min-heap property of the tree is maintained.
We append $01$ to the stack operation sequence, to record the out-stack and in-stack operations performed.

We now formally states the rules for adding an element to the right spine.

Incremental Construction Rule. For each $1 \le j < s$, let $T_j$ be the Cartesian tree constructed on $a_1, \ldots, a_j$, and suppose that $a_1’ \le a_2’ \ldots \le a_k’$ are the elements in the right spine of $T_j$.
Then $a_{j + 1}$ is added to $T_j$ as follows:

  1. If $a_{j + 1} < a_1’$, then set $a_1’$ as the left child of $a_{j + 1}$.

  2. If $a_{j + 1} > a_k’$, then set $a_{j + 1}$ as the right child of $a_k’$.

  3. Otherwise, there exists $1 \le i < k$, such that $a_i’ \le a_{j + 1} < a_{i + 1}’$. Replace $a_{i + 1}’$ by $a_{j + 1}$ as the right child of $a_i’$, and set $a_{i + 1}’$ as the left child of $a_{j + 1}$.

It is easy to verify that after the construction

  1. $a_{j + 1}$ is in the right spine of the tree.
  2. The min-heap property is maintained.

We want to include the element $1$ into the existing tree.
Since after inclusion, the in-order traversal of the tree gives the array $2, 4, 3, 1$, the node $1$ must be in the right spine of the tree.
In this case, the right spine consists of $2$ and $3$.
As $1$ is smaller than $2$, we pop $3$ and $2$ out of the stack, put $1$ into the stack, and set $2$ as the left child of $1$.
We append $001$ to the stack operation sequence, to record the stack operations performed.

Finally, we pop $1$ out of the stack, and append $0$ to the stack operation sequence, to record the stack operation performed.
We perform this operation, so that whatever sequence of $a_1, \ldots, a_s$, the stack operation sequence corresponding to the Cartesian Tree construction procedure stated as above has length $2s$.

The construction also reveal an interesting fact: each Cartesian Tree with $s$ node corresponding a $0$-$1$ sequence with length $2s$, such that each prefix of the sequence contains no less $1$’s than $0$’s.

Random Variable

Let \((\Omega, \mathcal{F}, \mathbb{P})\) be a probability space, and let \(\mathcal{B}\) be the collection of Borel sets in \(\mathbb{R}\).

Definition. A random variable \(Y: \Omega \rightarrow \mathbb{R}\) on \((\Omega, \mathcal{F}, \mathbb{P})\) is a function that is \(\mathcal{F} / \mathcal{B}\) measurable, i.e., \[ Y^{-1}(E) \doteq \{ \omega \in \Omega : Y(\omega) \in E \} \in \mathcal{F}, \quad \forall E \in \mathcal{B}. \]

It is not easy to verify the property for each \(E \in \mathcal{B}\). It suffices to check one of the following equivalent conditions:

  1. \(Y^{-1}( (-\infty, a] ) \in \mathcal{F}, \forall a \in \mathbb{R}\).
  2. \(Y^{-1}( (-\infty, a) ) \in \mathcal{F}, \forall a \in \mathbb{R}\).
  3. \(Y^{-1}( [a, \infty) ) \in \mathcal{F}, \forall a \in \mathbb{R}\).
  4. \(Y^{-1}( (a, \infty) ) \in \mathcal{F}, \forall a \in \mathbb{R}\).

The correctness follows from the following theorem.

Theorem. If there exists a subset \(\mathcal{S}\) that generates \(\mathcal{B}\), i.e., \(\sigma(\mathcal{S}) = \mathcal{B}\), such that for each \(E \in \mathcal{S}\), \(Y^{-1}(E) \in \mathcal{F}\), then \(Y\) is measurable.

Proof.

  1. If \(Y^{-1}(E) \in \mathcal{F}\), then \(Y^{-1}(\mathbb{R} - E) = \Omega - Y^{-1}(E) \in \mathcal{F}\).

  2. If \(Y^{-1}(E_n) \in \mathcal{F}\) for disjoint \(E_n, n \in \mathbb{N}_+\), then \[ Y^{-1}( \cup_{n \in \mathbb{N}_+} E_n ) = \cup_{n \in \mathbb{N}_+} Y^{-1}( E_n ) \in \mathcal{F}. \]

Let \(\mathcal{L} \doteq \{ E \in \mathcal{B} : Y^{-1}(E) \in \mathcal{F} \}\). Since \(\mathcal{L}\) is closed under complement and countable union, it is a \(\sigma\)-field in \(\mathbb{R}\). If \(\mathcal{L} \supset \mathcal{S}\), then \(\mathcal{L} \supset \sigma(\mathcal{S})\), the theorem follows.

\(\square\)

Distribution

Definition. The distribution \(\mu: \mathcal{B} \rightarrow \mathbb{R}_{\ge 0}\) of \(Y\) is a probability measure on \((\mathbb{R}, \mathcal{B})\) defined by \[ \mu(E) = \mathbb{P}[ Y \in E ] = \mathbb{P} \circ Y^{-1} (E), \quad \forall E \in \mathcal{B}, \] where \(Y \in E\) is a shorthand for \(Y^{-1}(E)\).

We verify that \(\mu\) is a probability measure. Note that \(\mu(\emptyset) = 0\), and \(\mu(\mathbb{R}) = 1\). \(\mu\) is countably additive: consider disjoint \(E_1, E_2, \ldots \in \mathcal{B}\), \[ \begin{aligned} \mu( \cup_{n \ge 1} E_n ) &= \mathbb{P}\big[ Y^{-1} \big( \cup_{n \ge 1} E_n \big) \big] \\ &= \mathbb{P}\big[ \cup_{n \ge 1} Y^{-1} \big( E_n \big) \big] \\ &= \sum_{n \ge 1} \mathbb{P}\big[ Y^{-1} \big( E_n \big) \big]\\ &= \sum_{n \ge 1} \mu( E_n ). \end{aligned} \]

Hence, \((\mathbb{R}, \mathcal{B}, \mu)\) constitutes a probability space.

Properties

Theorem. There can't be an uncountable collection of disjoint sets in \(\mathcal{B}\) with positive \(\mu\)-measure.

Proof. Let \(\mathcal{C} \subset \mathcal{B}\) be a disjoint collection of sets with positive \(\mu\)-measure. Consider \((0, 1]\). For each \(k \in \mathbb{N}_+\), let \[ \mathcal{D}_{1, k} \doteq \{ E \in \mathcal{C}, \mu \big( E \cap (0, 1] \big) \ge 1 / k \}. \] Then \(|\mathcal{D}_{1, k}| \le k\), since \(1 / k \cdot |\mathcal{D}_{1, k}| \le \sum_{E \in \mathcal{D}_{1, k} } \mu(x) \le 1\). Therefore the set \(\mathcal{C}_0 \doteq \cup_{k \ge 1} \mathcal{D}_{1, k}\) is countable.

Similarly we can define \(\mathcal{C}_n\), for each interval \((n, n + 1]\). If \(E \in \mathcal{C}\), then there exists at least one \(n \in \mathbb{Z}\), such that \(E \in \mathcal{C}_n\). So \(\mathcal{C} = \cup_{n \in \mathbb{Z} } \mathcal{C}_n\) is countable.

\(\square\)

Comment: what would be the value of \(1 / k \cdot |\mathcal{D}_{1, k}|\) if \(\mathcal{D}_{1, k}\) is uncountable? One possible way to avoid such multiplication is by applying the axiom of choice. Suppose that the set \(\mathcal{D}_{1, k}\) contains more than \(k\) elements, then by the axiom of choice we can iteratively pick at most \(k\) element (without replacement) from the set. We can not find more item from the set, since the intersection of \((0,1]\) with each element has \(\mu\)-measure at least \(1 / k\).

Expectation

Definition. Let \((\Omega, \mathcal{F}, \mathbb{P})\) be a probability space and \(Y\) is a random variable defined on it. The expectation of \(Y\) is defined by \[ \mathbb{E}[ Y ] \doteq \int Y \, d \mathbb{P} = \int_{ \Omega } Y(\omega) \, \mathbb{P} ( d \omega ). \]

For nonnegative \(Y\), \(\mathbb{E}[ Y ]\) is always defined. For general \(Y\), \(\mathbb{E}[ Y ]\) is defined if at least one of \(\mathbb{E}[ X_+ ]\) or \(\mathbb{E}[ X_- ]\) is finite. \(Y\) is integrable if \(\mathbb{E}[ |Y| ] < \infty\).

Composition

Theorem. Consider a function \(f: \mathbb{R} \rightarrow \mathbb{R}\) is that is \(\mathcal{B} / \mathcal{B}\) measurable. The composition of \(f\) and \(Y\), denoted by \(f \circ Y\) or \(f(Y)\), is a random variable.

Proof. For each Borel set \(E \in \mathcal{B}\),

  1. Its inverse image \(f^{-1} (E)\) is a Borel set since \(f\) is measurable \(\mathcal{B} / \mathcal{B}\).
  2. The inverse image of \(f^{-1} (E)\) belongs to \(\mathcal{F}\) since \(Y\) is a random variable.

It concludes that \(\big( f \circ Y \big)^{-1}(E) \in \mathcal{F}\).

\(\square\)

Expectation Under Composition

On the one hand, \(f \circ Y\) is random variable on \((\Omega, \mathcal{F}, \mathbb{P})\). It expectation is given by \[ \mathbb{E}[ f(Y) ] = \int_{ \Omega } f( Y(\omega) ) \, \mathbb{P} ( d \omega ). \]

Let \(\mu\) be the distribution of \(Y\). Since \((\mathbb{R}, \mathcal{B}, \mu)\) constitutes a probability space, and \(f\) is \(\mathcal{B} / \mathcal{B}\) measurable, it is a random variable defined on \((\mathbb{R}, \mathcal{B}, \mu)\). By definition, its expectation is given by \[ \mathbb{E}[ f ] = \int_{ \mathbb{R} } f(x) \, \mu (d x). \]

It is therefore of interest whether these two definitions are consistent, i.e., whether two integrals are equal.

Theorem. If \(f\) is nonnegative and measurable \(\mathcal{B} / \mathcal{B}\), then \[ \mathbb{E}[ f(Y) ] = \int_{ \Omega } f( Y(\omega) ) \, \mathbb{P} ( d \omega ) = \int_{ \mathbb{R} } f(x) \, \mu (d x) = \mathbb{E}[ f ]. \] Further, \(f\) (not necessarily nonnegative) is integrable with respect to \(\mu = \mathbb{P} \circ Y^{-1}\) if and only if \(f \circ Y = f(Y)\) is integrable with respect to \(\mathbb{P}\), in which case the above equation holds.

Proof.
1. Suppose \(f = \mathbb{1}_{A}\) is an indicator function for a measurable set, then \[ \begin{aligned} \mathbb{E}[ f(Y) ] &= \int_{ \Omega } f( Y(\omega) ) \, \mathbb{P} ( d \omega ) = \int_{ \Omega } \mathbb{1}_{A} ( Y(\omega) ) \, \mathbb{P} ( d \omega ) = \mathbb{P} [ Y \in A],\, \\ \mathbb{E}[ f ] &= \int_{ \mathbb{R} } f(x) \, \mu (d x) = \int_{ \mathbb{R} } \mathbb{1}_{A} (x) \, \mu (d x) = \mu (A) = \mathbb{P} [ Y \in A]. \end{aligned} \] Therefore, \(\mathbb{E}[ f(Y) ] = \mathbb{E}[ f ]\).

  1. By linearity, this holds for nonnegative simple functions.

  2. Suppose that \(f\) is a non-negative function that is measurable \(\mathcal{B} / \mathcal{B}\), then there is a sequence of non-negative simple functions \(f_n \uparrow f\). Hence \(f_n(Y) \uparrow f(Y)\). For each \(n \in \mathbb{N}_+\), it holds that \[ \mathbb{E}[ f_n(Y) ] = \mathbb{E}[ f_n ]. \] The claim follows by monotone convergence theorem.

For general function \(f\) that is measurable \(\mathcal{B} / \mathcal{B}\), applying the above claim to \(|f|\) establishes integrability. Decomposing \(f\) into positive part and negative parts proves the claim.

\(\square\)

Example. Taking \(f(x) = x, \forall x \in \mathbb{R}\), we see \[ \mathbb{E}[ Y ] = \int_{ \Omega } Y (\omega) \, \mathbb{P} (d \omega) = \int_{ \mathbb{R} } x \, \mu (d x). \]

Density

Let \((\Psi, \mathcal{M}, \lambda)\) be arbitrary measurable space.

Theorem. Let \(\delta: \Psi \rightarrow \mathbb{R}_{\ge 0}\) be a function that is measurable \(\mathcal{M} / \mathcal{B}\). Then we can define a measure by \[ \nu(E) \doteq \int_{E} \delta d \lambda, \quad \forall E \in \mathcal{M}. \] \(\nu\) is said to have density \(\delta\) with respect to \(\lambda\).

Note that \(\nu(E)\) can be \(\infty\).

Proof. Clearly \(\nu(\emptyset) = 0\). Let \(E_n \in \mathcal{M}, n \in \mathbb{N}_+\) be disjoint. Then via monotone convergence theorem, \[ \begin{aligned} \nu(\cup_{n \ge 1} E_n) &= \int \delta \cdot \mathbb{1}_{ \cup_{n \ge 1} E_n} d \lambda \\ &= \lim_{n \ge 1} \int \delta \cdot \left( \sum_{k \in [n] }\mathbb{1}_{ E_k } \right) d \lambda \\ &= \sum_{k \ge 1} \int \delta \cdot \mathbb{1}_{ E_k } d \lambda \\ &= \sum_{k \ge 1} \nu(E_k). \end{aligned} \]

\(\square\)

Theorem. Let \(Y: \Psi \rightarrow \mathbb{R}_{\ge 0}\) be measurable \(\mathcal{M} / \mathcal{B}\). Then \[ \int Y \, d \nu = \int Y \cdot \delta \, d \lambda. \]

Proof. 1. If \(Y = \mathbb{1}_{A}\) for \(A \in \mathcal{M}\), then \[ \begin{aligned} \int Y \, d \nu = \int \mathbb{1}_{A} \, d \nu = \nu(A) = \int_A \delta \, d \lambda = \int \mathbb{1}_{A} \cdot \delta \, d \lambda. \end{aligned} \]

  1. Via linearity of expectation, this holds for all non-negative simple functions.

  2. Let \(X_n, n \in \mathbb{N}_+\) be non-negative simple functions such that \(X_n \uparrow Y\). Then \(X_n \cdot \delta \uparrow Y \cdot \delta\). Via monotone convergence theorem, we get \[ \int Y \, d \nu = \lim_n \int X_n \, d \nu = \lim_n \int X_n \cdot \delta \, d \lambda = \int Y \cdot \delta \, d \lambda. \]

\(\square\)

Corollary. Let \(Y: \Psi \rightarrow \mathbb{R}\) be measurable \(\mathcal{M} / \mathcal{B}\). It is integrable with respect to \(\nu\) if and only if \(Y \cdot f\) is integrable with respect to \(\lambda\).

Proof.

By applying the previous theorem to \(|Y|\).

\(\square\)

Equality

Theorem. If \(Y\) is non-negative, then \[ \mathbb{E}[ Y ] = \int_{ \mathbb{R} } x \, d \mu = \int_{ \mathbb{R} } \mathbb{P}[ Y > t ] \, d t = \int_{ \mathbb{R} } \mathbb{P}[ Y \ge t ] \, d t, \] where \(\int_{ \mathbb{R} } \cdot \, d t\) is the Lebesgue integral.

Proof. Since there are countably many of points in \(\mathbb{R}\) for which \(\mathbb{P}(Y = x) \neq 0\), therefore the set for which \(\mathbb{P}(Y \ge t) \neq \mathbb{P}(Y > t)\) has Lebesgue measure 0. It holds that \[ \int_{ \mathbb{R} } \mathbb{P}[ Y > t ] \, d t = \int_{ \mathbb{R} } \mathbb{P}[ Y \ge t ] \, d t. \]

Case One: \(Y\) is simple.

If \(Y = \sum_{i \in [n] } a_i \mathbf{1}_{A_i}\) is a simple non-negative function such that the \[ a_0 \doteq 0 \le a_1 < \ldots < a_n \] are distinct, and the \(A_i\) are disjoint. For each \(i \in [n]\), define \(\Delta_i = a_i - a_{i - 1}\). Then \[ \begin{aligned} \mathbb{E}[ Y ] &= \sum_{i \in [n] } a_i \mathbb{P}(Y = a_i) \\ &= \sum_{i \in [n] } \sum_{j \in [i] } \Delta_j \mathbb{P}(Y = a_i) \\ &= \sum_{j \in [n] } \sum_{i = j }^n \Delta_j \mathbb{P}(Y = a_i) \\ &= \sum_{j \in [n] } \Delta_j \mathbb{P}(Y \ge a_i) \end{aligned} \]

Since for each \(t \in (a_{i - 1}, a_i]\), it holds that \(\mathbb{P}[Y \ge t] = \mathbb{P}[ Y \ge a_i]\), and for each \(t > a_n\), it holds that \(\mathbb{P}[Y \ge t] = 0\), we get \[ \begin{aligned} \mathbb{E}[ Y ] &= \sum_{j \in [n] } \Delta_j \mathbb{P}(Y \ge a_i) \\ &= \sum_{j \in [n] } \int_{ t \in (a_{i - 1}, a_i] } \mathbb{P}(Y \ge t) \, d t \\ &= \int_{ t \in (a_0, a_n] } \mathbb{P}(Y \ge t) \, d t \\ &= \int_{ t = 0 }^\infty \mathbb{P}(Y \ge t) \, d t. \end{aligned} \]

Since \(\mathbb{P}(Y \ge t) = \mathbb{P} (Y > t)\) almost everywhere \[ \mathbb{E}[ Y ] = \int_{ t = 0 }^\infty \mathbb{P}(Y \ge t) \, d t. = \int_{ t = 0 }^\infty \mathbb{P}(Y > t) \, d t. \]

Case Two: \(Y\) is general non-negative measurable function.

If \(Y\) is non-negative, then there is an increasing sequence of simple random variables \(\{ X_n \} \uparrow Y\). Hence \[ \begin{aligned} \mathbb{P}( X_n > t) &= \mathbb{P} \left( \{ \omega \in \Omega : X_n(\omega) > t \} \right) \\ &\uparrow \mathbb{P} (Y > t) = \mathbb{P} \left( \{ \omega \in \Omega : Y(\omega) > t \} \right). \end{aligned} \] As \[ \mathbb{E}[ X_n ] = \int_{ t = 0 }^\infty \mathbb{P}(X_n > t) \, d t. \] By applying monotone convergence theorem to both hand side,
\[ \mathbb{E}[ Y ] = \int_{ t = 0 }^\infty \mathbb{P}(Y > t) \, d t. \]

Remark: We can't argue that \(\mathbb{P}(X_n \ge t) \uparrow \mathbb{P}(Y \ge t)\), even though \(X_n \uparrow Y\). For example, the functions \(X_n = (1 - 1 / n) \cdot \mathbb{1}_{[0, 1]}\) increases to \(Y = \mathbb{1}_{[0, 1]}\). But \(\mathbb{P}(X_n \ge 1) = 0\) and \(\mathbb{P}(Y \ge 1) = 1\).

\(\square\)

Total Variation Distance and Product Measure

Posted on 2021-11-16

Let \(\mu_i, \nu_i\) be probability measures on the finite space \((\Omega_i, 2^{\Omega_i} )\), \(i \in [n]\). Define \(\mu = \prod_{i \in [n]} \mu_i\) and \(\nu = \prod_{i \in [n]} \nu_i\) be the product measure on the product space \((\Omega, 2^{\Omega})\). Then \[ \Vert \mu - \nu \Vert_{TV} \le \sum_{i \in [n] } \Vert \mu_i - \nu_i \Vert_{TV}, \] where \(\Vert \cdot \Vert_{TV}\) is the total variation distance.

Proof. Consider an \(x = (x_1, \ldots, x_n) \in \Omega\).

For each \(i \in 0, \ldots, n\), let \[ s(i) \doteq \mu_1 (x_1) \cdots \mu_i(x_i) \cdot \nu_{i + 1} (x_{i + 1} ) \cdots \nu_n (x_n). \] So \(s(0) = \nu(x)\) and \(s(n) = \mu(x)\). It follows that \[ \begin{aligned} \nu(x) - \mu(x) &= \sum_{i \in [n]} \left( s(i - 1) - s(i) \right) \\ &= \sum_{i \in [n]} \left( \big( \nu_i(x_i) - \mu_i(x_i) \big) \cdot \mu_1 (x_1) \cdots \mu_{i - 1} (x_{i - 1}) \cdot \nu_{i + 1} (x_{i + 1} ) \cdots \nu_n (x_n)\right). \end{aligned} \] Hence, \[ \begin{aligned} 2 \cdot \Vert \mu - \nu \Vert_{TV} &= \sum_{x \in \Omega} \vert \nu(x) - \mu(x) \vert \\ &= \sum_{x \in \Omega} \left\vert \sum_{i \in [n]} \big( \nu_i(x_i) - \mu_i(x_i) \big) \cdot \mu_1 (x_1) \cdots \mu_{i - 1} (x_{i - 1}) \cdot \nu_{i + 1} (x_{i + 1} ) \cdots \nu_n (x_n)\right\vert \\ &\le \sum_{x \in \Omega} \sum_{i \in [n]} \left\vert \nu_i(x_i) - \mu_i(x_i) \right\vert \cdot \mu_1 (x_1) \cdots \mu_{i - 1} (x_{i - 1}) \cdot \nu_{i + 1} (x_{i + 1} ) \cdots \nu_n (x_n) \\ &= \sum_{i \in [n]} \sum_{x_i \in \Omega_i} \left\vert \nu_i(x_i) - \mu_i(x_i) \right\vert \\ &= \sum_{i \in [n]} 2 \cdot \Vert \mu_i - \nu_i \Vert_{TV}. \end{aligned} \]

\(\square\)
