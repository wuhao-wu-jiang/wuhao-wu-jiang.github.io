BGW Protocol

Posted on 2024-11-09 Edited on 2024-11-15

The BGW is a secure multiparty computation protocol for polynomials over a finite field.

Model

Let ppp be a prime number, and let Fp\mathbb{F}_pFp​ be a finite field of order ppp.

In the setting, there are nnn parties, where the party iii holds data bi,0∈Fpb_{i, 0} \in \mathbb{F}_pbi,0​∈Fp​.
The objective is to compute a polynomial f(x1,…,xn)f (x_1, \ldots, x_n)f(x1​,…,xn​) over Fp\mathbb{F}_pFp​ in a privacy-preserving manner.

Privacy Guarantee.
We assume all parties are curious but honest.
A protocol is considered ttt-private, if any set of up to ttt players cannot learn more from protocol’s outcome than they could jointly infer from their own private inputs and outputs.

Protocol

We need the following protocol as a primitive for sharing information between the parties.

Shamir Secret Sharing Scheme S\mathcal{S}S
Party i∈[n]:i \in [n] :i∈[n]:

  1. Sample independent random primes bi,1,…,bi,t∈Fpb_{i, 1}, \ldots, b_{i, t} \in \mathbb{F}_pbi,1​,…,bi,t​∈Fp​.
  2. Let b⃗i≐(bi,0,bi,1,…,bi,t).\vec{b}_i \doteq (b_{i, 0}, b_{i, 1}, \ldots, b_{i, t}).bi​≐(bi,0​,bi,1​,…,bi,t​).
  3. Let gb⃗i(x)≐bi,0+bi,1⋅x+…+bi,t⋅xtg_{\vec{b}_i} (x) \doteq b_{i, 0} + b_{i, 1} \cdot x + \ldots + b_{i, t} \cdot x^tgbi​​(x)≐bi,0​+bi,1​⋅x+…+bi,t​⋅xt.
  4. For each user j∈[n]j \in [n]j∈[n]:
  5. \quad Send gb⃗i(j)g_{\vec{b}_i} (j)gbi​​(j) to user jjj

We call gb⃗i(j)g_{\vec{b}_i} (j)gbi​​(j) a share of the secret bi,0b_{i, 0}bi,0​.

Privacy Guarantee. It is easy to verify that S\mathcal{S}S is ttt-private.

The key result we would like to establish is that a polynomial f(x1,…,xn)f(x_1, \ldots, x_n)f(x1​,…,xn​) can also be computed in a ttt-private manner.

Since a polynomial can be constructed through additions, scalar multiplication, and the multiplication of (smaller) polynomials, it is sufficient to demonstrate how these operations can be performed in a ttt-private manner.
We begin by discussing the addition protocol.
Protocols for subtraction and scalar multiplication can be derived using similar techniques.
Finally, we present the multiplication protocol.

Addition

We start with a simple protocol, which tries to add up bk,0b_{k, 0}bk,0​ and bℓ,0b_{\ell, 0}bℓ,0​, for some parties k,ℓ∈[n]k, \ell \in [n]k,ℓ∈[n].

Adding bk,0b_{k, 0}bk,0​ and bℓ,0b_{\ell, 0}bℓ,0​:

Party kkk:

  • Invoke S\mathcal{S}S to send gb⃗k(j)g_{\vec{b}_k}(j)gbk​​(j) to each j∈[n]j \in [n]j∈[n].

Party ℓ\ellℓ:

  • Invoke S\mathcal{S}S to send gb⃗ℓ(j)g_{\vec{b}_\ell} (j)gbℓ​​(j) to each j∈[n]j \in [n]j∈[n].

Party i∈[n]i \in [n]i∈[n]:

  • On receiving gb⃗k(i)g_{\vec{b}_k}(i)gbk​​(i) and gb⃗ℓ(i)g_{\vec{b}_\ell}(i)gbℓ​​(i), compute gb⃗k(i)+gb⃗ℓ(i)g_{\vec{b}_k}(i) + g_{\vec{b}_\ell}(i)gbk​​(i)+gbℓ​​(i).

The key observation is that the following relation holds for polynomials:

gb⃗k(x)+gb⃗ℓ(x)=gb⃗k+b⃗ℓ(x)=(bk,0+bℓ,0)+(bk,1+bℓ,1)⋅x+…+(bk,t+bℓ,t)⋅xt.\begin{aligned} g_{\vec{b}_k}(x) + g_{\vec{b}_\ell}(x) &= g_{\vec{b}_k + \vec{b}_\ell}(x) \\ &= (b_{k, 0} + b_{\ell, 0}) + (b_{k, 1} + b_{\ell, 1}) \cdot x + \ldots + (b_{k, t} + b_{\ell, t}) \cdot x^t. \end{aligned} gbk​​(x)+gbℓ​​(x)​=gbk​+bℓ​​(x)=(bk,0​+bℓ,0​)+(bk,1​+bℓ,1​)⋅x+…+(bk,t​+bℓ,t​)⋅xt.​

Composition in the View of a Virtual Party

Thus, the addition operation can be viewed as a virtual party vvv holding data bk,0+bℓ,0b_{k, 0} + b_{\ell, 0}bk,0​+bℓ,0​, which is then shared with all parties i∈[n]i \in [n]i∈[n] via the Shamir Secret Sharing Scheme.
In this scheme, the polynomial coefficients are b⃗k+b⃗ℓ\vec{b}_k + \vec{b}_\ellbk​+bℓ​.
Consequently, each party iii receives a share of the secret, denoted as gb⃗k+b⃗ℓ(i)g_{\vec{b}_k + \vec{b}_\ell}(i)gbk​+bℓ​​(i).

Virtual Party vvv:

  • Invoke S\mathcal{S}S to send gb⃗k+b⃗ℓ(j)g_{\vec{b}_k + \vec{b}_\ell}(j)gbk​+bℓ​​(j) to each j∈[n]j \in [n]j∈[n].

This alternative view facilitates a clearer understanding of composition.

For example, suppose we want to compute bk,0+bℓ,0+bm,0b_{k, 0} + b_{\ell, 0} + b_{m, 0}bk,0​+bℓ,0​+bm,0​ for some parties k,ℓ,m∈[n]k, \ell, m \in [n]k,ℓ,m∈[n]. First, we can apply the protocol described above to compute bk,0+bℓ,0b_{k, 0} + b_{\ell, 0}bk,0​+bℓ,0​.

Then, we consider two parties: a virtual party vvv holding data bk,0+bℓ,0b_{k, 0} + b_{\ell, 0}bk,0​+bℓ,0​, and party mmm with data bm,0b_{m, 0}bm,0​. We can then invoke the addition protocol again to compute bk,0+bℓ,0+bm,0b_{k, 0} + b_{\ell, 0} + b_{m, 0}bk,0​+bℓ,0​+bm,0​. Importantly, communication with the virtual party is not actually required, as after the first protocol invocation, each party iii already possesses the secret share gb⃗k+b⃗ℓ(j)g_{\vec{b}_k + \vec{b}_\ell}(j)gbk​+bℓ​​(j).

Privacy Guarantee.
After the addition protocol, each party iii receives a share of the secret bk,0+bℓ,0b_{k, 0} + b_{\ell, 0}bk,0​+bℓ,0​, represented by gb⃗k+b⃗ℓ(i)g_{\vec{b}_k + \vec{b}_\ell}(i)gbk​+bℓ​​(i).
Since gb⃗k+b⃗ℓ(x)g_{\vec{b}_k + \vec{b}_\ell}(x)gbk​+bℓ​​(x) is a random polynomial of degree ttt, the output of the addition protocol remains ttt-private.

Multiplication

The multiplication is more intricate.
Note that:

gb⃗k(x)⋅gb⃗ℓ(x)=gb⃗k×b⃗ℓ(x)=(bk,0⋅bℓ,0)+(∑i,j≥0, i+j=1bk,i⋅bℓ,j)⋅x+⋯+(∑i,j≥0, i+j=2tbk,i⋅bℓ,j)⋅x2t.\begin{aligned} g_{\vec{b}_k}(x) \cdot g_{\vec{b}_\ell}(x) &= g_{\vec{b}_k \times \vec{b}_\ell}(x) \\ &= \left( b_{k, 0} \cdot b_{\ell, 0} \right) + \left(\sum_{i, j \ge 0, \, i + j = 1} b_{k, i} \cdot b_{\ell, j}\right) \cdot x + \cdots + \left(\sum_{i, j \ge 0, \, i + j = 2 t} b_{k, i} \cdot b_{\ell, j}\right) \cdot x^{2 t}. \end{aligned} gbk​​(x)⋅gbℓ​​(x)​=gbk​×bℓ​​(x)=(bk,0​⋅bℓ,0​)+(i,j≥0,i+j=1∑​bk,i​⋅bℓ,j​)⋅x+⋯+(i,j≥0,i+j=2t∑​bk,i​⋅bℓ,j​)⋅x2t.​

There are two problems with this polynomial:

  1. The coefficients of this polynomial are not independent.
  2. The polynomial has degree 2t2t2t, instead of ttt.

Hence the previous security analysis of the Shamir Secret Sharing Scheme does not hold.

Rerandomization

To fix the first problem, each party iii can generate a random polynomial

gi(x)=0+ci,1⋅x+ci,2⋅x2+⋯+ci,2t⋅x2t, g_i (x) = 0 + c_{i, 1} \cdot x + c_{i, 2} \cdot x^2 + \cdots + c_{i, 2t} \cdot x^{2t}, gi​(x)=0+ci,1​⋅x+ci,2​⋅x2+⋯+ci,2t​⋅x2t,

then they sends the shares gi(1),…,gi(n)g_i(1), \ldots, g_i(n)gi​(1),…,gi​(n) to other parties.

After sending and receiving new shares, each party iii then computes gb⃗k×b⃗ℓ(i)+∑j∈[n]gj(i)g_{\vec{b}_k \times \vec{b}_\ell}(i) + \sum_{j \in [n]} g_j(i)gbk​×bℓ​​(i)+∑j∈[n]​gj​(i).
The polynomial of interest is given by

gb⃗k×b⃗ℓ(x)+∑i∈[n]gi(x)=(bk,0⋅bℓ,0)+(∑i,j≥0, i+j=1bk,i⋅bℓ,j+∑i∈[n]ci,1)⋅x+⋯+(∑i,j≥0, i+j=2tbk,i⋅bℓ,j+∑i∈[n]ci,2t)⋅x2t.\begin{aligned} g_{\vec{b}_k \times \vec{b}_\ell}(x) + \sum_{i \in [n]} g_i (x) &= \left( b_{k, 0} \cdot b_{\ell, 0} \right) + \left(\sum_{i, j \ge 0, \, i + j = 1} b_{k, i} \cdot b_{\ell, j} + \sum_{i \in [n]} c_{i, 1} \right) \cdot x + \cdots + \left(\sum_{i, j \ge 0, \, i + j = 2 t} b_{k, i} \cdot b_{\ell, j} + \sum_{i \in [n]} c_{i, 2t} \right) \cdot x^{2 t}. \end{aligned} gbk​×bℓ​​(x)+i∈[n]∑​gi​(x)​=(bk,0​⋅bℓ,0​)+⎝⎛​i,j≥0,i+j=1∑​bk,i​⋅bℓ,j​+i∈[n]∑​ci,1​⎠⎞​⋅x+⋯+⎝⎛​i,j≥0,i+j=2t∑​bk,i​⋅bℓ,j​+i∈[n]∑​ci,2t​⎠⎞​⋅x2t.​

Note this is a random polynomial, in the sense that the coefficients of this polynomial are independent.
Therefore, the outputs gb⃗k×b⃗ℓ(i)+∑j∈[n]gj(i),i∈[n]g_{\vec{b}_k \times \vec{b}_\ell}(i) + \sum_{j \in [n]} g_j(i), i \in [n]gbk​×bℓ​​(i)+∑j∈[n]​gj​(i),i∈[n] are 2t2t2t-private.

Remark.
It appears sufficient for one party iii (if trusted) to generate a random polynomial gig_igi​ and share gi(1),…,gi(n)g_i(1), \ldots, g_i(n)gi​(1),…,gi​(n) with other parties.
Essentially, this adds a random coefficient to each coefficient of gb⃗k×b⃗ℓ(x)g_{\vec{b}_k \times \vec{b}_\ell}(x)gbk​×bℓ​​(x).

Truncation

To simplify notation, let

h(x)≐c0+c1⋅x+…+c2t⋅x2t h(x) \doteq c_0 + c_1 \cdot x + \ldots + c_{2t} \cdot x^{2t} h(x)≐c0​+c1​⋅x+…+c2t​⋅x2t

be a shorthand for the polynomial gb⃗k×b⃗ℓ(x)+∑i∈[n]gi(x)g_{\vec{b}_k \times \vec{b}_\ell}(x) + \sum_{i \in [n]} g_i (x)gbk​×bℓ​​(x)+∑i∈[n]​gi​(x).
We have shown that c1,…,cnc_1, \ldots, c_nc1​,…,cn​ are random integers from Fp\mathbb{F}_pFp​.

Our goal is to truncate its degree from 2t2t2t to ttt:

hˉ(x)≐c0+c1⋅x+…+ct⋅xt, \bar{h}(x) \doteq c_0 + c_1 \cdot x + \ldots + c_{t} \cdot x^{t}, hˉ(x)≐c0​+c1​⋅x+…+ct​⋅xt,

and for each party i∈[n]i \in [n]i∈[n], replaces its share h(i)h(i)h(i) by hˉ(i)\bar{h}(i)hˉ(i).

A naive approach is to reveal the secret shares of a set of 2t2t2t clients, for example h(1),…,h(2t)h(1), \ldots, h(2 t)h(1),…,h(2t), so that we can recover the entire polynomial h(x)h(x)h(x) (and therefore the secret value h(0)=c0h(0) = c_0h(0)=c0​).

Can we achieve this in a ttt-private manner?
The answer is “Yes”!
We are going to show that, there is a matrix TTT, s.t.,

hˉ1:n≐[hˉ(1)hˉ(2)⋮hˉ(n)],h1:n=[h(1)h(2)⋮h(n)],hˉ1:n=T×h1:n. \bar{h}_{1 : n} \doteq \begin{bmatrix} \bar{h}(1) \\ \bar{h}(2) \\ \vdots \\ \bar{h}(n) \end{bmatrix}, \quad h_{1 : n} = \begin{bmatrix} h(1) \\ h(2) \\ \vdots \\ h(n) \end{bmatrix}, \quad \bar{h}_{1 : n} = T \times h_{1 : n}. hˉ1:n​≐⎣⎢⎢⎢⎢⎡​hˉ(1)hˉ(2)⋮hˉ(n)​⎦⎥⎥⎥⎥⎤​,h1:n​=⎣⎢⎢⎢⎢⎡​h(1)h(2)⋮h(n)​⎦⎥⎥⎥⎥⎤​,hˉ1:n​=T×h1:n​.

Observe that, multiplying h1:nh_{1 : n}h1:n​ by TTT only involves addition and scalar multiplications of secret shares, which we have already shown can be performed in a ttt-private manner.
It remain to explain what TTT is.

Define

V=[1,1,⋯ ,11,2,⋯ ,2n−1⋯ 1,i,⋯ ,in−1⋯ 1,n,⋯ ,nn−1],c⃗=[c0c1⋮c2t0⋮0],cˉ=[c0c1⋮ct0⋮0]. V=⎣⎢⎢⎢⎢⎢⎢⎢⎡​1,1,1,1,​1,2,i,n,​⋯,⋯,⋯⋯,⋯⋯,​12n−1in−1nn−1​⎦⎥⎥⎥⎥⎥⎥⎥⎤​,c=⎣⎢⎢⎢⎢⎢⎢⎢⎢⎢⎢⎢⎡​c0​c1​⋮c2t​0⋮0​⎦⎥⎥⎥⎥⎥⎥⎥⎥⎥⎥⎥⎤​,cˉ=⎣⎢⎢⎢⎢⎢⎢⎢⎢⎢⎢⎢⎡​c0​c1​⋮ct​0⋮0​⎦⎥⎥⎥⎥⎥⎥⎥⎥⎥⎥⎥⎤​.

Then h1:n=V×c⃗h_{1: n} = V \times \vec{c}h1:n​=V×c, and hˉ1:n=V×cˉ\bar{h}_{1: n} = V \times \bar{c}hˉ1:n​=V×cˉ.
We also know that there exists a projection matrix PPP, s.t., cˉ=Pc⃗\bar{c} = P \vec{c}cˉ=Pc.
It follows that

hˉ1:n=V×cˉ=V×P×c⃗=V×P×V−1×h1:n. \bar{h}_{1 : n} = V \times \bar{c} = V \times P \times \vec{c} = V \times P \times V^{-1} \times h_{1 : n}. hˉ1:n​=V×cˉ=V×P×c=V×P×V−1×h1:n​.

Taking T=V×P×V−1T = V \times P \times V^{-1}T=V×P×V−1 computes the proof.

Remark. If we directly compute V−1×h1:nV^{-1} \times h_{1 : n}V−1×h1:n​, we recover all coefficients of h(x)h(x)h(x).
The key trick is to compute (V×P×V−1)×h1:n(V \times P \times V^{-1}) \times h_{1 : n}(V×P×V−1)×h1:n​ instead, bypassing the explicit computation of V−1×h1:nV^{-1} \times h_{1 : n}V−1×h1:n​.
