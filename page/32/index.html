Bit Guessing Game

Posted on 2020-10-25 Edited on 2020-11-02

The bit guessing game models two players, namely Alice and Bob, where Alice has a secret vector x⃗∈{0,1}n\vec x \in \{0, 1 \}^nx∈{0,1}n which Bob tries to learn. Bob is only allowed to ask Alice the value of

c⃗⋅x⃗\vec c \cdot \vec x c⋅x

for bit vector c⃗∈{0,1}n\vec c \in \{0, 1\}^nc∈{0,1}n he comes up with. Under this setting, it is trivial for Bob to learn each bit of xxx: he needs to just query Alice for e⃗i⋅x⃗\vec e_i \cdot \vec xei​⋅x, where e⃗i\vec e_iei​ is the indicator vector with its iii-th dimension equal to 1 and other dimensions equal to 0.

One way for Alice to prevent information leakage is to add some bounded noise ZZZ to the query result and returns

r(c⃗)≐c⃗⋅x⃗+Zr(\vec c) \doteq \vec c \cdot \vec x + Z r(c)≐c⋅x+Z

For example, ZZZ could be a bounded random variable. Is is still a chance for Bob to learn the vector x⃗\vec xx?

Theorem 1. If Bob is allowed to query r(c⃗)r(\vec c)r(c) for all c⃗∈{0,1}n\vec c \in \{0, 1\}^nc∈{0,1}n, and suppose that ZZZ is bounded by some real number RRR, then Bob can have an estimation y⃗\vec yy​ such that

∣x⃗−y⃗∣=∑i=1n∣xi−yi∣≤4R|\vec x - \vec y| = \sum_{i = 1}^n | x_i - y_i | \le 4R ∣x−y​∣=i=1∑n​∣xi​−yi​∣≤4R

Remark: In this case Bob queries for 2n2^n2n vectors. This result implies that is Bob is able to ask exponential number of questions, then it is hard to prevent information leakage. E.g., suppose R=n/400R = n / 400R=n/400, which is a relatively large number when nnn is large, the theorem says that Bob can recover 99%99\%99% of bits in x⃗\vec xx.
