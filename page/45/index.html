Chapter Two
Context-Free Grammar

A context-free grammar is a four-tuple ⟨N,T,S,R⟩\langle N, T, S, R\rangle⟨N,T,S,R⟩, where

  • N is a finite set of symbols called the variables(non-terminals),
  • T is a finite set of symbols, disjoint from N, called the terminals,
  • S∈NS \in NS∈N is the start symbol and
  • R is a finite set of rules(productions), with each rule of the form X→ωX \to \omegaX→ω, where X∈NX \in NX∈N and ω∈(N∪T)+\omega \in (N \cup T)^+ω∈(N∪T)+.
Chomsky Normal Form

A context-free grammar ⟨N,T,S,R⟩\langle N, T, S, R\rangle⟨N,T,S,R⟩ is in chomsky normal form if each rule in R is of one of the two forms

  • X→YZX \to YZX→YZ, where X,Y,Z∈NX, Y, Z \in NX,Y,Z∈N,
  • X→aX \to aX→a, where X∈N,a∈TX \in N, a \in TX∈N,a∈T.

In addition, we permit S→ϵS \to \epsilonS→ϵ, where SSS is the start variable.

Pushdown Automata

A pushdown automaton is a six-tuple ⟨Q,Σ,Γ,δ,S,F⟩\langle Q, \Sigma, \Gamma, \delta, S, F \rangle⟨Q,Σ,Γ,δ,S,F⟩, where

  • Q is a finite set of states,
  • Σ\SigmaΣ is a finite set of input symbols,
  • Γ\GammaΓ is a finite set of stack symbols,
  • δ:Q×Σϵ×Γϵ→P(Q×Γ)\delta : Q \times \Sigma_{\epsilon} \times \Gamma_{\epsilon} \to P(Q \times \Gamma)δ:Q×Σϵ​×Γϵ​→P(Q×Γ) is the transition function, where P(Q×Γ)P(Q \times \Gamma)P(Q×Γ) is the power set of Q×ΓQ \times \GammaQ×Γ,
  • S∈QS \in QS∈Q is the start state and
  • F⊆QF \subseteq QF⊆Q is a finite set of accept states.
Equivalence between Context-Free Grammar and Pushdown Automata

  1. For every context-free grammar G, there is an automaton M, such that M recognizes the language generated by G.

  2. If a pushdown automaton recognizes some language, then it is context-free.

Proof: For the convenience of proof, we modify a pushdown automaton M=⟨Q,Σ,Γ,δ,S,F⟩M = \langle Q, \Sigma, \Gamma, \delta, S, F \rangleM=⟨Q,Σ,Γ,δ,S,F⟩ to give it three features:

  • M accepts by entering empty stack.
  • These is only one accept state qaccpetq_{accpet}qaccpet​ in F.
  • Each transition either pushes a symbol onto the stack or pops one off the stack, but it does not do both at the same time.

The way to construct a context-free grammar G from the push-down automaton M is given by follow (Introduction To The Theory Of Computation 2nd Edition by Michael Sipser, p120):

(1) ∀p∈Q\forall p \in Q∀p∈Q, we put the rule Npp→ϵN_{pp} \to \epsilonNpp​→ϵ in G.
(2) ∀p,q,r∈Q\forall p,q,r \in Q∀p,q,r∈Q, we add to G a new rule Npq→NprNrqN_{pq} \to N_{pr} N_{rq}Npq​→Npr​Nrq​,
(3) ∀p,q,r,s∈Q\forall p, q, r, s \in Q∀p,q,r,s∈Q, if ∃a,b∈Σϵ\exists a, b\in \Sigma_\epsilon∃a,b∈Σϵ​, t∈Γt \in \Gammat∈Γ, s.t. (r,t)∈δ(p,a,ϵ)(r,t) \in \delta(p,a,\epsilon)(r,t)∈δ(p,a,ϵ) and $ (q,\epsilon) \in \delta(s,b,t)$, then we put the rule Npq→aNrsbN_{pq} \to a N_{rs}bNpq​→aNrs​b in G.

Heading down this way, we can get the grammar for pushdown automaton M1. But following (3), we also get productions like:

These rules are useless. They can not derive any string of terminals. To understand the real cause of the problem, take Ndn→NemN_{dn} \to N_{em}Ndn​→Nem​ for example. When the push down automaton goes from state qdq_dqd​ to state qeq_eqe​, it pushes a YYY into the stack. And a YYY is poped during the transition from state qmq_mqm​ to qnq_nqn​. But the YYY poped is not the same YYY pushed during the transition from state qdq_dqd​ to qeq_eqe​. At state qnq_nqn​, the stack does not recover to its initial state before the automaton entering state qdq_dqd​. Some additional non-terminal symbols are still in the stack.

Ullman give another way of converting a pushdown automaton into context-free grammar. In that way, we can avoid construct meaningless rules even for pushdown automaton M1. The key idea is, rule like Npq→aNrsbN_{pq} \to a N_{rs}bNpq​→aNrs​b is constructed if and only if in state qqq the stack is the same as what it is before the automaton entering state ppp.
