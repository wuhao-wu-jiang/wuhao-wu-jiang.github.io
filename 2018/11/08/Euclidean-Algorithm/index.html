Euclidean Algorithm

Posted on 2018-11-08 Edited on 2021-03-18

We discuss the Euclidean algorithm and use it to compute the multiplicative inverse.

Multiplicative Inverse.
Given x,y∈N+x, y \in \mathbb{N}^+x,y∈N+, zzz is called the multiplicative inverse of xxx (module yyy) if

xz≡1(mody). xz \equiv 1 \pmod y. xz≡1(mody).

Theorem 1

xxx has a multiplicative inverse module y  ⟺  gcd⁡(x,y)=1y \iff \gcd(x, y) = 1y⟺gcd(x,y)=1.

Proof.
Define the set S={ax+by∣a,b∈Z}S = \{ ax + by \mid a, b \in Z\}S={ax+by∣a,b∈Z}. Let d=a∗x+b∗yd = a^* x + b^* yd=a∗x+b∗y be the smallest positive integer in SSS. Our proof relies on the following lemma.

Lemma: gcd⁡(x,y)=d\gcd(x,y) = dgcd(x,y)=d.

First, as ddd is a linear combination of xxx and yyy, gcd⁡(x,y)∣d\gcd(x, y) | dgcd(x,y)∣d and gcd⁡(x,y)≤d\gcd(x, y) \le dgcd(x,y)≤d.

Second, we claim that d∣xd | xd∣x and d∣yd | yd∣y. By symmetry, we prove only the former. If not, suppose that d∤xd \nmid xd∤x. Then ∃k∈Z,ℓ∈[1,d−1]\exists k \in \mathbb{Z}, \ell \in [1, d - 1]∃k∈Z,ℓ∈[1,d−1], s.t.,

x=kd+ℓ. x = k d + \ell. x=kd+ℓ.

Since ℓ=x−kd\ell = x - k dℓ=x−kd is a linear combination of xxx and yyy, it holds that ℓ∈S\ell \in Sℓ∈S, contradicting ddd being the smallest positive element in SSS.

It concludes that ddd equals to gcd⁡(x,y)\gcd(x, y)gcd(x,y).

□\square□

The proof of the above theorem immediately follows from the lemma.

  1. gcd⁡(x,y)=1→x\gcd(x, y) = 1 \rightarrow xgcd(x,y)=1→x has a multiplicative inverse.
    In this case, d=a∗x+b∗y=1d = a^* x + b^* y = 1d=a∗x+b∗y=1. a∗a^*a∗ is th multiplicative inverse of xxx module yyy.

  2. xxx has a multiplicative inverse →gcd⁡(x,y)=1\rightarrow \gcd(x, y) = 1→gcd(x,y)=1.
    If xxx has a multiplicative inverse, there exists some zzz and lll, such that zx=ly+1zx = ly + 1zx=ly+1.

Theorem 2

If xxx has a multiplicative inverse module yyy, it is unique.

Proof.
Let aaa and bbb both be the multiplicative inverse of xxx module yyy, then

a≡a(xb)≡(ax)b≡b(mody) a \equiv a (xb) \equiv (ax) b \equiv b \pmod y a≡a(xb)≡(ax)b≡b(mody)

Basic Euclidean Algorithm

Euclidean algorithm computes gcd⁡(x,y)\gcd(x, y)gcd(x,y) as follows:

Correctness

If y=0y =0y=0, whether x=0x = 0x=0 or not, we have gcd⁡(x,y)=x\gcd(x, y) = xgcd(x,y)=x.

If x<yx < yx<y, xmod  y=xx \mod y = xxmody=x. The algorithm exchanges xxx and yyy at the first recursion. It suffices to consider only x≥y>0x \ge y > 0x≥y>0.

If x≥y>0x \ge y > 0x≥y>0, then gcd⁡(x,y)=gcd⁡(y,xmod  y)\gcd(x, y) = \gcd(y, x \mod y)gcd(x,y)=gcd(y,xmody). Let d=gcd⁡(x,y)d = \gcd(x, y)d=gcd(x,y). Clearly d∣xd | xd∣x and d∣yd | yd∣y. We claim that

d∣(xmod  y) d | (x \mod y) d∣(xmody)

Suppose x=kdx = k dx=kd and y=ℓdy = \ell dy=ℓd, then xmod  y=d⋅(kmod  ℓ)x \mod y = d \cdot (k \mod \ell)xmody=d⋅(kmodℓ), which finishes proof.

Time

Let nnn be the the total number of bits to represent xxx and yyy. Then the algorithm stops after O(n)O(n)O(n) recursions.

Proof.

We can assume x≥yx \ge yx≥y. Otherwise, they are swapped by a recursive call.

The key observation is that, after every two recursive calls, the first argument xxx shrinks at least by a factor of two.

  1. If y≤x/2y \le x / 2y≤x/2, the observation holds after the first call.
  2. If y>x/2y > x / 2y>x/2, then after two recursive call, the first argument becomes xmod  yx \mod yxmody, which is smaller than x/2x / 2x/2.

□\square□

Extended Euclidean Algorithm

We extend the basic version to return a pair a∗,b∗∈Na^*, b^* \in \mathbb{N}a∗,b∗∈N s.t. gcd⁡(x,y)=d=a∗x+b∗y\gcd(x, y) = d = a^* x + b^* ygcd(x,y)=d=a∗x+b∗y.

Correctness

Suppose s=a′y+b′⋅(xmod  y)s = a' y + b' \cdot (x \mod y)s=a′y+b′⋅(xmody). Then

s=a′y+b′(x−⌊x/y⌋⋅y)=b′x+(a′−b′⋅⌊x/y⌋)⋅y s = a' y + b'(x - \lfloor x / y \rfloor \cdot y) =b' x + (a' - b' \cdot \lfloor x / y \rfloor) \cdot y s=a′y+b′(x−⌊x/y⌋⋅y)=b′x+(a′−b′⋅⌊x/y⌋)⋅y

This is why the algorithm work.

□\square□
