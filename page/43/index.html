Priority Sampling

A trivial solution is to set S=[n]S = [n]S=[n] and we can answer the query exactly. This is an overkill if our goal is to only answer the query approximately.

Alternatively, we can sample each item i∈[n]i \in [n]i∈[n] independently with probability pip_ipi​ (to be determined), and maintain a random variable

Xi={wipii is chosen0 else \begin{aligned} X_i = \begin{cases} \frac{w_i}{p_i} & i \text{ is chosen} \\ 0 & \text{ else } \end{cases} \end{aligned} Xi​={pi​wi​​0​i is chosen else ​​

Note that

E[Xi]=wi\mathbb{E} \left[ X_i \right] = w_i E[Xi​]=wi​

E[∑i∈IXi]=∑i∈IE[Xi]=∑i∈Iwi\mathbb{E} \left[\sum_{i \in I} X_i \right] = \sum_{i \in I} \mathbb{E}[X_i] = \sum_{i \in I} w_i E[i∈I∑​Xi​]=i∈I∑​E[Xi​]=i∈I∑​wi​

Threshold Sampling

One possible implementation of this sampling scheme is called threshold sampling.

First, a threshold TTT is chosen. Then, for each item i∈[n]i \in [n]i∈[n], if wi≥Tw_i \ge Twi​≥T, then the algorithm record a value Xi=wiX_i = w_iXi​=wi​ deterministically. Mathematically, the two cases can be combined into one. Let uiu_iui​ be a random number generated between (0,1)(0, 1)(0,1). Then

Xi={max⁡{wi,T}ui<wiT0 else \begin{aligned} X_i = \begin{cases} \max \{ w_i, T \} & u_i < \frac{w_i}{T} \\ 0 & \text{ else } \end{cases} \end{aligned} Xi​={max{wi​,T}0​ui​<Twi​​ else ​​

The problem with threshold sampling is that it is hard to control the memory usage of the sample set SSS. If the nnn items are known offline, and we would like ∣S∣≈k|S| \approx k∣S∣≈k for some integer k≤nk \le nk≤n, then we can search for some TTT such that
