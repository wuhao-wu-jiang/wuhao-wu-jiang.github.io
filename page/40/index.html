Working Set BST

Posted on 2020-03-23 Edited on 2020-03-29

Various implementations of Balanced Binary Search Tree (BST) guaranteed membership queries in O(log⁡n)O(\log n)O(logn) time, where nnn is the number of elements in some set SSS. The working set property is a stronger requirement than O(log⁡n)O(\log n)O(logn) time complexity. Suppose that an item x∈Sx \in Sx∈S is accessed ttt queries before, then we want to perform find(x)\text{find} (x)find(x) in O(log⁡t)O(\log t)O(logt) time.

n0=1n1=2n2=4n3=16...ni=22i−1ni+1=ni2=22i...nk=22k−1\begin{aligned} & n_0 = 1 \\ & n_1 = 2 \\ & n_2 = 4 \\ & n_3 = 16 \\ & ... \\ & n_i = 2^{2^{i - 1} } \\ & n_{i + 1} = n_i^2 = 2^{2^i } \\ & ... \\ & n_k = 2^{2^{k - 1} } \end{aligned} ​n0​=1n1​=2n2​=4n3​=16...ni​=22i−1ni+1​=ni2​=22i...nk​=22k−1​

The largest tree has an index that equals to the smallest kkk with 2k−1≥log⁡n2^{k - 1} \ge \log n2k−1≥logn (assuming that n≥1n \ge 1n≥1). It implies that k=O(log⁡log⁡n)k = O (\log \log n)k=O(loglogn). The complexities for searching an element on these trees are

When a query find(x)\text{find} (x)find(x) is invoked, we first search it on T0T_0T0​. If we find it successfully, we stop. Otherwise we search it on T1T_1T1​ and so on. If xxx is accessed ttt queries before (and is not accessed in the latest t−1t - 1t−1 queries if t≥1t \ge 1t≥1), then it is contained in the tree TixT_{i_x}Tix​​, where

ix≐ the smallest non-negative integer, s.t. 2ix−1≥ti_x \doteq \text{ the smallest non-negative integer, } s.t. \ 2^{i_x - 1} \ge t ix​≐ the smallest non-negative integer, s.t. 2ix​−1≥t

(1+log⁡1)+(1+log⁡2)+(1+log⁡4)+(1+log⁡16)+...+(1+log⁡22ix−1)=ix+1+∑i=0ix−12i=ix+1+2ix−1=O(log⁡log⁡t)+O(log⁡t)=O(log⁡t)\begin{aligned} &(1 + \log 1) + (1 + \log 2) + (1 + \log 4) + (1 + \log 16) + ... + (1 + \log 2^{2^{i_x - 1} }) \\ &=i_x + 1 + \sum_{i = 0}^{i_x - 1} 2^{i} \\ &=i_x + 1 + 2^{i_x} - 1 \\ &=O(\log \log t) + O(\log t) \\ &=O(\log t) \end{aligned} ​(1+log1)+(1+log2)+(1+log4)+(1+log16)+...+(1+log22ix​−1)=ix​+1+i=0∑ix​−1​2i=ix​+1+2ix​−1=O(loglogt)+O(logt)=O(logt)​

Finally, we need to update the trees T0T_0T0​, T1T_1T1​, … TixT_{i_x}Tix​​ to reflect the last access of xxx. In particular, we 1) insert xxx into T0T_0T0​, T1T_1T1​, … Tix−1T_{i_x - 1}Tix​−1​; 2) for each i<ixi < i_xi<ix​, we delete the least recently accessed item; 3) update the access time of xxx in TixT_{i_x}Tix​​.

Step 1) takes O(log⁡t)O(\log t)O(logt) time. To implement steps 2) and 3) in O(log⁡t)O(\log t)O(logt) time, we create a min-heap HiH_iHi​ for each tree TiT_iTi​ that keeps the the access time of items in TiT_iTi​. Both fetching least recently accessed item and updating access time can be performed in O(log⁡ni)O(\log n_i)O(logni​) time. Therefore, steps 2) and 3) take O(log⁡t)O(\log t)O(logt) time in total.
