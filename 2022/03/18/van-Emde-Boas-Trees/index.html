van Emde Boas Tree

Posted on 2022-03-18 Edited on 2022-03-26

Problem

Let d∈N+d \in \mathbb{N}^+d∈N+ such that log⁡log⁡d∈N+\log \log d \in \mathbb{N}^+loglogd∈N+.
We want to design a data structure VVV, that maintains nnn elements from the finite domain Zd≐{0,…,d−1}\mathbb{Z}_d \doteq \{0, \ldots, d - 1 \}Zd​≐{0,…,d−1}, with the following properties:

It supports insert, successor operations in O(log⁡log⁡d)O(\log \log d)O(loglogd) time.

Base Case

If d=2d = 2d=2, then the tree consists of a bit array VVV of size two.
Insertion and deletion involves setting the corresponding bit to 111 and 000, respectively.
Finding successor of an element involving scanning the array.
All operations has time complexity O(1)O(1)O(1).

General Cases

Naive Array

We could maintain a bit array VVV of size ddd.
Insertion and deletion can still be performed in O(1)O(1)O(1) time.
However, successor has time complexity O(d)O(d)O(d).

Naive Recursion

We can partition the bit array VVV into d\sqrt{d}d​ clusters, denoted VVV.cluster[0][0][0], …\ldots…, VVV.cluster[d−1][\sqrt{d} - 1][d​−1].
Each cluster consists of d\sqrt{d}d​ consecutive elements, and maintains a summary bit for deciding whether it is empty.
The summary bits are grouped into an array VVV.summary.

Now insertion time is still O(1)O(1)O(1).
Deletion and successor can be performed in O(d)O(\sqrt{d})O(d​) time.

van Emde Boas Tree

van Emde Boas Tree (vEB Tree) is an advanced data structure based on the divide-and-conquer strategy.

Definition

Denote the structure over domain ddd as vEB(d)\text{vEB}(d)vEB(d).
It is defined recursively as follows:

Definition vEB(d)\text{vEB}(d)vEB(d):

  1. If d=2d = 2d=2, then vEB(d)\text{vEB}(d)vEB(d) is just a simple bit array of size 222.
  2. Otherwise, vEB(d)\text{vEB}(d)vEB(d) has the following components:
    • Two elements, max and min, each of size log⁡d\log dlogd bits.
    • Children trees, c0,…,cd−1c_0, \ldots, c_{\sqrt{d} - 1}c0​,…,cd​−1​, each of type vEB(d)\text{vEB} \left(\sqrt{d}\right)vEB(d​).
    • Summary tree, sss, of type vEB(d)\text{vEB} \left(\sqrt{d}\right)vEB(d​).

Hence, if d>2d > 2d>2, then vEB(d)\text{vEB}(d)vEB(d) contains (1+d) \big(1 + \sqrt d\big)\,(1+d​) vEB(d)\text{vEB} \left(\sqrt{d}\right)vEB(d​) subtrees.

Let VVV be an instance of vEB(d)\text{vEB}(d)vEB(d).
If an element x∈Zdx \in \mathbb{Z}_{d}x∈Zd​ is stored in VVV, we write x∈Vx \in Vx∈V.
The components of VVV have the following functionalities.

  • VVV.max and VVV.min maintain the maximum and minimum elements in the tree. These two elements will not be stored in the children trees or the summary tree.
  • For each i∈Zdi \in \mathbb{Z}_{\sqrt{d}}i∈Zd​​, the child tree V.ciV.c_iV.ci​ stores elements x∈Vx \in Vx∈V for which 1) x∉{V.max⁡,V.min⁡}x \notin \{ V.\max, V.\min \}x∈/{V.max,V.min}; 2) ⌊x/d⌋=i\lfloor x / \sqrt{d} \rfloor = i⌊x/d​⌋=i.
  • The summary tree V.sV.sV.s stores the i∈Zdi \in \mathbb{Z}_{\sqrt d}i∈Zd​​ for which V.ciV.c_iV.ci​ is not empty.

Time Analysis

Theorem. Inserting an element xxx into an vEB(d)\text{vEB}(d)vEB(d) instance VVV has time complexity O(log⁡log⁡d)O(\log \log d)O(loglogd).

Proof.
The claim holds trivially if VVV contains zero or one element.
Otherwise, let T(d)T(d)T(d) be the time complexity for inserting xxx into VVV.
We show that

T(d)≤T(d)+O(1). T(d) \le T(\sqrt{d}) + O(1). T(d)≤T(d​)+O(1).

Solving the recursion gives T(d)∈O(log⁡log⁡d)T(d) \in O(\log \log d)T(d)∈O(loglogd).

If V.chV.c_hV.ch​ is non-empty, the only one recursion, i.e., insert(V.cxh,x)(V.c_{x_h}, x)(V.cxh​​,x), is executed.
Therefore, T(d)≤T(d)+O(1)T(d) \le T(\sqrt{d}) + O(1)T(d)≤T(d​)+O(1) holds.

If V.chV.c_hV.ch​ is empty, two recursions, i.e., insert(V.s,x)(V.s, x)(V.s,x), insert(V.cxh,x)(V.c_{x_h}, x)(V.cxh​​,x), are executed.
However, in this case, the second recursion has time complexity O(1)O(1)O(1).
Hence, T(d)≤T(d)+O(1)T(d) \le T(\sqrt{d}) + O(1)T(d)≤T(d​)+O(1) still holds.

□\square□

Theorem. Finding the successor of an element xxx from an vEB(d)\text{vEB}(d)vEB(d) instance VVV has time complexity O(log⁡log⁡d)O(\log \log d)O(loglogd).

Proof.
The successor operation of VVV invokes at most one recursive call, therefore its running time T(d)T(d)T(d) satisfies T(d)≤T(d)+O(1)T(d) \le T(\sqrt{d}) + O(1)T(d)≤T(d​)+O(1), which solves to T(d)∈O(log⁡log⁡d)T(d) \in O(\log \log d)T(d)∈O(loglogd).

□\square□

Space Analysis

Theorem. vEB(d)\text{vEB}(d)vEB(d) can be implemented with O(d)O(d)O(d) space.

Proof.
Denote the space usage of vEB(d)\text{vEB}(d)vEB(d) as S(d)S(d)S(d).
Then we have S(2)=O(1)S(2) = O(1)S(2)=O(1).
For d>2d > 2d>2, a vEB(d)\text{vEB}(d)vEB(d) structure stores

  1. Elements min⁡\minmin and max⁡\maxmax, which takes Θ(1)\Theta(1)Θ(1) words.
  2. Pointers to 1+d1 + \sqrt{d}1+d​ subtrees, which takes Θ(d)\Theta(\sqrt{d})Θ(d​) words.
  3. 1+d1 + \sqrt{d}1+d​ subtrees, each taking space S(d)S(\sqrt{d})S(d​).

Therefore, the total space usage is given by

S(d)=(1+d)⋅S(d)+Θ(d). S(d) = (1 + \sqrt{d}) \cdot S(\sqrt{d}) + \Theta (\sqrt{d}). S(d)=(1+d​)⋅S(d​)+Θ(d​).

To solve the recursion, assume that S(4)=1S(4) = 1S(4)=1 (by proper scaling).
Further, assuming that S(d)≤d−2S(d) \le d - 2S(d)≤d−2.
Then, we can verity that the recursion holds:

S(d)≤(1+d)⋅(2d−2)+d=d−2. S(d) \le (1 + \sqrt{d}) \cdot (2 \sqrt{d} - 2) + \sqrt{d} = d - 2. S(d)≤(1+d​)⋅(2d​−2)+d​=d−2.

□\square□

Theorem. vEB(d)\text{vEB}(d)vEB(d) can be implemented with O(nlog⁡log⁡d)O(n \log \log d)O(nloglogd) space.

Proof.
One improvement to the previous approach is to avoid maintaining all d\sqrt{d}d​ pointers to the children trees c0,…,cdc_0, \ldots, c_{\sqrt{d}}c0​,…,cd​​, and constructing all of them explicitly.

Instead, we construct a child tree only if it is non-empty, and keep a hash table in the vEB(d)\text{vEB}(d)vEB(d) structure, which stores the pointers to these non-empty children trees.

Inserting an element has time complexity O(log⁡log⁡d)O(\log \log d)O(loglogd), therefore it can creates at most O(log⁡log⁡d)O(\log \log d)O(loglogd) pointers, as well as subtrees.
Since when a subtree is created, it takes space O(1)O(1)O(1), therefore the total space usage is O(nlog⁡log⁡d)O(n \log \log d)O(nloglogd).

This bound is tight.
Suppose we want to insert an element xxx into a vEB(d)\text{vEB}(d)vEB(d) structure.
If cxhc_{x_h}cxh​​ is empty, then we invoke the recursion insert(s,xh)(s, x_h)(s,xh​) to insert xhx_hxh​ into the summary.
However, in this case, we need to crete the subtree for cxhc_{x_h}cxh​​ (which takes O(1)O(1)O(1) space), insert the pointer for cxhc_{x_h}cxh​​ into the hash table (which also takes O(1)O(1)O(1) space).
This can happen for each recursion, therefore the total space overhead is O(log⁡log⁡d)O(\log \log d)O(loglogd) for a single insertion.

□\square□
