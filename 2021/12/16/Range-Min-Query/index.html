Range Min Query

Posted on 2021-12-16 Edited on 2022-01-08

Problem

Definition (Range Min Query). Let a1,…,an∈Ra_1, \ldots, a_n \in \mathbb{R}a1​,…,an​∈R.
Given ℓ,r∈[n]\ell, r \in [n]ℓ,r∈[n], ℓ≤r\ell \le rℓ≤r, return min⁡{aℓ,…,ar}\min \{ a_\ell, \ldots, a_r \}min{aℓ​,…,ar​}.

Basic Solutions

A data structure for the Range Min Query (RMQ) has complexity <p(n),q(n)>\left< p(n), q(n) \right>⟨p(n),q(n)⟩ if it has pre-processing and query-answering time p(n)p(n)p(n) and q(n)q(n)q(n) on input instance of size nnn, respectively.

A1\mathcal{A}_1A1​ Precompute-None

Scan aℓ,…,ara_\ell, \ldots, a_raℓ​,…,ar​ to find the min.

  • This is an <0,O(n)>\left< 0, O(n) \right>⟨0,O(n)⟩ solution.

A2\mathcal{A}_2A2​ Complete Binary Tree

Construct a complete binary tree with a1,…,ana_1, \ldots, a_na1​,…,an​ being the leaf nodes.
Each internal node of the tree is augmented with the minimum value of its subtree.

  • This is an <O(n),O(log⁡n)>\left< O(n), O(\log n) \right>⟨O(n),O(logn)⟩ solution.

A3\mathcal{A}_3A3​ Precompute-All

Create a n×nn \times nn×n table MMM.
For each ℓ,r∈[n]\ell, r \in [n]ℓ,r∈[n], ℓ≤r\ell \le rℓ≤r, the (ℓ,r)(\ell, r)(ℓ,r)-th entry records min⁡{aℓ,…,ar}\min \{ a_\ell, \ldots, a_r \}min{aℓ​,…,ar​}.

  • This is an <O(n2),O(1)>\left< O(n^2), O(1) \right>⟨O(n2),O(1)⟩ solution.

  • p(n)=O(n2)p(n) = O(n^2)p(n)=O(n2).

    • Dynamic Programming.
    • ∀ℓ,r∈[n],ℓ≤r\forall \ell, r \in [n], \ell \le r∀ℓ,r∈[n],ℓ≤r,

      M[ℓ,r]={aℓℓ=r,min⁡{M[ℓ,r−1],M[ℓ+1,r]},ℓ<r. M[\ell, r] = \begin{cases} a_\ell & \ell = r, \\ \min \{ M[\ell, r - 1], M[\ell + 1, r] \}, & \ell < r. \end{cases} M[ℓ,r]={aℓ​min{M[ℓ,r−1],M[ℓ+1,r]},​ℓ=r,ℓ<r.​

  • q(n)=O(1)q(n) = O(1)q(n)=O(1).

Approach Two

Can we achieve <O(n),O(1)>\left< O \big( n \big), O \big( 1 \big) \right>⟨O(n),O(1)⟩ under approach one?

The key is to observe that the following are two different problems.

For the divide-and-conquer framework to work to achieve complexity <O(n),O(1)>\left< O(n), O(1) \right>⟨O(n),O(1)⟩, it suffices to find an algorithm for the second problem.

Block Type

W.L.O.G., consider the first block B1={a1,…,as}B_1 = \{ a_{ 1 }, \ldots, a_{ s} \}B1​={a1​,…,as​} be a block of sss elements.
We can construct a binary tree over the elements, such that it satisfies

  1. Min Heap Property: each node of the tree is the smallest one in the subtree rooted at this node.
  2. The in-order traversal of the tree gives the array B1B_1B1​.

Such a binary tree is called a Cartesian Tree and the type of the block.

On the other hand, suppose we are given a Cartesian Tree tree constructed on B1B_1B1​, and suppose that the value of each node of the tree is hidden.
Hence, we only know the topology or shape of the binary tree.
For all 1≤ℓ≤r≤s1 \le \ell \le r \le s1≤ℓ≤r≤s, we can still return the index of the minimum element among aℓ,…,asa_\ell, \ldots, a_saℓ​,…,as​, purely based on the topology of the binary tree.

In this sense, the answers to the RMQ on B1B_1B1​ is entirely determined by its corresponding Cartesian Tree.

  1. Enumerate all possible Cartesian Trees.

  2. For each Cartesian Tree, invoke the sparse table algorithm A3\mathcal{A}_3A3​ algorithm on this tree, so that all possible range min query for this tree can be answer in O(1)O(1)O(1) time.

    To answer a RMQ on BiB_iBi​, access the pre-computed result on its corresponding Cartesian Trees.

Determining the Block Type

In this section, we show how to construct the corresponding Cartesian Tree given the block B1B_1B1​, and to create a unique identifier ∈O(n)\in O(n)∈O(n) for the Cartesian Tree.

We illustrate the idea with an example.
In this example, B1=(2,4,3,1)B_1 = (2, 4, 3, 1)B1​=(2,4,3,1).
We will construct the Cartesian Tree, by including the elements from B1B_1B1​ one by one.
Further, we maintain a stack and a stack operation sequence, both of which are initially empty.
The stack keeps elements from the right spine of the tree, and the stack operation sequence records the “in stack” and “out stack” operations as we include more elements from B1B_1B1​ into the tree.

We first include the element 222 into the tree.
Since the tree consists of only one node, it is the root of the tree.
It is also the only node in the right spine of the tree, so we put 222 into the stack, and append 111 to the stack operation sequence, to record the in-stack operation.

Next we include the element 444 into the tree.
Since the tree satisfies min-heap property, 444 must be the child of 222.
Since the in-order traversal of the tree gives the array 2,42, 42,4, the node 444 must be the right child of 222.
We put 444 into the stack, and append 111 to the stack operation sequence, to record the in-stack operation.

We want to include the element 333 into the existing tree.
Since after inclusion, the in-order traversal of the tree gives the array 2,4,32, 4, 32,4,3, the node 333 must be in the right spine of the tree.
Indeed, this is a general principle for adding a new element into the tree.

Lemma (In the Right Spine). For each j∈[s]j \in [s]j∈[s], let TjT_jTj​ be the Cartesian tree constructed on a1,…,aja_1, \ldots, a_ja1​,…,aj​.
Then aja_jaj​ be must in the right spine of TjT_jTj​.

Proof.

  1. If aja_jaj​ has right-child in TjT_jTj​, then it is not the last element visited by the in-order traversal of TjT_jTj​.
  2. If aja_jaj​ has a parent node in TjT_jTj​ and is the left-child of its parent, then it is not the last element visited by the in-order traversal of TjT_jTj​.

□\square□

In this case, the right spine consists of 222 and 444.
We compare 333 with 444.
As it is smaller than 444, we pop 444 out of the stack.
We compare 333 with 222.
As it is larger than 222, we put 333 into the stack.
We include 333 as the right-child of 222, and set 444 as the left child of 333.
We see that after these operations, the stack still contains the nodes in the right spine of the tree.
Further, the min-heap property of the tree is maintained.
We append 010101 to the stack operation sequence, to record the out-stack and in-stack operations performed.

We now formally states the rules for adding an element to the right spine.

It is easy to verify that after the construction

  1. aj+1a_{j + 1}aj+1​ is in the right spine of the tree.
  2. The min-heap property is maintained.

We want to include the element 111 into the existing tree.
Since after inclusion, the in-order traversal of the tree gives the array 2,4,3,12, 4, 3, 12,4,3,1, the node 111 must be in the right spine of the tree.
In this case, the right spine consists of 222 and 333.
As 111 is smaller than 222, we pop 333 and 222 out of the stack, put 111 into the stack, and set 222 as the left child of 111.
We append 001001001 to the stack operation sequence, to record the stack operations performed.

Finally, we pop 111 out of the stack, and append 000 to the stack operation sequence, to record the stack operation performed.
We perform this operation, so that whatever sequence of a1,…,asa_1, \ldots, a_sa1​,…,as​, the stack operation sequence corresponding to the Cartesian Tree construction procedure stated as above has length 2s2s2s.

The construction also reveal an interesting fact: each Cartesian Tree with sss node corresponding a 000-111 sequence with length 2s2s2s, such that each prefix of the sequence contains no less 111’s than 000’s.
