Sparse Recovery

Posted on 2021-03-31 Edited on 2021-04-08

Model

The focus of this article is recovering a sparse vector x∈Rnx \in \mathbb{R}^nx∈Rn in the streaming setting. Initially, x=0x = 0x=0. Then we see a sequence of updates (i,Δ)(i, \Delta)(i,Δ) each causing the change xi←xi+Δx_i \leftarrow x_i + \Deltaxi​←xi​+Δ. There are three popular studied models

  1. insertion only: Δ=1\Delta = 1Δ=1 for each update.
  2. strict turnstile: Δ\DeltaΔ can be negative, but it is guaranteed that x≥0x \ge 0x≥0 all the time.
  3. general turnstile: Δ\DeltaΔ can be negative, and entries of xxx can be negative.

Let supp(x)⊂[n]\text{supp}(x) \subset [n]supp(x)⊂[n] be the set of nonzero indices of xxx. The vector xxx is called kkk sparse if supp(x)≤k\text{supp}(x) \le ksupp(x)≤k.

Goal. Given an integer kkk, design a data structure such that, if xxx is kkk sparse, the structure returns xxx exactly; otherwise, the return can be arbitrary.

As we can’t actually store real numbers, we use the following assumption:

The updates are integers and the entries of xxx are bounded by ±M\pm M±M.

With this restriction, it suffices to solve this problem in a finite field Fp\mathbb{F}_pFp​ where p≥max⁡{n,M}p \ge \max \{ n, M \}p≥max{n,M}. As opposed to the naive solution that keeps all entries of xxx and uses O(nlog⁡M)O(n \log M)O(nlogM) bits of memory, we show that the memory can be reduced to O(klog⁡(nM))O( k \log (nM) )O(klog(nM)) bits.

Algorithm

The problem is easy to solve in the insertion only model. We can use a hash set that keeps the first kkk occurrence of the indices. If the we detect the k+1k + 1k+1 new index, we can just report error.

The scenario is a little more complicated for the other two models. E.g., let n=4n = 4n=4 and k=1k = 1k=1. Consider the following stream

(1,1),(2,1),(3,−1),(4,1),(1,−1),(2,−1),(3,1). (1, 1), (2, 1), (3, -1), (4, 1), (1, -1), (2, -1), (3, 1). (1,1),(2,1),(3,−1),(4,1),(1,−1),(2,−1),(3,1).

The vector xxx is finally 111-sparse and the only non-zero index is revealed at the end of the stream.

Given y=Enc(x)=Hxy = \mathbb{Enc}(x) = H xy=Enc(x)=Hx, how can we recover xxx? We can loop over every subset S⊂[n]S \subset [n ]S⊂[n] of size 2k2k2k and compute x′=HS−1yx' = H_S^{-1} yx′=HS−1​y. This is because we don’t store HHH. For the finite field Fp\mathbb{F}_pFp​, there exists at least one primitive element whose powers give all nonzero elements.
