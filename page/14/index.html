Interpretation of Weights

The weights have another interpretation.
Suppose that we associate each element inserted into A0A_0A0​ with a set, which contains only the element itself.
We will maintain these sets, such that

For each 0≤i≤L0 \le i \le L0≤i≤L, and each v∈Aiv \in A_iv∈Ai​, the set associated with vvv has size wiw_iwi​, and vvv is the only element in the set that is kept in the arrays. Further, the sets associated with different elements in the arrays are disjoint.

Clearly this holds when an element is inserted into A0A_0A0​.
Now, suppose we are going to perform a compaction at an array AiA_iAi​, for some 0≤i<L0 \le i < L0≤i<L.
We delete half of the elements in AiA_iAi​, and keep the rest of the elements.
We can pair up the elements deleted and the ones kept one by one, and merge their associated sets.
For each merged set, it contains wi+1w_{i + 1}wi+1​ elements, and only one element in the merge set is kept in the arrays.
Hence, we can associate it with the only element kept.

The above discussion gives a new interpretation of the weights.

If an element is associated with weight wiw_iwi​, it represents wiw_iwi​ unique elements in the original stream.

Basic KLL Sketch

In this section, we reduce the space usage from O(ϵ−1⋅log⁡2(ϵn))O \left( \epsilon^{-1} \cdot \log^2 (\epsilon n) \right)O(ϵ−1⋅log2(ϵn)) to

The algorithm is a randomized one and has failure probability δ\deltaδ.

The algorithm is similar to the MRL sketch, but
sets k=2/ϵ⋅ln⁡2/δk = 2 / \epsilon \cdot \sqrt{ \ln 2 / \delta }k=2/ϵ⋅ln2/δ​ and L=⌈log⁡(n/k)⌉L = \lceil \log (n / k) \rceilL=⌈log(n/k)⌉ to achieve the space usage stated above.
Besides, the compaction step is different.

Random Parity

Instead of always moving the odd-indexed elements to the next array, it chooses uniformly at random either the odd-indexed elements, or even-indexed elements.
In particular, it replaces the line 4 of the COMPRESS algorithm with the following lines:

  1. Flip an unbiased coin;
  2. If the coin turns out head:
  3. \qquad Move Ai[1],Ai[3],…,Ai[k−1]A_i[1], A_i[3], \ldots, A_i[k - 1]Ai​[1],Ai​[3],…,Ai​[k−1] into Ai+1A_{i + 1}Ai+1​;
  4. Else:
  5. \qquad Move Ai[0],Ai[2],…,Ai[k−2]A_i[0], A_i[2], \ldots, A_i[k - 2]Ai​[0],Ai​[2],…,Ai​[k−2] into Ai+1A_{i + 1}Ai+1​;
