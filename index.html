Turing Machine Decidability

Posted on 2025-04-09 Edited on 2025-04-10

We are interested in whether a problem is algorithmically unsolvable. Knowing when a problem is algorithmically unsolvable is useful because then you realize that the problem must be simplified or altered before you can find an algorithmic solution.

Decidable Languages

Regular Languages

Theorem. Let
\(A_{DFA} = \{ \langle B, w \rangle \mid B \text{ is a DFA that accepts input string } w \}\),
\(A_{NFA} = \{ \langle B, w \rangle \mid B \text{ is an NFA that accepts input string } w \}\), and
\(A_{REX} = \{ \langle R, w \rangle \mid R \text{ is a regular expression that generates string } w \}\).
Then \(A_{DFA}\), \(A_{NFA}\), and \(A_{REX}\) are decidable languages.

Proof Hint. The proof involves constructing Turing machines that simulate the behavior of a DFA, NFA, or a regular expression on the given input—similar to how such simulations are implemented in standard programming languages.

Theorem. Let \(E_{DFA} = \{ \langle A \rangle \mid A \text{ is a DFA and } L(A) = \emptyset \}\).
Then \(E_{DFA}\) is a decidable language.

Proof Hint. Design a Turing machine that checks whether there exists a path from the start state to any accept state.

Theorem. Let \(EQ_{DFA} = \{ \langle A, B \rangle \mid A \text{ and } B \text{ are DFAs and } L(A) = L(B) \}\).
Then \(EQ_{DFA}\) is a decidable language.

Proof Hint. Test whether both \(L(A) \cap \overline{L(B)}\) and \(L(B) \cap \overline{L(A)}\) are empty (since regular languages are closed under complement and intersection, both resulting languages are regular). If both are empty, then \(L(A) = L(B)\).

Context-Free Languages

Theorem. Let \(A_{CFG} = \{ \langle G, w \rangle \mid G \text{ is a CFG that generates string } w \}\).
Then \(A_{CFG}\) is a decidable language.

Theorem. Let \(E_{CFG} = \{ \langle G \rangle \mid G \text{ is a CFG and } L(G) = \emptyset \}\).
Then \(E_{CFG}\) is a decidable language.

Theorem. Every context-free language is decidable.

Proof Hint. * Simulate a parsing algorithm for CFGs using a Turing machine.
* One possible parsing algorithm: Convert the CFG to Chomsky Normal Form (CNF) using the following steps:
- Add a new start variable \(S_0\) and the rule \(S_0 \rightarrow S\)
- Eliminate all \(\epsilon\)-productions of the form \(A \rightarrow \epsilon\)
- Eliminate all unit productions (rules of the form \(A \rightarrow B\))
- Convert all remaining rules into CNF

If $G$ is in CNF, any derivation of a string $w \in L(G)$ has exactly $2|w| - 1$ steps, which bounds the parsing time.

Undecidable Language

Theorem. Let \(A_{TM} = \{ \langle M, w \rangle \mid M \text{ is a TM and } M \text{ accepts } w \}\).
Then \(A_{TM}\) is undecidable.

The Turing machine \(U\) is interesting in its own right.

Universal Turing Machine
$U = $ “On input \(\langle M, w \rangle\), where \(M\) is a TM and \(w\) is a string:

  1. Simulate \(M\) on input \(w\).
  2. If \(M\) ever enters its accept state, accept; if \(M\) ever enters its reject state, reject.”

Theorem. Some languages are not Turing-recognizable.

Proof Hint.
Assume that a TM \(H\) decides \(A_{TM}\). Use \(H\) to build a TM \(D\) that takes input \(\langle B \rangle\) and accepts it exactly when \(B\) does not accept \(\langle B \rangle\). Finally, run \(D\) on itself. The contradiction arises as follows:

  • \(H\) accepts \(\langle M, w \rangle\) exactly when \(M\) accepts \(w\).
  • \(D\) rejects \(\langle B \rangle\) exactly when \(B\) accepts \(\langle B \rangle\).
  • \(D\) rejects \(\langle D \rangle\) exactly when \(D\) accepts \(\langle D \rangle\) — a contradiction.

We say that a language is co-Turing-recognizable if it is the complement of a Turing-recognizable language.

Theorem. A language is decidable iff it is both Turing-recognizable and co-Turing-recognizable.

Corollary. \(A_{TM}\) is not co-Turing-recognizable.
Proof. We know \(A_{TM}\) is Turing-recognizable.
If \(A_{TM}\) were also co-Turing-recognizable, then \(A_{TM}\) would be decidable (by the theorem above).
But we know \(A_{TM}\) is undecidable, so \(A_{TM}\) cannot be co-Turing-recognizable.

Turing Machine

Posted on 2025-04-07 Edited on 2025-04-08

Turing Machine

Recall that when finite automata perform computations, they read input symbols one by one. This process can be modeled by assuming the existence of a tape, where the input symbols are arranged from left to right, and a read-head moves from left to right to scan the symbols.

Turing machines extend finite automata in the following ways:

  1. A Turing machine can both read from and write to the tape.
  2. The read–write head can move both left and right.
  3. The tape is infinite in length.

These extensions provide the Turing machine with access to infinite memory, in a way that is more flexible than pushdown automata.
Additionally, we assume:

  1. The special states for accepting and rejecting take effect immediately upon being entered.

Formally, the model is defined as thus.

Definition. A Turing machine is a 7-tuple \((Q, \Sigma, \Gamma, \delta, q_0, q_{accept}, q_{reject})\), where \(Q\), \(\Sigma\), and \(\Gamma\) are all finite sets, and:
1. \(Q\) is the set of states,
2. \(\Sigma\) is the input alphabet, which does not contain the blank symbol,
3. \(\Gamma\) is the tape alphabet, where \(\sqcup \in \Gamma\) and \(\Sigma \subseteq \Gamma\),
4. \(\delta : Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}\) is the transition function,
5. \(q_0 \in Q\) is the start state,
6. \(q_{accept} \in Q\) is the accept state, and
7. \(q_{reject} \in Q\) is the reject state, with \(q_{reject} \neq q_{accept}\).

Computation

A Turing machine \(M = (Q, \Sigma, \Gamma, \delta, q_0, q_{accept}, q_{reject})\) computes as follows:

  • Initialization
    • The input \(w = w_1 w_2 \ldots w_n \in \Sigma^*\) is written on the leftmost \(n\) squares of the tape, with blank symbols (\(\sqcup\)) filling the rest.
      Since \(\Sigma\) does not contain the blank symbol, the first blank indicates the end of the input.
    • The head starts on the leftmost square of the tape.
  • Computation
    • The computation proceeds according to the transition function \(\delta\).
    • If \(M\) ever attempts to move its head left off the left-hand end of the tape, the head stays in place.
    • The computation continues until \(M\) enters either the accept or reject state. If neither occurs, \(M\) continues running indefinitely.

The intermediate computation state of a Turing machine can be represented by a configuration, which captures the current state, the contents of the tape, and the position of the head.

Configuration. A configuration \(u q v\) of the Turing machine consists of a state \(q \in Q\) and two strings \(u, v \in \Gamma^*\), indicating that the current state is \(q\), the current tape contents is \(uv\), and the head is scanning the first symbol of \(v\).

The computation of a Turing machine can thus be viewed as a sequence of configurations, where each step updates the configuration according to the transition function.

Language

Turing-Recognizable Language. The set of strings that \(M\) accepts is called the language of \(M\), or the language recognized by \(M\), and is denoted by \(L(M)\).
A language is Turing-recognizable if some Turing machine recognizes it.

A Turing machine \(M\) can fail to accept an input either by entering the \(q_{reject}\) state and explicitly rejecting, or by entering an infinite loop.
In practice, distinguishing a machine that is looping from one that is merely taking a long time can be challenging.

Turing-Decidable Language. Deciders are Turing machines that halt on all inputs — i.e., they never loop.
A language is Turing-decidable (or simply decidable) if some decider recognizes it.

Theorem. The collection of Turing-decidable languages is closed under the operations of:
- union
- concatenation
- star
- complementation
- intersection

Theorem. The collection of Turing-recognizable languages is closed under the operations of:
- union
- concatenation
- star
- intersection
- homomorphism

Variants of Turing Machine

The standard Turing machine model is equivalent to the following models:

Multitape Turing machine: A Turing machine with multiple tapes, each having its own head for reading and writing. The input appears on the first tape, and the others start blank. The transition function is adjusted to allow reading, writing, and moving the heads on all tapes simultaneously. Formally, \(\delta : Q \times \Gamma^k \rightarrow Q \times \Gamma^k \times \{L, R, S\}^k\), where \(k\) is the number of tapes.

Nondeterministic Turing machine: A Turing machine where, at any computation step, multiple transitions may be possible. The transition function is \(\delta : Q \times \Gamma \rightarrow P(Q \times \Gamma \times \{L, R\})\). The computation tree branches based on different possibilities, and if any branch reaches the accept state, the machine accepts the input.

Enumerator: A Turing machine with an attached printer, which outputs strings. The machine sends strings to the printer whenever it wants to add a new string to the output list. The language enumerated by \(E\) is the collection of all the strings that it eventually prints out.
