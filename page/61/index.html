Random Sampling and Cuts II

Posted on 2015-12-01 Edited on 2018-04-07

Sampling Theorem

In 1994, Karger gave the following result:

[Sampling Karger] [Using randomized sparsification to approximate minimum cuts] Given a graph G and a collection of random variables \(X_e (X_e \le M)\) associated with all edges e, with probability \(1 - O(n^{-d})\) that every cut in the instance \(G(X_e)\) generated by putting weight \(X_e\) on each e has value with \((1 - \epsilon, 1 + \epsilon)\) times its expected value, where \[ \epsilon = \sqrt{2(d + 2)M\ln n \over c'} \] and c' is G's minimum expected value.

As an application, if we fix \(\epsilon\) and let \(p = \Theta({\ln n \over \epsilon^2 c})\), where c is the minimum cut of G, then G' approximate all its original graph cuts by a factor p with high probability.

The limitation is that either \(\epsilon\) or p depends on c. When c is little, we could not reduce many edges.

Improvement: In 1996, Benczur and Karger improved previous sampling method such that the dependence on c is removed.

New Result:

Given a graph G and a parameter \(\epsilon\), there is a new graph G' such that 1. G' has \(O({n \ln n \over \epsilon^2})\) edges. 2. every cut in G' has value \((1 \pm \epsilon)\) times its original cut value.

The intuition is that in the dense parts of the graph, each edge can be sample with smaller probability in order to preserve cut value while edge should be sample with higher probability where the graph is sparse.

Before moving on we first formalize the notion of a dense region. We characterize a dense region by Strong Connectivity.

k-Connected

A graph G is k connected if its minimum cut value is no less than k.

Vertex-Induced Subgraph

\(G'=<V',E'>\) is a vertex induced subgraph of \(G =<V,E>\) if \(V'\) is a subset of \(V\) and \(E'\) is all edges of \(E\) whose endpoints are both in \(V'\).

k-Strong Component

A k-Strong Component is a maximal k-connected vertex-induced subgraph.

Strong Connectivity

The strong connectivity \(c_e\) of an edge e is the maximum value of k such that a k-Strong component contains e.

Generating the graph

Here is how new sampling technique operates

Given a parameter \(\epsilon\), let \(\rho = {16(d + 2)M \ln n \over \epsilon^2}\), we assign to each edge a random weight \(X_e\), which takes value \(1/p\) with probability \(p\) and \(0\) with probability \((1 - p)\), where \(p = \min \{ \rho / c_e, 1 \}\). When \(X_e = 0\), we eliminate e from the new graph.

Immediately follows from the definition of \(X_e\) we see that \(E[X_e] = 1\).

To check the sampling technique works, we need to verify two things: 1. The number of edge generated is \(O(n \ln n / \epsilon^2)\) 2. Each cut does not deviate much from its original value.

Number of edges

Theorem 1 > If an undirected connected graph has k(n - 1) edges, then it must has a k strong component.

Proof: We prove it by induction. When k = 1, then the graph itself is a k-strong component. When n = 2, the theorem clearly holds.

Now we consider the general case. Suppose that the graph does not contain a k-strong component. Then there exist a cut \(C = <A, B>\) in the graph such that \(cut(A,B) < k\). Now both A and B have less than n nodes. By induction, A has less then k(|A| - 1) edges inside and B has less then k(|B| - 1) edges inside. The total number of edges is less than k(|A| - 1) + k + k(|B| - 1) = k(|A|+|B| - 1) = k(n - 1). A contradiction.

Theorem 2 > \(\Sigma {1 \over c_e} < 2(n -1)\)

Proof: Suppose that \(\Sigma {1 \over c_e} \ge 2(n -1)\), then by theorem 1, there is a 2-strong component. Let e* be the edge with minimum \(c_{e^*}\) in that component. Then there is a cut C in the 2-strong component containing e* such that \(cut(C) \le c_{e^*}\). Now

\[ {\underset{e \in C}{\Sigma }} c_e^{-1} \le c_{e^*}^{-1} c_{e^*} = 1 < 2 \]

A contradiction.

Error analysis

To analyze the errors, we denote \(F_i\) the set of edges whose strong connectivity are between \([2^i, 2^{i+1}]\) for \(i \ge 0\). Let \(G_i = <V, F_i \cup F_{i + 1} \cup F_{i + 2}...>\).

In \(G_i\), each edge is sampled in the following way \[ X_e = \begin{cases} 1, if \ e \notin F_i \\ \begin{Bmatrix} {1 \over p} with \ probability \ p, \\ 0 \ with \ probability \ 1 - p \end{Bmatrix} \ if \ e \in F_i \end{cases} \]

Then, by sampling theorem, the sampled graph \(G_i'\) is within \((1 \pm \epsilon)G_i\).

So \[ \begin{aligned} G' & = \Sigma_{i = 0}^{\log m} F_i' \\ & = \Sigma G_i' - \Sigma G_{i + 1} \\ & = \Sigma (1 \pm \epsilon)G_i - \Sigma G_{i + 1} \\ & = (1 \pm \epsilon)G_0 \pm \epsilon\Sigma G_{i + 1} \\ & \in G \pm \epsilon \log m \ G \end{aligned} \]

But there is a log m term in the final result. We can eliminate the log m term with more sophisticated techniques.

The random variable \(X_e\) assigned to e in \(G_i\) is revised as follows: \[ X_e = \begin{cases} 2^{i - j}, if \ e \notin F_i \ and \ e \in F_j \\ \begin{Bmatrix} {1 \over p} with \ probability \ p, \\ 0 \ with \ probability \ 1 - p \end{Bmatrix} \ if \ e \in F_i \end{cases} \]

It can be shown that every cut in \(G_i\) is at least \(2^i\) heavy.

So \[ \begin{aligned} G' & = \Sigma_{i = 0}^{\log m} F_i' \\ & = \Sigma G_i' - \Sigma_{i = 0}^{\log m} \Sigma_{j = i + 1}^{\log m} 2^{i - j} F_j \\ & = \Sigma (1 \pm \epsilon)G_i - \Sigma_{i = 0}^{\log m} \Sigma_{j = i + 1}^{\log m} 2^{i - j} F_j \\ & = \Sigma F_i + (1 \pm \epsilon)\Sigma_{i = 0}^{\log m} \Sigma_{j = i + 1}^{\log m} 2^{i - j} F_j - \Sigma_{i = 0}^{\log m} \Sigma_{j = i + 1}^{\log m} 2^{i - j} F_j \\ & = G \pm 2 \epsilon \ \Sigma_{i = 0}^{\log m} \Sigma_{j = i + 1}^{\log m} 2^{i - j} F_j \\ & = G\pm 2 \epsilon \ G \end{aligned} \]

K-Connected Spanning Subgraph

Posted on 2015-11-23 Edited on 2015-12-01

K-Connected Spanning Subgraph

Background

Consider that you are given a transportation network, which is a graph. Now you are asked to find out a sub-graph that the transportation capacity between any two sites in the sub-graph is above some threshold k.

This sub-graph could be viewed as kind of a backbone. Its importance is as follows: even if some lines in the transportation network break down, so long as the lines of this sub-graph operates, we can guarantee that the network is able to transport k capacity cargo between any two sites.

If we examine this problem deeply, we may ask the following questions: 1. Does such a sub-graph exist? 2. If such a sub-graph exist, how to find one with the minimum number of edges?

Solution

Existence could be answered right now. If the minimum cut of the graph is no less than k, then we could find such a sub-graph. A trivial case is the graph itself. But the second one is difficult. It is believed that this is a NP hard problem.

Approximation algorithms are proposed. One of them was given by Hiroshi Nagamochi and Toshihide ibaraki [A Linear-Time Algorithm for Finding a Sparse k-Connected Spanning Subgraph of a k-Connected Graph] in 1992.

We firstly formalize the notions. To simplify the problem, we focus on graphs that are undirected and consist of no multiple edges and self loops.

k-Connected Graph: A graph is k-Connected if the minimum cut of the graph is no less than k.

Nagamoci-Ibaraki Algorithm Given a graph \(G=<V,E>\), if \(F_i = <V, E_i>\) is the maximal spanning forest of \(G = <V, E - E_1 - E_2 - .. - E_{i - 1}>\), the \(G_i = <V, E_1 \cup E_2 \cup ... \cup E_{i - 1}>\) is a k-Connected Sub-graph with \(k|V| - k(k + 1)\) edges if \(k \le \lambda(G)\), where \(\lambda(G)\) is the minimum cut of G.

Here only a scratch of proof is given. Consider a particular cut C in G, whose value \(\lambda(C)\) is no less than \(\lambda(G)\). For any \(G_i\) with \(i <= \lambda(G)\), it must consist of at least one edge of C. Otherwise we could add an arbitrary edge in\(C-E_1-E_2-...-E_{i - 1}\) to \(F_i\) such that \(F_i\) is still a forest, contradicting \(F_i\) is maximal.

Finding a maximal forest in \(G=<V,E>\) requires \(O(|V| + |E|)\) time. So \(O(k(|V| + |E|))\) is required for k-Connected Graph. Nagamochi and Ibaraki did it in \(O(|V| + |E|)\) time. They scaned all nodes and edges once and classify edges into different \(E_i\), here \(i \in [1...|E|]\).

Pseudo Code:

  1. h(v) = 0, \(\forall v \in V\)
  2. $E_i = , i $
  3. while \(\exists u\) that has not been scanned
    • find u with maximum h(u)
    • marked u as scanned
    • for all \((u,v) \in E\) that has not been scanned
      • marked (u,v) as scanned.
      • if h(u) == h(v) then
        • ++h(u)
      • end if
      • ++h(v)
      • \(E_{h(v)} = E_{h(v)} \cup {(u,v)}\)
    • end for

end while

Analysis

To see that the algorithm indeed works, we have to check two things:

  1. \(F_i=<V, E_i>\) is a forest, i.e., \(F_i\) does not contain a cycle.
  2. \(F_i=<V, E_i>\) is maximal.

\(F_i\) doesn't contain a cycle: suppose that when u is being scanned and (u,v) is added to \(E_i\), a cycle is created. Then some edge incident to v belongs to \(F_i\). Thus h(v) >= i before (u, v) is added to \(E_i\). But (u,v) is added to \(E_{h(v)}\) after \('++h(v)'\). So \((u,v) \notin E_i\). A contradiction.

\(F_i\) is maximal. If \(F_i\) is not maximal, there is some edge \((u,v) \in E_j\) for some \(j > i\) such that \(<V, E_i \cup \{(u,v)\}>\) is a forest. Denote the tree containing u in \(F_i\) by \(T_u\) and the one containing v in \(F_i\) by \(T_v\). Further, denote the order of nodes in \(T_u\) and \(T_v\) being scanned by \([x_1, x_2,..., u,...]\) \([y_1,y_2,...v,...]\) respectively. Without losing generality, suppose \(x_1\) is scanned before \(y_1\). When \(x_1\) is scanned, we must have \(h(x_1) == i - 1\) and \(h(y_1) \le i - 1\). After \(x_1\) is scanned, \(h(x_1) == h(x_2) ==i\). For any other node \(z \ne {x_1}\) such that \(h(z) == i\), we guarantee that \(h(z) == i -1\) before \(x_1\) is scanned and \(z \in [x_2, x_3,...]\), otherwise z is scanned before \(x_1\). Now \(h(x_2) =i > h(y_1)\), it's scanned before \(y_1\). Following this way, u is scanned before \(y_1\). When u is scanned, \(h(v) \le i - 1\). So (u, v) is add to some \(E_j\) for \(j \le i\). A contradiction.

The number of edges in \(G_i\)

Suppose the order of nodes being scanned in V is \(v_1, v_2, ..\). The value of \(h(v_i)\) could not exceed i, since there are at most one edge \((v_j, v_i)\) incident to \(v_i\) for some j less than i. So \(v_i\) does not appear in forest \(F_k\) for \(k > i\). Then \(F_k\) contains at most |V| - 1 - (k - 1) = |V| - k edges. Thus the number of edges in \(G_i\) is at most \[ \Sigma_{i = 1}^{k} (|V| - i) = k|V| - k(k + 1) /2 \]

Connectivity of a Graph

Posted on 2015-10-16 Edited on 2021-09-26

Suppose the edge connectivity of a graph \(G\) is \(c\). In the previous article we know that the number of cuts with weight up to \(kc\) is bounded by \(O((2n)^{2k})\), where \(k\) is a real number and \(n\) is the number of vertices in \(G\).

Let \(f(x)\) to be the number of cut with value \(xc\) and define \(F(k) = \Sigma_{x \le k} f(x)\). From the discussion above, \(F(k) \le \lambda (2n)^{2k}\) for some constant \(\lambda\).

If we remove each edge with probability \(p\), then the graph becomes disconnected with probability at most \[ \Pr[\text{Disconnected}] = \Sigma_{x} f(x)p^{xc}. \]

To maximize \(\Pr[\text{Disconnected}]\), we take \(F(k) = \lambda (2n)^{2k}\) for every \(k\).

By the definition of \(F(k)\), \(f(k) = \lambda ((2n)^{2k} - (2n)^{2(k-1)})\).

Then, for \(k \ge 2\), \[ \begin{aligned} f(k) p^{kc} &= \lambda ((2n)^{2k} - (2n)^{2(k-1)}) p^{kc} \\ &= \lambda \left( \int_{k -1}^{k} {\partial (2n)^{2x} \over \partial x} dx \right) \cdot p^{kc} \\ &\le \lambda \int_{k -1}^{k} {\partial(2n)^{2x} \over \partial x} p^{xc}dx \end{aligned} \]

Sum up this inequality over \(k\), \[ \begin{aligned} \Pr[\text{Disconnected}] &\le f(1)p^c + \lambda \int_1^\infty {\partial (2n)^{2x} \over \partial x} p^{xc} dx \\ & = \lambda (2n)^2 p^c - \lambda {2\ln 2n \over 2 \ln 2n + c \ln p} (2n)^2 p^c \end{aligned} \]

If \(p = (2n)^{-{a \over c} }\) for some parameter \(a\), then \(\Pr[\text{Disconnected}] = O(n^2 p^c / (a - 2)) = O(n^{2 - a} / (a -2))\). When \(a = 3\), the graph is disconnected with probability \(O(1/n)\). (p.s: for large \(c\), \(p \approx 1 - {a \ln 2n \over c}\))
