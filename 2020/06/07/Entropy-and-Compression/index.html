Entropy and Compression

Posted on 2020-06-07 Edited on 2021-07-11

One interpretation of entropy of a random variable is as a measure of how many unbiased, independent random bits in expectation we can extract from the random variable.
Another one comes from compression of a sequence.

Assume that the sequence studied (denoted as sss) is a binary one.
It consists of a concatenation of the outcomes of nnn independent Bernoulli random variables.
Without lose of generality, we assume that each bit comes up 111 with probability p≤1/2p \le 1 / 2p≤1/2.
This is one of the simplest models of a sequence.

The fact that sequence could be a biased one makes it possible to represent it by a new sequence with shorter length in expectation.
For example, suppose that p=14p = \frac{1}{4}p=41​, then for a pair of consecutive bits, it have

  1. probability 116\frac{1}{16}161​ of being 111111,
  2. probability 316\frac{3}{16}163​ of being 010101,
  3. probability 316\frac{3}{16}163​ of being 101010,
  4. probability 916\frac{9}{16}169​ of being 000000.

If we use 111111111 to represent 111111, 110110110 to represent 010101, 101010 to represent 101010, and 000 to represent 000000, then the expected number of representation bits per pair is

3⋅116+3⋅316+2⋅316+1⋅916=2716<2\begin{array}{r} 3 \cdot \frac{1}{16} + 3 \cdot \frac{3}{16} + 2 \cdot \frac{3}{16} + 1 \cdot \frac{9}{16} = \frac{27}{16} < 2 \end{array} 3⋅161​+3⋅163​+2⋅163​+1⋅169​=1627​<2​

In general, compression is not limited to the way described above.
We call the rule with which we compress a string a compression function.

Compression Function

Definition. Let S={0,1}nS = \{0, 1\}^nS={0,1}n be the set of binary sequence of length nnn, and T={0,1}+T = \{ 0, 1\}^+T={0,1}+ the set of binary sequence of any positive length. A compression function h:S→Th: S \rightarrow Th:S→T is an injective (one-to-one) function from SSS to TTT.

In other words, each s∈Ss \in Ss∈S is assigned a unique non-empty sequence (of arbitrary length) by hhh.

Observe that the size of SSS is 2n2^n2n. On the other hand, the number of sequences with length less than nnn is ∑i=1n−12i=2n−1<2n\sum_{i = 1}^{n - 1} 2^{i} = 2^n - 1 < 2^n∑i=1n−1​2i=2n−1<2n.
For any hhh, it is impossible for hhh to map every s∈Ss \in Ss∈S to a sequence with length less than nnn.
Under adversarial input, hhh can not compress at all.

The hope is that when there is a distribution D\mathfrak{D}D on SSS, the expected length of the compressed sequences would be small:

E[∣h(s)∣]=∑s∈Sps⋅∣h(s)∣ \mathbf{E} [ |h(s)| ] = \sum_{s \in S} p_s \cdot |h(s)| E[∣h(s)∣]=s∈S∑​ps​⋅∣h(s)∣

To illustrate the meaning of entropy, we study a simple case where D\mathfrak{D}D is the joint distribution of nnn i.i.d. Bernoulli random variables that come up 111 with probability p≤1/2p \le { 1 / 2 }p≤1/2 (by symmetry, the claim holds for the case of p>1/2p > { 1 / 2 }p>1/2).

The following theorem formalizes how good a compression function we can find.

Entropy as Lower and Upper Bounds

Theorem.

  1. ∀δ>0\forall \delta > 0∀δ>0, ∃\exists∃ a compression hhh and integer N>0N > 0N>0, s.t., ∀n≥N\forall n \ge N∀n≥N, it holds that

E[∣h(s)∣]≤(1+δ)nH(p) \mathbf{E}[ |h(s)| ] \le (1 + \delta) n \mathbf{H}(p) E[∣h(s)∣]≤(1+δ)nH(p)

  1. ∀δ>0\forall \delta > 0∀δ>0, ∃\exists∃ integer N>0N > 0N>0, s.t., ∀n≥N\forall n \ge N∀n≥N, it holds that for any compression hhh,

E[∣h(s)∣]≥(1−δ)nH(p) \mathbf{E}[ |h(s)| ] \ge (1 - \delta) n \mathbf{H}(p) E[∣h(s)∣]≥(1−δ)nH(p)

Intuitively, the entropy H(p)\mathbf{H}(p)H(p) is the measure of the average number of bits in expectation we need after compression to represent a Bernoulli random variable that comes up 111 with probability ppp.

The high level idea of the proof is that, with high probability, the number of ones in a s∈Ss \in Ss∈S is roughly npnpnp. There are roughly 2nH(p)2^{n\mathbf{H}(p)}2nH(p) such sequences. Therefore, we can use nH(p)n \mathbf{H}(p)nH(p) bits to represent each sequence.

Huffman Code

In this section, we show that the upper bound can be achieved by Huffman code.

∀δ>0\forall \delta > 0∀δ>0, ∃\exists∃ a compression hhh and integer N>0N > 0N>0, s.t., ∀n≥N\forall n \ge N∀n≥N, it holds that

E[∣h(s)∣]≤(1+δ)nH(p) \mathbf{E}[ |h(s)| ] \le (1 + \delta) n \mathbf{H}(p) E[∣h(s)∣]≤(1+δ)nH(p)

We begin with an important property of Huffman Code. Suppose that we have an alphabet U\mathbb{U}U such that probability associated with each element in U\mathbb{U}U is 2−l2^{-l}2−l for some integer lll. Then ∃h\exists h∃h, such that

EX∈U[∣h(X)∣]=nH(X) \mathbf{E}_{X \in \mathbb{U} } [ |h(X)| ] = n \mathbf{H}(X) EX∈U​[∣h(X)∣]=nH(X)

For example, if U={0,1,2,3}\mathbb{U} = \{0, 1, 2, 3 \}U={0,1,2,3} and the distribution p={0.5,0.25,0.125,0.125}p = \{ 0.5, 0.25, 0.125, 0.125 \}p={0.5,0.25,0.125,0.125}, then we can have the following encoding

The expected coding length is

0.5⋅1+0.25⋅2+2⋅0.125⋅3=1.750.5 \cdot 1 + 0.25 \cdot 2 + 2 \cdot 0.125 \cdot 3 = 1.75 0.5⋅1+0.25⋅2+2⋅0.125⋅3=1.75

which is exactly the entropy of the random variable.

In general, if we have a random variable XXX, we can round down its probability to the nearest integer negative power of 222. The expected code length is given by

∑ipi⌈log⁡1pi⌉≤H(p)+1\sum_{i } p_i \left\lceil \log \frac{1}{p_i} \right\rceil \le \mathbf{H}(p) + 1 i∑​pi​⌈logpi​1​⌉≤H(p)+1

Intuitively, the ⌈log⁡1pi⌉\left\lceil \log \frac{1}{p_i} \right\rceil⌈logpi​1​⌉ has enough slot to accommodate all elements with probabilities pip_ipi​’s.

In particular, we view {0,1}n\{0, 1\}^n{0,1}n as a large alphabet Σ\mathbf{\Sigma }Σ. The alphabet has entropy n⋅H(p)n \cdot \mathbf{H}(p)n⋅H(p). We can have a encoding such that the expected output length is at most n⋅H(p)+1n \cdot \mathbf{H}(p) + 1n⋅H(p)+1.
