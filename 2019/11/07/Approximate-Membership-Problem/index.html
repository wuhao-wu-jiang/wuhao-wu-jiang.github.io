Approximate Membership Problem

Posted on 2019-11-07 Edited on 2020-04-06

In the approximate membership problem, we have subset S⊂US \subset US⊂U, that supports two operations:

  1. update(xxx): S←S∪{x}S \leftarrow S \cup \{x\}S←S∪{x}
  2. query(xxx): return True if x∈Sx\in Sx∈S else False.

where UUU is the domain that elements xxx belongs to. If U=232U = 2^{32}U=232, then each representation of xxx takes 323232 bits. If U=264U = 2^{64}U=264, then each xxx needs 646464 bits. If UUU is the website address, the length of its elements might is unlimited. Suppose that the average length is about 151515 chars, and each char takes 888 bits, then the representation could occupy most 120120120 bits.

We can solve the membership problem exactly by dictionaries, such as balanced-binary search tree and hash set. This requires space O(nlog⁡U)O(n \log U)O(nlogU) bits. If we allows false positive, we can break this bound.

The idea is to have a bit array AAA with length O(kn)O(kn)O(kn) bits and a pairwise hash family HHH that maps the elements in UUU uniformly at random to each bit in the array. After picking a hash function uniformly at random from HHH, the operation on SSS is performed as follows:

  1. update(xxx): A[h(x)]←1A[h(x)] \leftarrow 1A[h(x)]←1
  2. query(xxx): return True if A[h(x)]=1A[h(x)] = 1A[h(x)]=1 else False.
