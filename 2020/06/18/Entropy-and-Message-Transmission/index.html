Entropy and Message Transmission

Posted on 2020-06-18 Edited on 2020-06-30

We discuss another application that sheds light on meaning of entropy, the message transmission problem. In this model, a sender transmits a message to the receiver through a channel. In real world, the channel could be an optical fiber, a wireless channel, a hard disk etc. In the final application, the computer that writes and reads information from the hard disk is both the sender and the receiver.

In real world the channel is not perfect and the message transmitted is distorted by noise. It is natural to ask whether the message can be transmitted accurately under the noise, i.e., whether the receiver can recover the noisy message.

To study the problem, we need a mathematical model for both the channel and the noise. We focus on the simplest binary channel with bits of 0 and 1. The noise causes the bits to flip and is modelled by the distribution on the bit-flips. We study the simplest one that flips each bit independently with some identical probability p<0.5p < 0.5p<0.5. For a message with length nnn, the number of bit flips follows a Binomial distribution B(n,p)B(n, p)B(n,p). We call such channel a binary symmetric channel.

To protect the message against noise, a naïve way is to send the each bit multiple times. For example, if the sender wants to send a bit 1, it sends 101010 copies of 111 as 111111111111111111111111111111. The receiver decides that the bit sent is 1, if the majority of the 101010 bits it receives is 1.

Could it be improved? Repeating each bit may not be the mot efficient way against the noise. In general, if the sender wants to send a message of kkk bits, it can convert it into a new message of nnn bits and sends the new one. The receiver considers the nnn bits received as a whole, and try to recover the kkk-bit message the sender wants to send.

We call the method used by the sender to convert the original message the encoding function, and the one used by the receiver to recover the message the decoding function.

Definition. A (k,n)(k, n)(k,n) encoding function Enc:{0,1}k→{0,1}n\text{Enc}: \{0, 1\}^k \rightarrow \{0, 1\}^nEnc:{0,1}k→{0,1}n and a (k,n)(k, n)(k,n) decoding function Dec:{0,1}n→{0,1}k\text{Dec}: \{0, 1\}^n \rightarrow \{0, 1\}^kDec:{0,1}n→{0,1}k, where n≥kn \ge kn≥k.

Since the noise flips the bits randomly, it is impossible to have an encoding function and a decoding function without error (unless p=0p = 0p=0). Instead, we aim to control the probability of error within some specified threshold δ\deltaδ. To achieve this, we add redundancy to the message and encode a kkk-bit one into an nnn-bit one. Now, n−kn - kn−k is the amount of redundancy introduced. We would like to make n−kn - kn−k as small as possible. On the other hand, the value of n−kn-kn−k should positively related to ppp. The larger ppp is, the noisier the channel is and the larger n−kn - kn−k should be.

For fixed δ\deltaδ and ppp, the Shannon’s Theorem says that the smallest possible value of n−kn - kn−k we can achieve is roughly nH(p)n\mathbf{H}(p)nH(p). Intuitively, H(p)\mathbf{H}(p)H(p) is the amount of randomness the noise exerts on each bit. Therefore 1−H(p)1 - \mathbf{H}(p)1−H(p) is the maximum amount of information we can transmit by one bit.
