Divided by Three

Posted on 2019-09-18 Edited on 2019-09-21

Given a number x=(x1x2x3...xn)10x = (x_1x_2x_3...x_n)_{10}x=(x1​x2​x3​...xn​)10​ in decimal form, what is reminder divided by 3.

Note that for any positive integer kkk,

10k≡1k≡1mod  310^k \equiv 1^k \equiv 1 \mod 3 10k≡1k≡1mod3

Therefore,

x=x110n−1+x210n−2+...+xn≡x1+x2+...+xnmod  3x = x_1 10^{n - 1} + x_2 10^{n - 2} + ... + x^n \equiv x_1 + x_2 + ... + x_n \mod 3 x=x1​10n−1+x2​10n−2+...+xn≡x1​+x2​+...+xn​mod3

□\square□.

Primality Test

Posted on 2019-09-03 Edited on 2019-12-02

Although there is no polynomial algorithm for computing the prime decomposition of an integer, testing primality can be done in polynomial time. The central idea of is to find a certificate that we can verify in polynomial time for the compositeness of an integer.

We begin with Fermat’s Test, which gives false positive but works for most cases. It relies on the Fermat’s Little Theorem and Lagrange Theorem.

Fermat’s Little theorem.

If nnn is a prime number, then for any x∈[n−1]x \in [n - 1]x∈[n−1], we have

xn−1≡1mod  nx^{n - 1} \equiv 1 \mod n xn−1≡1modn

where throughout the article we use [n−1][n - 1][n−1] to denote the integer set {1,2,...,n−1}\{1, 2, ..., n - 1\}{1,2,...,n−1}.

Proof: Consider the set of numbers ⟨x,2x,3x,...,(n−1)x⟩\langle x, 2x, 3x, ..., (n - 1)x \rangle⟨x,2x,3x,...,(n−1)x⟩. Note that all numbers are different in terms of mod  n\mod nmodn. Otherwise ix≡jxmod  nix \equiv jx \mod nix≡jxmodn implies that (i−j)x≡0mod  n(i - j) x \equiv 0 \mod n(i−j)x≡0modn. But i≠ji \neq ji=j and x≠0x \neq 0x=0 concludes that nnn in not prime, a contradiction. ■\blacksquare■

Algorithm : Fermat’s Test.

  1. Input: an integer nnn.
  2. Output: whether nnn is a prime number.
  3. Take a number of x∈[n−1]x \in [n - 1]x∈[n−1] uniformly at random.

Theorem

Proof: x2≡1mod  nx^2 \equiv 1 \mod nx2≡1modn implies (x−1)(x+1)≡0mod  n(x - 1)(x + 1) \equiv 0 \mod n(x−1)(x+1)≡0modn. As nnn is a prime, x−1≡0mod  nx - 1 \equiv 0 \mod nx−1≡0modn or x+1≡0mod  nx + 1 \equiv 0 \mod nx+1≡0modn. ■\blacksquare■

Observe that when n>2n > 2n>2, n−1≠1n - 1 \neq 1n−1=1.

Algorithm 2: Rabin Miller Test

  1. Input: an integer nnn.

  2. Output: whether nnn is a prime number.

  3. If n≠2n \neq 2n=2 and nnn is even, output “NO”.

  4. If nnn is the power of some integer, output “NO”.

  5. Find the maximum integer kkk, such that n−1=2kmn - 1 = 2^k mn−1=2km for some odd number mmm.

  6. Take a number of x∈[n−1]x \in [n - 1]x∈[n−1] uniformly at random.

  7. Check whether xn−1≡1mod  nx^{n - 1} \equiv 1 \mod nxn−1≡1modn. If not, output “NO”. [Now Fermat’s Test Is Passed. Need Further Verification.]

  8. Otherwise, compute the certificate

    C=(xm,x2m,x4m,...,x2k−1m,x2km)C = (x^m, x^{2m}, x^{4m}, ..., x^{2^{k - 1}m}, x^{2^km}) C=(xm,x2m,x4m,...,x2k−1m,x2km)

    We must have x2km≡1x^{2^km} \equiv 1x2km≡1 since it passes Fermat’s test. We check whether ∃i\exists i∃i, such that x2im≠±1x^{2^i m} \neq \pm 1x2im=±1 but x2i+1m=1x^{2^{i + 1} m} = 1x2i+1m=1. If so, we find a pseudo square root of unity and output “NO”.

  10. Output “YES”.

We claim that line 4 runs in polynomial time. If nnn is the power of some integer, the smallest integer is 222, which gives the power value log⁡n\log nlogn. Therefore we need only to check for each integer i∈[log⁡n]i \in [\log n]i∈[logn] whether it holds that n1/in^{1/i}n1/i is a integer, which can be computed in polynomial time.

A set of kkk servers are receiving requests and they are required to report to a coordinator the numbers of requests they process. An event is trigerrd by the coordinator if the culmulative requests received by the serves reaches a threshold τ\tauτ. The goal is to minimized the communicaiton between the coordinator and the servers.

A naive approach asks the servers to send a notificiation to the coordinator each time they process a request, resulting in O(τ)O(\tau)O(τ) bits communication in total.

An alternative approach achieves O(hlog⁡τ)O(h \log \tau)O(hlogτ) words communication cost. Instead of response to the coordinator every a new request comes, the server reports only after some requests reaches. However, we don’t want the coordinator to miss the τ\tauτ threshold, i.e., it must triger the event when exactly τ\tauτ requests come in total.

Here is the scheme.

  1. If τ≤5⋅h\tau \le 5 \cdot hτ≤5⋅h, the server report each time a request comes.
